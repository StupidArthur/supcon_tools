# UA 自动化测试与工具页面实施计划

> 面向执行 Agent。目标是在现有 `ua_test_gui + ua_mocker + ua_test_harness + tpt_api` 基础上，落地完整的 API 自动化测试系统，并在桌面工具中提供：
>
> 1. **测试任务页面**：选择用例、触发任务、实时查看进度、停止任务、查看实时日志；
> 2. **测试用例页面**：查看全部 UA-1～UA-3 用例、筛选、查看详情、查看实现状态；
> 3. **历史结果页面**：查看历史任务、汇总、逐用例结果、步骤、日志、证据、性能指标和清理结果；
> 4. **新版 Mock 能力**：支持当前全部测试用例需要的静态节点、变化节点、可写节点、断线、异常、性能和历史造数场景；
> 5. **Python 自动化执行器**：按文档中的稳定用例 ID 执行，支持回归、探索、响应时间和性能测试。

本计划按“先打通骨架，再逐步补齐用例”的顺序执行。不要直接一次性写完 115 个用例后再集成 GUI；
