# `review3` 阶段 0～8 锁定验收体系建设计划

## 一、任务身份

你是本项目的“验收基础设施 Agent”，不是业务功能实现 Agent。

你的任务是继续完成：

```text
review3/tools/stage_verification/
```

下已经开始但尚未完全收口的阶段验证体系，为“单阀门二阶水箱模板”的阶段 0～8 建立：

1. 独立的 reviewer-authored acceptance suite；
2. 可执行的 manifest 门禁；
3. reviewer-owned baseline；
4. 自动验收结果；
5. 人工验收 attestation；
6. 阶段完成后的 accepted checkpoint；
7. 下一阶段开始前的锁定入口。

在本任务全部完成之前：

* 不得继续实现阶段 5 的在线调参业务；
* 不得实现趋势页面、批量页面或其他阶段 6～8 业务；
* 不得以“顺便修复”为理由修改水箱、阀门、PID 核心算法；
* 不得修改内置模板的业务内容；
* 不得为了让验收通过而放宽原有契约；
* 不得删除已有测试、降低断言或增加 skip/xfail。

# 五、第一批工作：验收器 Bootstrap

在写九阶段套件前，先证明验收器自身可靠。

## 5.1 修复项目根目录识别

当前验证器不能只向上寻找 `.git`。

实现以下概念：

```python
@dataclass(frozen=True)
class RepositoryLayout:
    git_root: Path
    project_root: Path
    verifier_root: Path
```

项目根目录识别规则：

1. 显式传入 `--repo-root` 时，校验该目录包含：

   * `tools/stage_verification/manifests`
   * `todo/second_order_tank_implementation_playbook.md`
   * `config-tool`
2. 未传入时，从 `verify_stage.py` 所在目录向上寻找上述项目标志；
3. 再单独调用 Git 获得 `.git` 根目录；
4. 允许 `project_root != git_root`；
5. 所有 manifest 路径、cwd 和文件快照均相对于 `project_root`；
6. Git 命令可以在 `project_root` 执行，但不得误把同仓库其他项目纳入快照。

增加测试：

```text
test_project_root_can_be_below_git_root
test_explicit_repo_root_must_contain_project_markers
test_manifest_path_is_resolved_from_project_root
test_snapshot_does_not_include_sibling_projects
test_windows_backslash_paths
test_unicode_project_path
```

## 5.2 统一 Manifest 校验真源

不得继续同时维护互相可能漂移的：

```text
manifest.schema.json
validate_manifest() 手工规则
```

优先方案：

* 使用 `jsonschema` 校验 `manifest.schema.json`；
* 在 Python 层只补充 schema 无法表达的跨字段规则。

必须检查：

```text
stage 与文件名一致
command id 唯一
gate id 唯一
自动 gate 的 checks 非空
每个 gate 引用的 check 都能实际生成
manual gate 不得包含 automated-only 字段
locked_acceptance_paths 非空
allowed_paths 与 forbidden_paths 存在明显冲突时配置失败
命令 cwd 不允许逃出 project_root
json-output 不允许污染工作树
```

增加以下负面测试：

```text
unknown manifest property rejected
duplicate command id rejected
duplicate gate id rejected
missing locked_acceptance_paths rejected
invalid stage rejected
unknown gate check rejected
cwd path traversal rejected
manifest modified after baseline rejected before command execution
```

* manifest 被修改时不得执行命令；
* acceptance test 被修改、删除或重命名时失败；
* evidence 文件被修改后 attestation 失效；
* final tree 变化后 attestation 失效；
* 命令超时必须清理整个进程树；
* 固定端口不得残留；
* JSON 结果不得污染下一轮 scope；
* untracked 文件也必须进入范围和空白检查；
* 不得通过重新生成普通 SHA 掩盖 reviewer HMAC 失配。

增加 Windows 进程树测试：

```text

父进程超时
验证器终止父子进程
端口在限定时间内可重新绑定
```
