# 精馏塔参考稳态最终收尾修复指令

## 1. 背景与当前状态

上一轮已经完成精馏塔稳态求解器及动态验证机制。审查者完成了 §21 完整验收，结果如下：

```text
精馏塔专项测试：62 passed
least_squares 最大缩放残差：2.58e-7
1800 秒连续稳态窗口：通过
3600 秒阀位模式漂移：通过
直接流量模式/阀位模式等价性：通过
xD：0.85
xB：0.015
乙醇回收率：95.69%
完整生成耗时：约 32 秒
```

核心机理和稳态求解已经通过，不要重新设计模型，不要重新调试阀门/PID，也不要进入长时间试参循环。

本轮只处理以下三类收尾问题：

1. 快速验证模式可能生成“伪通过”的参考状态；
2. `STEADY` 严格加载器的门禁不完整；
3. `todo/5.md` 中固定回流比 `R=3` 与当前理想级模型不相容，需要修正规格口径。

## 2. 本轮统一决策：回流比不再作为硬约束

### 2.1 已确认的模型结果

在当前模型中，同时满足：

```text
12 块理想理论板
F = 100 kg/h
zF = 0.25
xD = 0.85
xB = 0.015
Ptop = 101.325 kPa
回流罐液位 = 50%
塔釜液位 = 50%
```

时，稳态求解器得到：

```text
R/D = 0.6158988
```

如果强制：

```text
R/D = 3.0
xB = 0.015
```

模型得到：

```text
xD ≈ 0.919345
```

强制同时满足 `R/D=3、xD=0.85、xB=0.015` 时，当前方程组不可闭合。

### 2.2 最终处理原则

本版本按以下优先级执行：

```text
一级硬目标：
F、zF、xD、xB、Ptop、回流罐液位、塔釜液位

求解结果：
回流流量 R、蒸汽流量、冷却水流量、各板状态

初猜：
R/D = 3.0 只作为 least_squares 初始猜测，不作为稳态残差约束
```

不要向稳态残差中增加 `R/D=3` 方程，不要为了得到 `R=3` 引入塔板效率，不要改变塔板数量，也不要改变 VLE 或动态机理。

第一版接受求解器给出的实际参考回流比约 `0.6159`。

如果以后需要现场化标定，再单独讨论实际塔板数、Murphree 效率和回流比范围，不属于本轮。

## 5. 修改三：加强 least_squares 求解结果门禁

文件：

```text
tools/generate_ethanol_water_reference_state.py
```

当前只检查：

```python
result.status > 0
```

这不够。正状态只表示求解器满足某个停止条件，不代表残差一定合格。

建议提取一个小函数，便于单元测试，例如：

```python
def validate_solver_result(result) -> None:
    residuals = np.asarray(result.fun, dtype=np.float64)

    if result.status <= 0:
        raise ReferenceStateGenerationError(...)

    if not np.all(np.isfinite(result.x)):
        raise ReferenceStateGenerationError(
            "least_squares 解向量包含 NaN/Inf"
        )

    if residuals.size == 0 or not np.all(np.isfinite(residuals)):
        raise ReferenceStateGenerationError(
            "least_squares 残差包含 NaN/Inf"
        )

    max_scaled_residual = float(np.max(np.abs(residuals)))
    if max_scaled_residual > 1.0:
        raise ReferenceStateGenerationError(
            f"least_squares 最大缩放残差超限: "
            f"{max_scaled_residual} > 1.0"
        )
```

这里的 `1.0` 对应每一项残差不超过其定义的验收缩放值。

错误信息至少包含：

```text
status
message
nfev
cost
optimality
max_scaled_residual
```

不要调整现有残差缩放，不要重新调 `ftol/xtol/gtol`，也不要增加 `max_nfev`。

## 6. 修改四：`check_convergence()` 必须拒绝 NaN/Inf

文件：

```text
tools/generate_ethanol_water_reference_state.py
```

当前代码直接执行：

```python
if value > threshold:
    return False
```

但对于 `NaN`：

```python
NaN > threshold
```

结果为 `False`，可能错误地绕过门禁。

### 6.1 在任何范围比较前检查有限性

`check_convergence()` 开头列出全部必需字段：

```python
required_metrics = [
    "max_abs_dM_tray_dt",
    "max_abs_dnE_tray_dt",
    "abs_dM_drum_dt",
    "abs_dM_sump_dt",
    "abs_dN_vapor_dt",
    "abs_dP_top_dt",
    "max_abs_dT_dt",
    "drum_level_pct",
    "sump_level_pct",
    "top_pressure_kpa",
    "top_ethanol_wt",
    "bottom_ethanol_wt",
    "ethanol_recovery_pct",
    "mass_residual_rel",
    "ethanol_residual_rel",
    "energy_residual_rel",
]
```

逐项要求：

```python
if key not in metrics:
    return False

try:
    value = float(metrics[key])
except (TypeError, ValueError):
    return False

if not math.isfinite(value):
    return False
```

完成有限性检查后，再执行现有阈值判断。

第一周期导数为 `NaN` 的设计可以保留，但该周期必须明确返回 `False`。
