## 2. 已确认的根因

### 2.1 当前压力回路没有因果作用

当前实现等价于：

```python
V = vapor_boilup
V_condense = V
dN_vapor = vapor_boilup - V_condense
```

所以正常计算中：

```text
dN_vapor = 0
```

冷却水阀只更新开度，没有参与 `Q_C`、`V_condense` 或 `dN_vapor`。压力 PID 虽然连接到 `cooling_valve_pct`，但 MV 对 PV 没有因果作用。

### 2.2 蒸汽流量与过程蒸气量被错误地当成同一个量

当前 `steam_valve_pct` 直接决定 `vapor_boilup_kgmol_per_s`，DSL 又把 `vapor_boilup_kg_h` 当作蒸汽流量副回路 PV。

必须拆成：

```text
steam_flow_kg_h       公用工程蒸汽质量流量，DCS 可测
reboiler_duty_kw      蒸汽冷凝提供的热负荷
vapor_boilup_kgmol_s  塔釜产生的过程乙醇—水蒸气量，模型内部量
vapor_boilup_kg_h     过程蒸气等效质量流量，诊断量
```

蒸汽流量副 PID 的 PV 必须接 `column_1.steam_flow_kg_h`，不能接过程气相流量。

### 2.3 所谓 STEADY 只是线性初猜

当前 `initialization_mode=STEADY` 只建立线性浓度剖面，没有伪时间松弛、残差判定或参考状态自动加载。`auto_save_reference_state` 也没有有效调用链。

### 2.4 DSL 启动时 PID 把标称阀位拉向零

模型阀门按标称流量初始化，但 DSL 中多数 PID 的初始 `MV=0`。第一个周期后，PID 覆盖模型标称阀位，造成回流、采出、蒸汽和冷却立即偏离参考工况。

### 2.5 物理越界被显示限幅掩盖

当前内部回流罐存量可以超过最大存量数倍，而发布位号时只把液位钳制为 100%。这会产生物理上不存在的数据。

## 4. 阀门模型修正

### 4.1 等百分比阀必须能全关

当前公式：

\[
f(x)=R^{x-1}
\]

在 `x=0` 时仍有 `1/R` 流量，不能作为具有切断能力的过程阀正式模型。

改为归一化等百分比特性：

\[
f(x)=\frac{R^x-1}{R-1},\quad 0\le x\le1
\]

满足：

```text
f(0)=0
f(1)=1
```

反函数：

\[
x=\frac{\ln(1+f(R-1))}{\ln R}
\]

必须同步修改：

```text
ValveActuator.characteristic_function()
_compute_initial_valve_pct()
相关单元测试
```

### 4.2 阀门额定流量统一使用工业侧质量流量

阀门额定流量固定为：

| 阀门 | 特性 | 额定流量 | 满行程时间 |
|---|---|---:|---:|
| 进料 | 等百分比 | 150 kg/h | 10 s |
| 回流 | 等百分比 | 150 kg/h | 10 s |
| 塔顶采出 | 线性 | 50 kg/h | 12 s |
| 塔底采出 | 线性 | 120 kg/h | 12 s |
| 蒸汽 | 等百分比 | 100 kg/h | 15 s |
| 冷却水 | 等百分比 | 7000 kg/h | 15 s |

过程液体质量流量进入塔模型前，根据本周期流股组成换算成 kmol/s：

```text
feed:       使用 feed_ethanol_wt
reflux:     使用 reflux drum 组成
```

公用工程蒸汽和冷却水保持 kg/h，不转换成过程混合物 kmol/s。

## 5. 再沸器模型

### 5.1 蒸汽阀产生公用工程蒸汽流量

```python
steam_flow_kg_h = STEAM_MAX_FLOW_KG_H * steam_valve.flow_fraction
```

可用热负荷：

\[
Q_{R,available}=\frac{\dot m_{steam}}{3600}
\Delta H_{steam}\eta_R
\]

单位为 kW。

### 5.2 过程蒸发量由热量决定

每个内部子步计算塔釜泡点：

```python
T_bubble_sump, y_boil = bubble_point_temperature(x_sump, p_sump)
```

定义：

```python
Q_loss_sump = UA_sump * max(T_sump - T_ambient, 0.0)
Q_subcool = min(
    max(Q_R_available - Q_loss_sump, 0.0),
    M_sump * Cp_liquid * max(T_bubble_sump - T_sump, 0.0) / tau_sump_heat,
)
Q_superheat_release = (
    M_sump * Cp_liquid * max(T_sump - T_bubble_sump, 0.0) / tau_phase
)
Q_for_vaporization = max(
    0.0,
    Q_R_available - Q_loss_sump - Q_subcool + Q_superheat_release,
)
V_boil = Q_for_vaporization / delta_h_vap_sump
```

固定：

```python
tau_sump_heat_s = 30.0
tau_phase_s = 2.0
```

要求：

- 塔釜明显低于泡点时，蒸汽热量优先升温；
- 接近泡点后才形成正常过程蒸气；
- 蒸汽阀加大必须先增加 `steam_flow_kg_h` 和 `Q_R`，随后增加 `V_boil`；
- 不允许再由外部直接输入 `vapor_boilup` 作为正式 DSL 控制路径；
- 直接 `vapor_boilup_kgmol_per_s` 只可保留为测试/兼容模式，并明确标注 bypass。

塔釜能量衡算必须使用实际 `Q_R_available`，不能再通过目标 `V_boil` 反推一个恰好抵消汽化项的 `Q_R`。

## 6. 冷凝器、气相存量和压力模型

### 6.1 必须增加的气相状态

增加三个状态：

```text
N_vapor       气相总物质量，kmol
nE_vapor      气相乙醇物质量，kmol
U_vapor       气相总内能，kJ
```

派生：

```python
yE_vapor = nE_vapor / N_vapor
T_vapor = temperature_from_vapor_internal_energy(U_vapor, N_vapor, yE_vapor)
P_top = N_vapor * R * T_vapor / vapor_volume_m3
```

只维护 `N_vapor` 而不维护 `nE_vapor` 会破坏动态乙醇守恒；只维护物质量而不维护气相能量会让压力温度耦合失真。因此三个状态必须一起实现。

### 6.2 气相焓和内能辅助函数

允许修改 `components/thermo/ethanol_water.py` 中的能量辅助函数，VLE 部分保持冻结。

纯组分气相焓改为：

\[
h_{v,i}(T)=C_{p,l,i}(T_{b,i}-T_{ref})+
\Delta H_{vap,i}(T_b)+C_{p,v,i}(T-T_{b,i})
\]

固定：

```python
T_BOIL_WATER_K = 373.15
T_BOIL_ETHANOL_K = 351.50
CP_VAPOR_WATER_KJ_PER_KMOL_K = 33.6
CP_VAPOR_ETHANOL_KJ_PER_KMOL_K = 65.6
R_KJ_PER_KMOL_K = 8.314462618
```

混合气相：

```python
h_vapor = (1-yE) * h_vapor_water(T) + yE * h_vapor_ethanol(T)
u_vapor = h_vapor - R * T
```

新增：

```python
def vapor_internal_energy_kj_per_kmol(y_ethanol, temperature_k): ...

def temperature_from_vapor_internal_energy(
    internal_energy_kj,
    total_moles_kgmol,
    y_ethanol,
) -> float: ...
```

由于本阶段使用常数热容，反算温度必须使用解析式，不要增加逐周期求根开销。

### 6.3 冷却能力

冷却水实际质量流量：

```python
cooling_flow_kg_h = COOLING_WATER_MAX_FLOW_KG_H * cooling_valve.flow_fraction
```

两项能力上限：

\[
Q_{flow}=\frac{\dot m_{cw}}{3600}C_{p,cw}\Delta T_{cw,max}
\]

\[
Q_{UA}=UA_C\max(T_{vapor}-T_{cw,in},0)
\]

\[
Q_{C,available}=\min(Q_{flow},Q_{UA})
\]

冷凝单位摩尔需移除的热量：

```python
T_condensate, _ = bubble_point_temperature(yE_vapor, P_top)
h_v = vapor_enthalpy_kj_per_kmol(yE_vapor, T_vapor)
h_l = liquid_enthalpy_kj_per_kmol(yE_vapor, T_condensate)
delta_h_condense = max(h_v - h_l, 1.0)
```

冷凝能力：

```python
V_condense_capacity = Q_C_available / delta_h_condense
```

### 6.4 实际冷凝量必须允许消耗气相库存

不能使用 `V_condense=min(V_top, capacity)`，否则冷凝量永远不能超过当前进塔蒸气量，压力升高后无法主动降低气相库存。

采用：

```python
N_floor = P_VAPOR_FLOOR_KPA * vapor_volume_m3 / (R * T_vapor)
inventory_release_capacity = max(N_vapor - N_floor, 0.0) / tau_condenser_inventory_s
vapor_available = V_top + inventory_release_capacity
V_condense = min(V_condense_capacity, vapor_available)
```

固定：

```python
P_VAPOR_FLOOR_KPA = 70.0
tau_condenser_inventory_s = 5.0
```

实际冷凝负荷：

```python
Q_C_actual = V_condense * delta_h_condense
```

### 6.5 气相动态方程

令塔顶板离开的过程蒸气为 `V_top`、组成 `yE_top`、焓 `hV_top`：

\[
\frac{dN_v}{dt}=V_{top}-V_{condense}-V_{vent}
\]

\[
\frac{dn_{E,v}}{dt}=V_{top}y_{E,top}-
V_{condense}y_{E,v}-V_{vent}y_{E,v}
\]

气相控制体采用开口系统能量衡算：

\[
\frac{dU_v}{dt}=V_{top}h_{V,top}-
V_{condense}h_{V,v}-V_{vent}h_{V,v}-Q_{loss,v}
\]

冷凝器从气相带走的潜热已经通过“蒸气离开气相控制体、以液体进入回流罐”的焓差计入总系统。实现时必须选择一个一致的控制体，禁止同时在 `dU_vapor` 和总能量中重复扣除 `Q_C`。

推荐实现方式：

```text
气相控制体流出按 vapor enthalpy
回流罐流入按 condensate liquid enthalpy
两者差值 Q_C_actual 作为对外冷凝器负荷和全系统能量输出
```

### 6.6 回流罐入口必须使用实际冷凝量

改为：

```text
dM_drum/dt  = V_condense - reflux - distillate
dnE_drum/dt = V_condense*yE_vapor - (reflux+distillate)*xE_drum
```

回流罐入口液体焓使用冷凝液温度和 `yE_vapor` 组成。

不得继续假设进入回流罐的流量恒等于 `V_boil`。

## 8. 物理边界与失败语义

### 8.1 禁止广泛 clip

只允许修正绝对值不超过 `1e-12` 的浮点越界。

以下情况必须抛出带状态上下文的 `RuntimeError`，让引擎进入现有安全失败路径：

```text
M <= 0
nE < -1e-12
nE > M + 1e-12
N_vapor <= 0
nE_vapor < -1e-12
nE_vapor > N_vapor + 1e-12
温度不在 250～500 K
压力不在 50～160 kPa(a)
任意状态 NaN/Inf
```

### 8.2 液位量程和设备极限分开

保留：

```text
m_drum_100pct_kmol = 1.0
m_sump_100pct_kmol = 3.0
```

增加物理高高限：

```text
m_drum_trip_kmol = 1.05
m_sump_trip_kmol = 3.15
```

发布：

```python
raw_level_pct = M / M_100pct * 100
display_level_pct = min(max(raw_level_pct, 0.0), 100.0)
```

但 `raw_level_pct` 必须同时对外发布。若存量超过 trip 值，立即报错，不能继续以 100% 平线运行。

低低限：

```text
drum/sump raw level < 2%：报物理失败
```

正常闭环验收要求原始液位始终在 10～90%，稳态在 47～53%。

### 9.1 删除运行时自动保存


参考状态只能由显式离线工具生成：

```text

```

正式文件：

```text

```

### 9.2 初始化模式重新定义

```text
STEADY：必须加载参数哈希匹配、残差合格的参考状态；缺失或不匹配直接报错
WARM_GUESS：只建立线性浓度/泡点初猜，供稳态生成器使用，不得用于正式 DSL

```

不得再把线性初猜称为 STEADY。

### 9.3 稳态生成流程

生成器固定使用 `ClockMode.GENERATOR`，关闭分析仪噪声和漂移。

步骤：

1. 用第 3 节物料衡算建立 `F/D/B/R/V` 初猜；
2. 用 `WARM_GUESS` 建立 12 板浓度和泡点温度初猜；
3. 启用临时稳态调节器，而不是正式 PID：
   - 回流罐存量偏差修正 `D`；
   - 塔釜存量偏差修正 `B`；
   - 塔顶压力偏差修正冷却水流量；
   - 上部/下部组成偏差缓慢修正回流和蒸汽；
4. 用动态模型进行伪时间松弛；
5. 连续满足收敛窗口后保存状态；
6. 从保存文件重新加载，在正式标称阀位下再运行 3600 s，验证无漂移；
7. 输出控制初始化表并更新两套 DSL。

临时稳态调节器只存在于生成脚本中，不能放进正式模型。

### 9.4 收敛阈值

连续 1800 s 同时满足：

```text

max(abs(dnE_tray/dt))      <= 1e-9 kmol/s
abs(dM_drum/dt)            <= 1e-8 kmol/s
abs(dM_sump/dt)            <= 1e-8 kmol/s
abs(dN_vapor/dt)           <= 1e-9 kmol/s

abs(dT_any/dt)             <= 1e-4 K/s
47% <= raw drum level      <= 53%
47% <= raw sump level      <= 53%
abs(P_top-101.325)         <= 0.10 kPa
0.82 <= top ethanol wt     <= 0.88
0.010 <= bottom ethanol wt <= 0.020
ethanol recovery           >= 95%
质量闭合相对误差            <= 0.1%
乙醇闭合相对误差            <= 0.2%

```


## 10. PID 和 DSL 无扰初始化

### 10.1 启动原则

正式 DSL 第一个周期必须满足：

```text
模型阀门 command_pct = reference nominal command
模型阀门 actual_pct  = reference nominal actual
PID.MV                = 同一阀门 nominal command
PID.PV                = reference measured value
PID.SV/CSV            = reference target
```

不得出现“模型阀门按稳态开度启动，但 PID.MV=0”的双重初始状态。

### 10.2 固定初始种子

在稳态文件尚未生成前，使用以下种子；生成成功后，两套 DSL 改用 JSON 中的精确值：

| 回路 | PV | SV/CSV | MV |
|---|---:|---:|---:|
| feed_flow_pid | 100.0 kg/h | 100.0 | 88.56% |
| pressure_pid | 101.325 kPa | 101.325 | 81.85% |
| reflux_drum_level_pid | 50.0% | 50.0 | 56.29% |
| reboiler_level_pid | 50.0% | 50.0 | 59.88% |
| top_temp_pid | 由最终参考状态提供 | 同 PV | 由最终参考状态提供 |
| reflux_flow_pid | 由最终参考状态提供 | CSV=由最终参考状态提供 | 83.85% |
| bottom_temp_pid | 由最终参考状态提供 | 同 PV | 60.47 kg/h |
| steam_flow_pid | 60.47 kg/h | CSV=60.47 | 85.84% |
| top_quality_pid | 0.85 | 0.85 | 由最终参考状态提供 |
| bottom_quality_pid | 0.015 | 0.015 | 60.47 kg/h |

表中 `84.43 kg/h`、`83.85%` 等基于初始猜测回流比 R/D=3 的固定值仅作为阀门默认工况或求解初猜，**不能被描述为正式参考状态硬目标**。正式 PID 标称值（包括 reflux 流量、reflux 阀位、温度主回路的 MV 等）必须由最终参考状态的 `nominal_pid_values` 和 `control_initialization` 提供，不能把 `0.6158988` 反向写成代码里的固定流量。

温度主回路的准确 SV 必须取最终参考状态中的第 3、10 板温度。

### 10.3 串级关系

基础控制：

```text
top_temp_pid.MV       → reflux_flow_pid.CSV
reflux_flow_pid.MV    → reflux_valve_pct
bottom_temp_pid.MV    → steam_flow_pid.CSV
steam_flow_pid.MV     → steam_valve_pct
```

质量控制：

```text
top_quality_pid.MV    → reflux_flow_pid.CSV
bottom_quality_pid.MV → steam_flow_pid.CSV
```

修改：

```text
steam_flow_pid.PV → column_1.steam_flow_kg_h
pressure_pid.MV   → column_1.cooling_valve_pct
```

压力回路必须通过新冷凝模型产生实际作用。

不要修改 PID 标准算法来掩盖错误初值。优先修正 DSL 初始值和参考状态。如果现有 PID 提供状态跟踪接口，则在实例初始化时用 nominal MV 初始化内部跟踪项。

## 11. 守恒诊断修正

当前 `F-D-B` 是系统总质量积累率，不是动态守恒残差。正式输出需要区分：

```text
mass_accumulation_kg_h
mass_closure_residual_kg_h
ethanol_accumulation_kg_h
ethanol_closure_residual_kg_h
energy_accumulation_kw
energy_closure_residual_kw
```

定义：

\[
r_M=\frac{dM_{inventory}}{dt}-(F-D-B-V_{vent})
\]

\[
r_E=\frac{dn_{E,inventory}}{dt}-
(Fz_F-Dx_D-Bx_B-V_{vent}y_v)
\]

能量残差必须包含液相状态、气相 `U_vapor`、进出料、蒸汽热负荷、冷凝器移热、散热和放空。

为兼容现有上游位号：

```text
mass_balance_residual_kg_h    → 指向 closure residual
ethanol_balance_residual_kg_h → 指向 closure residual
energy_balance_residual_kw    → 指向 closure residual
```

新增 accumulation 位号，不要再把积累率命名为 residual。

## 17. 最终验收定义

只有同时满足以下条件，乙醇—水精馏塔才可进入 AI 数据生成阶段：

1. 阶段 A VLE 指标不退化；
2. 蒸汽和冷却水是独立的公用工程流量；
3. 冷却阀对压力具有经过测试的负向作用；

10. 至少一个大周期方案通过 0.5 s 参考轨迹等效性认证；
11. 离线生成速度达到最低性能目标；
12. 全量测试通过，仓库无运行时生成文件和未跟踪临时文件。
