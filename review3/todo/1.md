# PID 算法重构实施方案

## 1. 任务目标

将当前简易位置式 PID 保留为待删除的旧算法，并重新实现一个面向工业 AI 数据源、核心行为与中控 ECS-700 对齐的新 PID。

本次目标不是完整复刻 ECS-700 PID 功能块，而是完成以下内容：

1. 对齐 `PB/TI/TD` 的参数含义和核心增量式算法。
2. 增加工程量程与操作限幅。
3. 增加正反作用。
4. 增加 ECS-700 数值模式 `MODE`。
5. 支持 AUTO、CAS、手动类模式的基本运行逻辑。
6. 支持工业 AI 当前需要读取的全部位号。
7. 保证 MAN/AUTO/CAS 切换时不会发生明显输出突跳。
8. 增加基本抗积分饱和和不完全微分。
9. 保持现有 `type: PID` 配置继续指向新 PID。

参考资料：

* `pid_guide.pdf` 中“3.3 常规 PID 功能块”。
* 当前实现：`components/programs/pid.py`
* 算法基类：`components/programs/base.py`
* 注册入口：`components/programs/__init__.py`
* 周期调用与属性存储：`controller/expression.py` 中的 `AlgorithmNode.step()`
* OPC UA 写值落点：`controller/engine.py` 中的 `_apply_external_overrides()`

## 2. 旧 PID 的处理

### 2.1 文件重命名

将当前文件：

```text
components/programs/pid.py
```

移动为：

```text
components/programs/pid_delete.py
```

Python 文件名和类名不能使用连字符，因此代码层采用下划线；对外注册名可以保留 `PID-DELETE`。

### 2.2 类名和元数据

将旧类：

```python
class PID(BaseProgram):
```

改为：

```python
class PID_DELETE(BaseProgram):
```

同时修改：

```python
name = "pid_delete"
chinese_name = "旧版PID（待删除）"
```

旧算法的计算逻辑不要修改，只完成重命名和注册隔离，便于对照测试。

### 2.3 注册方式

旧 PID 不得再占用注册名 `PID`。

注册为：

```python
InstanceRegistry.register_algorithm("PID-DELETE", PID_DELETE)
```

可以额外注册 Python 风格别名：

```python
InstanceRegistry.register_algorithm("PID_DELETE", PID_DELETE)
```

不得将旧类继续注册到：

```python
InstanceRegistry.register_algorithm("PID", ...)
```

### 2.4 新 PID 文件

重新创建：

```text
components/programs/pid.py
```

其中定义：

```python

    ...
```

新类注册为：

```python

```

所有现有 YAML 中：


如需临时运行旧算法，使用：

## 5. AUTO、CAS 与 MODE 的关系

## 5.1 MODE 是唯一真实模式

采用 ECS-700 数值：

| MODE | 名称   | 本次实现行为               |
| ---: | ---- | -------------------- |
|    1 | OOS  | 保持最后有效 MV，不执行 PID    |
|    2 | IMAN | 按手动类模式处理             |
|    3 | TR   | 跟踪当前外部 MV，不执行 PID    |
|    4 | MAN  | 外部写 MV，PID 不自动计算     |
|    5 | AUTO | 使用本地 SV 自动计算         |
|    6 | CAS  | 使用 CSV 作为外给定自动计算     |
|    7 | RCAS | 本次简化为使用 CSV 的远程外给定模式 |
|    8 | ROUT | 本次简化为远程手动模式          |

### 约束

* 不实现 ECS-700 完整模式优先级、自动回退和权限仲裁。
* 不增加 `SWAM`、`SWSV`、`SWROUT`、`SWRCAS` 等模式切换开关。
* 模式直接由 `MODE` 数值决定。
* 非法模式值不得静默当成 AUTO。

## 5.2 AUTO 兼容位号

根据上游模板，`AUTO` 作为控制模式位号使用，并通过以下数值判断状态：

```text
5   -> 自动
6   -> 串级
7   -> 外给定
2;4 -> 手动
```

因此实现为：

```python
self.AUTO = int(self.MODE)
```

`AUTO` 是 `MODE` 的兼容别名，不是布尔值。

即：

```text
MODE=5 -> AUTO=5
MODE=6 -> AUTO=6
MODE=7 -> AUTO=7
MODE=4 -> AUTO=4
```

即使 OPC UA 客户端写入 `AUTO`，下一周期也必须由 `MODE` 重新覆盖，不得反向改变模式。

## 5.3 CAS 兼容位号

实现为当前是否处于本地串级模式：

```python
self.CAS = 1 if int(self.MODE) == 6 else 0
```

`MODE=7` 是 RCAS/外给定，仍通过 `AUTO=7` 识别，不将 `CAS` 置为 1。

`CAS` 是派生状态，不作为独立控制输入。

## 6. SV 与 CSV 的处理

### 6.1 AUTO

当：

```text
MODE=5
```

PID 使用本地 `SV`：

```python
effective_sv = self.SV
```

### 6.2 CAS 和 RCAS

当：

```text
MODE in {6, 7}
```

PID 使用：

```python
effective_sv = self.CSV
```

为了兼容上游工业 AI 始终读取 `.SV` 作为“当前设定值”的行为，在 CAS/RCAS 模式中，应让对外的 `SV` 反映当前有效设定值：

```python
self.SV = clamped_csv
```

但需要内部保存 AUTO 模式的本地设定值，例如：

```python
self._local_sv
```

行为要求：

* AUTO 模式下，外部写 `SV` 更新 `_local_sv`。
* CAS/RCAS 模式下，`CSV` 是计算来源。
* CAS/RCAS 模式下，对外 `SV` 镜像当前有效的 `CSV`。
* 从 CAS 返回 AUTO 时，恢复此前保存的 `_local_sv`。
* CAS 模式下写 `SV` 不得改变当前有效串级设定值。

如果 YAML 明确给 `execute(SV=...)` 传值，该值视为本地 SV 来源并更新 `_local_sv`。

## 8. 工程量程与操作限幅

## 8.1 SV/PV 工程量程

使用：

```text
SVSCL = PV/SV 工程量程下限
SVSCH = PV/SV 工程量程上限
```

转换为内部百分比：

[
PV_%=
100\cdot\frac{PV-SVSCL}{SVSCH-SVSCL}
]

[
SV_%=
100\cdot\frac{SV-SVSCL}{SVSCH-SVSCL}
]

不要将 PV 强制限制在 0～100%。测量值可以超量程，以便模拟异常工况。

## 8.2 MV 工程量程

内部 `MV%` 转换为工程输出：

[
MV=
MVSCL+
\frac{MV_%}{100}(MVSCH-MVSCL)
]

反向转换：

[
MV_%=
100\cdot\frac{MV-MVSCL}{MVSCH-MVSCL}
]

## 8.3 SV 实际限幅

实际参与控制的设定值必须满足：

[
SVL\le SV\le SVH
]

约束：

```text
SVSCL <= SVL <= SVH <= SVSCH
```

AUTO 模式限制本地 SV，CAS/RCAS 模式限制 CSV 对应的有效 SV。

## 8.4 MV 实际限幅

最终输出必须满足：

[
MVL\le MV\le MVH
]

约束：

```text
MVSCL <= MVL <= MVH <= MVSCH
```

必须同时对手动 MV 和自动 MV 生效。

## 9. 正反作用

按照 ECS-700 手册：

```text
SWPN=OFF/0 -> 正作用，PV 增大时 MV 增大
SWPN=ON/1  -> 反作用，PV 增大时 MV 减小
```

内部误差定义：

```python
if bool(self.SWPN):
    # 反作用
    error_pct = sv_pct - pv_pct
else:
    # 正作用
    error_pct = pv_pct - sv_pct
```

不得只改变展示属性；`SWPN` 必须实际改变 PID 输出方向。

## 10. 核心 PID 算法

## 10.1 使用增量式算法

不得延续旧 PID 的位置式计算：

```python
MV = P + I + D
```

采用 ECS-700 标准 PID 的增量式结构：

[
\Delta MV_n=
\frac{100}{PB}
\left(
\Delta E_n+
\frac{T_s}{TI}E_n+
\Delta U_n
\right)
]

其中：

[
\Delta E_n=E_n-E_{n-1}
]

[
MV_n=MV_{n-1}+\Delta MV_n
]

`PB` 是比例度：

[
K_p=\frac{100}{PB}
]

因此：

* PB 变小，比例作用增强。
* PB 变大，比例作用减弱。
* 必须满足 `PB > 0`。

## 10.2 积分项

当：

```text
TI > 0
```

积分增量：

[
I_{\Delta}=\frac{T_s}{TI}E_n
]

当：

```text
TI = 0
```

积分项完全关闭：

```python
i_delta = 0.0
```

不得除零，不得沿用旧积分状态产生输出。

## 10.3 不完全微分

使用手册中的不完全微分：

[
U_n=
\frac{TD}{KD\cdot T_s+TD}
\left[
U_{n-1}+KD(E_n-E_{n-1})
\right]
]

[
\Delta U_n=U_n-U_{n-1}
]

当：

```text
TD = 0
```

执行：

```python
u_now = 0.0
d_delta = 0.0
```

`KD` 默认值：

```text
KD=10.0
```

要求：

```text
KD >= 0
TD >= 0
```

本次只实现标准 PID，不增加 `PID_OPT`。PV 微分先行和 PV 比例微分先行作为明确的非目标。

## 11. 内部状态设计

新 PID 至少维护以下内部状态：

```python
self._prev_error_pct: float
self._prev_derivative_state: float
self._mv_pct: float
self._previous_mode: int | None
self._initialized: bool
self._local_sv: float
self._last_valid_params: dict[str, float]
```

这些内部属性：

* 不加入 `stored_attributes`。
* 不暴露到 OPC UA。
* 不通过 YAML 直接配置。

### 初始化规则

如果实例第一次执行时已经处于 AUTO/CAS/RCAS：

* `_prev_error_pct` 初始按 0 处理。
* 允许第一个周期产生比例响应。
* `_mv_pct` 从配置的初始 `MV` 反向换算。

如果实例第一次处于 MAN/IMAN/TR/ROUT：

* 以当前 MV 建立内部跟踪状态。
* 后续进入 AUTO/CAS 时执行无扰初始化。

## 12. 各 MODE 的运行行为

## 12.1 MODE=1：OOS

本次简化行为：

```python
self.MV = clamp(self.MV, self.MVL, self.MVH)
```

并且：

* 不执行 P/I/D。
* 不推进误差和微分状态。
* 保持最后有效输出。
* 不实现 `OOSVAL`。

## 12.2 MODE=2、4、8：手动类模式

包括：

```text
2 = IMAN
4 = MAN
8 = ROUT
```

行为：

1. 外部可以通过 OPC UA 写 `MV`。
2. PID 不自动修改 MV。
3. MV 仍受 `MVL/MVH` 限制。
4. 每周期将内部 `_mv_pct` 跟踪到当前 MV。
5. 更新必要的误差历史，为切回自动模式做准备。

不要增加独立 `MANMV`；当前工具直接使用 `MV` 作为手动输出命令。

## 12.3 MODE=3：TR

本次不增加 `TV` 位号，简化为：

* 当前 `MV` 作为跟踪值。
* 不执行 P/I/D。
* 更新 `_mv_pct` 和自动切换所需状态。
* 行为接近“外部输出跟踪”。

## 12.4 MODE=5：AUTO

使用：

```python
effective_sv = self._local_sv
```

执行完整 PID。

## 12.5 MODE=6：CAS

使用：

```python
effective_sv = self.CSV
```

执行完整 PID。

同时：

```python
self.SV = effective_sv
self.CAS = 1
```

## 12.6 MODE=7：RCAS

本次简化为：

```python
effective_sv = self.CSV
```

执行与 CAS 相同的 PID 计算，但状态保持：

```python
self.AUTO = 7
self.CAS = 0
```

这样上游可以通过 `AUTO=7` 将其识别为“外给定”，不会误判为普通 CAS。

## 13. 无扰切换

从手动类模式进入自动类模式时：

```text
2/3/4/8 -> 5/6/7
```

不得重新从 0 计算绝对输出。

处理原则：

1. `_mv_pct` 以切换前实际 MV 为起点。
2. 将 `_prev_error_pct` 对齐到当前误差，避免首周期比例突跳。
3. 将微分内部状态对齐或清零，避免首周期微分冲击。
4. 切换后的第一个周期最多允许产生一个周期的积分增量。
5. MV 不得跳回默认值或初始值。

建议切换处理伪代码：

```python
if entered_automatic_mode:
    self._mv_pct = self._engineering_mv_to_pct(self.MV)
    self._prev_error_pct = current_error_pct
    self._prev_derivative_state = 0.0
```

然后本周期：

```python
p_delta = 0.0
d_delta = 0.0
```

可以保留：

```python
i_delta = cycle_time / TI * current_error_pct
```

验收时，MAN 切 AUTO 的输出变化不得超过该周期积分贡献及数值误差。

---

## 14. 抗积分饱和

不得只在最终执行：

```python
MV = max(MVL, min(MVH, MV))
```

必须阻止积分继续推动已经饱和的输出。

推荐两阶段计算：

```python
non_integral_delta = kp * (p_delta + d_delta)
integral_delta = kp * i_delta

candidate = mv_pct + non_integral_delta + integral_delta
```

如果：

```python
candidate > mvh_pct and integral_delta > 0
```

则本周期取消正向积分：

```python
candidate = mv_pct + non_integral_delta
```

如果：

```python
candidate < mvl_pct and integral_delta < 0
```

则本周期取消负向积分：

```python
candidate = mv_pct + non_integral_delta
```

最后：

```python
new_mv_pct = clamp(candidate, mvl_pct, mvh_pct)
```

必须允许有助于退出饱和的积分方向继续作用。

本次不实现：

* `ATL_OPT`
* `RRL`
* `BKIN/BKOUT`
* 上下游抗饱和传播

---

## 15. 建议的单周期伪代码

下面仅用于约束执行顺序，不要求 Agent 原样复制：

```python
def execute(self, PV=None, SV=None, CSV=None, MODE=None, SWPN=None):
    # 1. 接收本周期输入
    self._apply_inputs(PV=PV, SV=SV, CSV=CSV, MODE=MODE, SWPN=SWPN)

    # 2. 校验运行参数；无效运行时写值恢复为上次有效参数
    self._validate_or_restore_runtime_params()

    mode = int(self.MODE)

    # 3. 派生上游兼容状态
    self.AUTO = mode
    self.CAS = 1 if mode == 6 else 0

    # 4. 处理设定值来源
    if mode == 5:
        effective_sv = clamp(self._local_sv, self.SVL, self.SVH)
        self.SV = effective_sv
    elif mode in (6, 7):
        effective_sv = clamp(self.CSV, self.SVL, self.SVH)
        self.SV = effective_sv
    else:
        effective_sv = clamp(self._local_sv, self.SVL, self.SVH)

    # 5. 非自动模式
    if mode in (1, 2, 3, 4, 8):
        self.MV = clamp(self.MV, self.MVL, self.MVH)

        if mode in (2, 3, 4, 8):
            self._track_manual_state(effective_sv)

        self._previous_mode = mode
        return

    # 6. 自动模式只允许 5/6/7
    if mode not in (5, 6, 7):
        self._previous_mode = mode
        return

    # 7. 工程量归一化
    pv_pct = self._sv_to_pct(self.PV)
    sv_pct = self._sv_to_pct(effective_sv)

    # 8. 正反作用
    error_pct = (
        sv_pct - pv_pct
        if bool(self.SWPN)
        else pv_pct - sv_pct
    )

    # 9. 从手动类模式切入自动类模式时无扰初始化
    if self._previous_mode not in (5, 6, 7):
        self._prepare_bumpless_transfer(error_pct)

    # 10. 计算增量 PID
    p_delta = error_pct - self._prev_error_pct
    i_delta = (
        self.cycle_time / self.TI * error_pct
        if self.TI > 0
        else 0.0
    )
    d_delta = self._calculate_filtered_derivative(error_pct)

    kp = 100.0 / self.PB

    # 11. 抗积分饱和和输出限幅
    self._mv_pct = self._calculate_limited_mv_pct(
        p_delta=p_delta,
        i_delta=i_delta,
        d_delta=d_delta,
        kp=kp,
    )

    # 12. 转回工程量
    self.MV = self._pct_to_engineering_mv(self._mv_pct)
    self.MV = clamp(self.MV, self.MVL, self.MVH)

    # 13. 更新内部状态
    self._prev_error_pct = error_pct
    self._previous_mode = mode
```

## 16. 参数校验

## 16.1 初始化校验

构造新 PID 时，以下错误必须抛出清晰的 `ValueError`：

```text
cycle_time <= 0
PB <= 0

TD < 0
KD < 0
SVSCH <= SVSCL
MVSCH <= MVSCL
SVL < SVSCL
SVH > SVSCH
SVL > SVH

MVH > MVSCH

任意数值为 NaN 或无穷大
```

错误信息必须包含参数名和值。

示例：

```text

```

## 16.2 运行时写值校验

当前 `controller/engine.py` 的 `_apply_external_overrides()` 会直接：

```python
setattr(instance, attr_name, value)
```

因此不能假设所有运行时写入都是合法的。

新 PID 每周期执行前应校验运行参数：

* 合法：更新 `_last_valid_params`。
* 非法：恢复该参数上一次有效值并记录 warning。
* 不因一次非法 UA 写值让整个 Engine 进入 SAFE STATE。
* `PV` 超量程不是参数错误，不得恢复或限制。
* `SV/CSV/MV` 超限时执行操作限幅，不作为配置错误。
* `AUTO`、`CAS` 的外部写值直接忽略，并在本周期重新派生。

## 17. 在线修改参数的连续性

PB、TI、TD、KD 在线变化时：

* 不清空当前 MV。
* 不清空所有历史状态。
* 不重新执行位置式 PID。
* 新参数从下一个增量周期生效。
* TD 从 0 切换到正值时，应初始化微分状态，避免首周期微分脉冲。
* SWPN 改变时属于控制方向突变，可以重新对齐误差和微分状态，但不得清零 MV。

## 18. 配置迁移

现有配置中的：

```yaml
PB: 12
```

表示旧算法中的比例增益 12，不再等价于新 PID 的 `PB=12%`。

如果希望近似保持旧比例强度，可以按量程换算：

[
PB_{\text{new}}=
\frac{100\cdot(MVSCH-MVSCL)}
{K_{\text{legacy}}\cdot(SVSCH-SVSCL)}
]

例如水箱：

```text
旧比例增益 Klegacy = 12
PV/SV 量程 = 0～2 m
MV 量程 = 0～100 %
```

则：

[
PB_{\text{new}}=
\frac{100\cdot100}{12\cdot2}
\approx416.67%
]

因此不得机械保留所有示例中的：

```yaml
PB: 12
```

并宣称行为兼容。

## 18.1 活跃水箱配置建议

对使用 2 m 高水箱、0～100% 阀位的配置，至少增加：

```yaml
params:
  PB: 416.67
  TI: 30.0
  TD: 0.15
  KD: 10.0

  MODE: 5
  SWPN: 1

  SVSCL: 0.0
  SVSCH: 2.0
  SVL: 0.0
  SVH: 2.0

  MVSCL: 0.0
  MVSCH: 100.0
  MVL: 0.0
  MVH: 100.0
```

此值只用于获得接近旧比例强度的初始起点，最终应通过测试重新整定。

优先更新：

```text
config/tank_constant_sv.yaml
config/tank_structured.yaml
config/典型水箱液位控制.yaml
classical_config/典型水箱液位控制.yaml
```

`config/old_version/` 不要求迁移。

`controller/running_config/` 中的运行时副本不要作为算法标准配置来源；如测试会扫描这些文件，则同步迁移，否则可以重新生成。
