## 4.1 Python Engine 原子批次

主要修改：

```text
controller/engine.py
datacenter/engine_api.py
```

实现：

```http
POST /api/instances/{runtimeName}/writes
GET  /api/instances/{runtimeName}/writes/{batchId}
```

可额外实现：

```http
GET /api/instances/{runtimeName}/writes
```

用于查看 batch 列表，但不要增加验收依赖。

### 请求

```json
{
  "writes": [
    {"tag": "pid2.SV", "value": 0.8},
    {"tag": "pid2.PB", "value": 25.0}
  ],
  "confirm_timeout_s": 3.0
}
```

### 成功 POST 响应

```json
{
  "ok": true,
  "batch_id": "stable-unique-id",
  "status": "pending",
  "writes": [
    {"tag": "pid2.SV", "value": 0.8},
    {"tag": "pid2.PB", "value": 25.0}
  ],
  "accepted_cycle_count": 120
}
```

POST 不得直接返回 `applied`。

### 状态查询

```json
{
  "ok": true,
  "batch_id": "...",
  "status": "pending|applied|failed",
  "writes": [...],
  "accepted_cycle_count": 120,
  "confirmed_cycle_count": 121,
  "error": null
}
```

### 内部实现要求

在 `UnifiedEngine` 中增加一个内部原子批次队列，建议结构：

```python
@dataclass(frozen=True)
class AtomicWrite:
    tag: str
    value: float

def queue_atomic_writes(self, writes: list[AtomicWrite]) -> None:
    ...
```

该方法是内部实现，不需要注册为公共验收接口。

要求：

1. 整批在入队前完成验证；
2. 任一字段无效时整批拒绝；
3. 不得先把合法字段放进现有单字段队列，再发现非法字段；
4. 一个 batch 的全部字段在同一次 `_apply_pending_changes()` 中应用；
5. 应用过程处于同一把 Engine 锁内；
6. 同一周期生成的 snapshot 必须包含全部目标值；
7. 不允许半批应用。

不要通过循环调用 HTTP `/params` 或 `/override` 模拟原子批次。

### 可写字段

按正式规范实现。至少支持：

```text
pid2.SV
pid2.PB
pid2.TI
pid2.TD
pid2.KD
```

`pid2.MV` 必须遵守当前 PID 模式和正式规范。

必须拒绝：

```text
unknown tag
pid2.PV
tank_1.level
tank_2.level
valve_1.current_opening
实时流量
AUTO/CAS 等派生状态名称
NaN
Infinity
不存在的 runtimeName
违反模式约束的写值
```

错误必须是 HTTP 4xx，并说明字段或原因。

### Batch 状态管理

在 `EngineBinding` 内维护线程安全 batch registry，建议字段：

```python
_batch_lock
_batches: dict[str, WriteBatchRecord]
```

记录：

```text
batch_id
writes
status
accepted_cycle_count
confirmed_cycle_count
accepted_monotonic
confirm_timeout_s
error
```

在 `push_snapshot()` 中：

1. 取得同一份完整 snapshot；
2. 对所有 pending batch 检查全部字段；
3. 只有同一 snapshot 同时匹配全部目标时才改为 `applied`；
4. `confirmed_cycle_count` 使用该 snapshot 的 cycle_count；
5. 单字段匹配不得提前 applied。

超时使用：

```python
time.monotonic()
```

在状态查询和 snapshot 推送时均可更新超时状态。

不同 batch 的 ID、内容和状态必须完全隔离。

### 兼容性

保留现有：

```http
POST /api/instances/{runtimeName}/params
POST /api/instances/{runtimeName}/override
```

不要删除旧接口。

可以修正 `/params` 当前把 runtimeName 当作 programName 的问题，但不要让 `/params` 成为新的 batch API。

## 9.2 Realtime / Batch 互斥

在 `SystemBinding` 中增加受 `mu` 保护的：

```go
activeBatches int
```

实现内部 batch lease：

```text
开始 Batch：
- 持锁；
- 若实时 proc != nil 且仍运行，返回错误；
- activeBatches++；
- 解锁。

结束 Batch：
- defer 持锁 activeBatches--。
```

`Start()` 在启动实时进程前：

```text
若 activeBatches > 0，返回“批量任务正在运行”错误。
```

要求：

* 多个 Batch 可以并发；
* 实时与任意 Batch 互斥；
* Batch 失败后 activeBatches 必须恢复；
* Start 失败不得启动子进程；
* RunBatch 被拒绝不得生成 CSV。

不要新增 `CanRunBatch` 公共方法。
