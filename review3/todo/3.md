# 乙醇—水连续精馏塔动态模型实施设计

> 适用仓库：`review3`
> 目标读者：负责落地实现、仿真标定和测试的 Agent
> 文档性质：实施约束，不是概念建议
> 第一版范围：无化学反应的乙醇—水连续精馏、基础 DCS 控制、质量控制、AI 数据生成和 OPC UA 位号输出

---

## 1. 最终目标

在现有统一 DSL、`BaseProgram`、`UnifiedEngine` 和工业 PID 基础上，新增一个具有真实工艺身份的动态精馏对象：

> **常压乙醇—水连续溶剂回收中试塔**

它不是通用流程模拟器，也不是简单传递函数。第一版必须具备：

1. 明确的乙醇—水物系和产品规格；
2. 逐塔板动态物料衡算；
3. 简化但守恒的能量衡算；
4. 乙醇—水非理想汽液平衡；
5. 全凝器、回流罐、釜式再沸器和产品采出；
6. 塔压、液位、流量、温度和浓度动态；
7. 外部 PID 可以读取 PV、输出阀位并形成串级控制；
8. AI 可以读取标准回路位号并写入 `SV/MV/MODE`；
9. 在线模式以真实时间运行，离线模式快速生成 3 天至 30 天数据；
10. 0.5 秒参考模型与较大周期之间具有明确的等效性认证流程。

第一版不包含：

* 化学反应或反应精馏；
* 三元、多元杂质体系；
* 分子筛脱水、萃取精馏或共沸剂；
* 严格速率基传质模型；
* 液泛、漏液、雾沫夹带的完整水力学关联式；
* 通用流程图编辑器；
* Aspen Plus / Aspen Dynamics 文件兼容；
* 冷态开车的完整操作规程。

以上内容必须保持为后续扩展，不能为了“显得完整”混入第一版。

## 2. 工艺边界与参考工况

### 2.1 工艺任务

装置处理已经过滤、无固体颗粒的乙醇—水进料。该对象可以代表：

* 含乙醇废液或清洗溶剂回收；
* 发酵液经过前处理或粗浓缩后的精馏段；
* 小型连续乙醇回收中试装置。

普通单塔精馏不得宣称生产无水乙醇。产品规格必须低于乙醇—水普通精馏的共沸边界。

### 2.2 参考设备

| 项目      |            第一版参考值 | 约束              |
| ------- | ----------------: | --------------- |
| 塔板数量    |              12 块 | 塔板从上到下编号 1～12   |
| 进料塔板    |             第 7 块 | `feed_stage=7`  |
| 冷凝器     |               全凝器 | 单独建模，不计入 12 块塔板 |
| 再沸器     |           釜式平衡再沸器 | 单独建模            |
| 操作压力    | 塔顶 101.325 kPa(a) | 允许正常波动约 ±5 kPa  |
| 塔径      |            0.25 m | 中试尺度参考值         |
| 塔板间距    |            0.45 m | 用于气相体积与压降估算     |
| 单板正常持液  |            3～5 kg | 最终值由动态标定确定      |
| 回流罐有效容积 |           0.08 m³ | 50% 为正常液位       |
| 塔釜有效容积  |           0.10 m³ | 50% 为正常液位       |

塔板数量第一版固定为 12。不要为了支持任意塔板数量而引入动态位号、动态 UI 和复杂序列化。内部实现可使用数组，但对外位号固定展开到 12 块塔板。

### 2.3 参考生产工况

| 项目         |       参考值 |
| ---------- | --------: |
| 进料总流量      |  100 kg/h |
| 进料乙醇质量分数   |      0.25 |
| 进料温度       |      60 ℃ |
| 塔顶乙醇质量分数目标 |      0.85 |
| 塔底乙醇质量分数目标 |     0.015 |
| 乙醇回收率目标    |      ≥95% |
| 参考塔顶产品量    | 约 28 kg/h |
| 参考塔底产品量    | 约 72 kg/h |
| 初始回流比      |     约 3.0 |
| 参考回流量      | 约 84 kg/h |
| 回流罐正常液位    |       50% |
| 塔釜正常液位     |       50% |

根据总物料和乙醇组分衡算：

[
F=D+B
]

[
Fz_F=Dx_D+Bx_B
]

当 `F=100 kg/h`、`zF=0.25`、`xD=0.85`、`xB=0.015` 时，理论产品量约为：

[
D=\frac{F(z_F-x_B)}{x_D-x_B}\approx 28.14;kg/h
]

这些值是稳态求解和初始化的目标，不允许直接硬编码成每周期固定输出。

## 3. 总体软件结构

### 3.1 使用一个复合工艺模型

新增一个复合模型：

```text
ETHANOL_WATER_DISTILLATION
```

它内部包含：

* 12 块塔板；
* 全凝器；
* 回流罐；
* 再沸器/塔釜；
* 六个阀门/执行机构的动态；
* 乙醇—水热力学；
* 浓度分析仪的滞后和噪声；
* 工艺统计量和守恒诊断。

不要把 12 块塔板拆成 12 个 DSL 节点。拆分会增加对象、AST 表达式和拓扑调度开销，也会形成大量不必要的循环依赖。

PID 保持为模型外部的现有 `PID` 节点。精馏模型不能在内部重新实现 PID。

## 4. 热力学模型

### 4.1 内外单位

模型内部统一使用：

| 物理量  | 内部单位      |
| ---- | --------- |
| 物质量  | kmol      |
| 摩尔流量 | kmol/s    |
| 温度   | K         |
| 压力   | kPa(a)    |
| 焓    | kJ/kmol   |
| 热负荷  | kW，即 kJ/s |
| 组成   | 摩尔分数      |

对外位号使用：

| 物理量    | 对外单位     |
| ------ | -------- |
| 流量     | kg/h     |
| 温度     | ℃        |
| 压力     | kPa(a)   |
| 产品浓度   | 质量分数 0～1 |
| 塔板内部组成 | 摩尔分数 0～1 |
| 液位/阀位  | %        |

所有质量分数和摩尔分数转换必须集中在热力学模块中，禁止在模型各处重复实现。

### 4.2 纯组分常数

至少包含：

```text
MW_WATER = 18.01528 kg/kmol
MW_ETHANOL = 46.0684 kg/kmol
```

蒸气压使用 Antoine 方程：

[
\log_{10} P_i^{sat}=A_i-\frac{B_i}{T+C_i}
]

必须在代码中明确每组系数对应的压力单位、温度单位和有效温度范围。不得混用 `mmHg/℃` 与 `bar/K` 系数。

推荐直接采用 NIST Chemistry WebBook 给出的 `bar/K` 形式，并在模块注释中记录来源：

* 乙醇：https://webbook.nist.gov/cgi/cbook.cgi?ID=C64175&Mask=4
* 水：https://webbook.nist.gov/cgi/cbook.cgi?ID=C7732185&Mask=4

如温度跨越某一组 Antoine 系数的有效区间，应按温度选择对应系数段；不得无条件外推。

### 4.3 非理想汽液平衡

乙醇—水不能使用固定相对挥发度作为正式模型。正式后端采用：

```text
气相：理想气体
液相：NRTL 活度系数
逸度修正：第一版忽略
Poynting 修正：第一版忽略
```

汽液平衡关系：

[
y_iP=x_i\gamma_iP_i^{sat}(T)
]

定义：

[
K_i=\frac{\gamma_iP_i^{sat}(T)}{P}
]

泡点温度通过求解下式获得：

[
\sum_i x_iK_i(T,P)=1
]

然后：

[
y_i=x_iK_i
]

NRTL 参数必须来自公开来源，并在源码中记录：参数方向、温度表达式、能量单位和组分编号。可以采用美国能源部 Savannah River National Laboratory 报告中列出的水—乙醇温度相关参数作为候选，但必须通过实验 VLE 点验证后才能定稿：

https://sti.srs.gov/fulltext/SRNL-STI-2021-00391.pdf

实验验证可参考 101.3 kPa 下乙醇—水 VLE 数据：

https://pubs.acs.org/doi/10.1021/je2008704

Agent 不得凭记忆填写 NRTL 参数。参数方向填反通常不会报错，但会产生错误的共沸点和产品纯度。

### 4.4 求解器约束

仓库当前只有 NumPy 依赖，不应为了一个二元泡点求解强制引入 SciPy。

实现一个有界求根器：

```python
def bubble_point_temperature(
    x_ethanol: float,
    pressure_kpa: float,
    previous_temperature_k: float | None = None,
) -> tuple[float, float]:
    """返回 (T_bubble_K, y_ethanol)。"""
```

要求：

* 默认温度搜索区间 340～380 K，可根据压力扩展；
* 优先使用上一周期温度作为初值；
* 使用有界二分、割线或混合方法；
* 最大迭代次数固定；
* 达不到容差时抛出有上下文的异常，不得返回 NaN；
* 对 12 块塔板使用 NumPy 数组批量计算活度系数和蒸气压；
* 可以建立 VLE 缓存或查表，但查表误差必须通过测试限定。

### 4.5 热力学单元测试

至少验证：

1. 101.325 kPa 下纯水泡点接近 373.15 K；
2. 101.325 kPa 下纯乙醇泡点接近 351.5 K；
3. `x_ethanol=0` 时 `y_ethanol=0`；
4. `x_ethanol=1` 时 `y_ethanol=1`；
5. 所有 VLE 输出有限且位于 0～1；
6. 选择至少 5 个公开实验点比较 `T/x/y`；
7. 平均温度误差和气相组成误差必须在文档声明的限值内；
8. 能正确表现乙醇—水共沸趋势；
9. 任何普通精馏计算不得越过物系允许的纯度边界；
10. 质量分数与摩尔分数互相转换后可逆。

建议初始验收限值：

```text
纯组分泡点误差 ≤ 0.5 K
实验点平均温度误差 ≤ 1.0 K
实验点气相乙醇摩尔分数平均绝对误差 ≤ 0.02
```

如果候选 NRTL 参数不能满足，不得通过调整塔模型掩盖，必须先修正热力学后端。

## 5. 动态精馏模型

### 5.1 状态变量

每块塔板 `i=1..12` 至少维护：

```text
M[i]       液相总持液量，kmol
nE[i]      液相乙醇物质量，kmol
U[i]       液相总内能或等价焓状态，kJ
T[i]       塔板温度，K（由能量状态或泡点关系求出）
xE[i]      液相乙醇摩尔分数
yE[i]      平衡气相乙醇摩尔分数
L[i]       向下液相流量，kmol/s
V[i]       向上气相流量，kmol/s
P[i]       塔板压力，kPa(a)
```

回流罐至少维护：

```text
M_drum
nE_drum
U_drum
level_drum
```

再沸器/塔釜至少维护：

```text
M_sump
nE_sump
U_sump
level_sump
T_sump
```

塔内气相存量至少维护一个集中参数状态，用于形成塔压动态：

```text
N_vapor_total
P_top
```

不允许把压力永远固定为常数，同时又对外提供“压力 PID”。

### 5.2 逐板总物料衡算

[
\frac{dM_i}{dt}=L_{i-1}+V_{i+1}+F_i-L_i-V_i
]

实际数组方向可以不同，但必须在代码中统一说明“1号板在塔顶，12号板在塔底”。

### 5.3 乙醇组分衡算

[
\frac{dn_{E,i}}{dt}
=L_{i-1}x_{E,i-1}+V_{i+1}y_{E,i+1}+F_iz_{E,F}
-L_ix_{E,i}-V_iy_{E,i}
]

然后：

[
x_{E,i}=\frac{n_{E,i}}{M_i}
]

水组分由总量减乙醇得到，仍需在测试中检查乙醇和水分别守恒。

### 5.4 能量衡算

每块塔板采用：

[
\frac{dU_i}{dt}
=L_{i-1}h^L_{i-1}+V_{i+1}h^V_{i+1}+F_ih_F
-L_ih^L_i-V_ih^V_i-Q_{loss,i}
]

第一版允许：

* 液相和气相热容采用温度相关或分段常数；
* 混合焓第一版可忽略，但必须在文档中声明；
* 塔体散热采用 `UA(T-T_ambient)`；
* 汽化潜热采用纯组分潜热按组成加权；
* 能量模型必须保证量纲一致。

不允许仅用浓度查表生成温度，然后声称再沸负荷和冷凝负荷满足能量守恒。如果第一阶段暂时采用泡点代数温度，必须把它标记为中间里程碑，不能作为最终验收版本。

### 5.5 液相水力学

第一版使用降阶堰流关系：

[
L_i=L_{nom,i}\left(\frac{M_i}{M_{nom,i}}\right)^{1.5}
]

要求：

* `M_i` 趋近于 0 时流量趋近于 0；
* 对流量设置物理上限，但不能长期靠上限维持稳态；
* 正常工况下所有塔板持液量保持在标称值合理范围；
* 液位扰动可以逐板传播，而不是瞬时传遍全塔。

### 5.6 气相、再沸和冷凝

再沸蒸气量由再沸器净热负荷决定：

[
V_{boil}\approx\frac{Q_R-Q_{sensible}-Q_{loss}}{\Delta H_{vap}}
]

全凝器冷凝量由冷凝负荷决定，并受进入冷凝器的蒸气量限制。

塔压由气相存量和冷凝/再沸不平衡形成：

[
\frac{dN_v}{dt}=V_{boil}-V_{condense}-V_{vent}
]

[
P=\frac{N_vRT}{V_{gas}}
]

沿塔压降可采用：

[
\Delta P_i=\Delta P_{dry}+K_vV_i^2
]

正常工况要求压力从塔顶到塔底单调增加。

### 5.7 数值积分

建议第一版使用显式 Heun/RK2，而不是最简单的前向 Euler：

```text
k1 = f(state)
predicted = state + dt*k1
k2 = f(predicted)
state_next = state + dt*(k1+k2)/2
```

模型对外仍执行一次 `execute()`。内部可根据数值稳定性使用子步：

```yaml
params:
  max_internal_step: 0.5
```

规则：

```text
substeps = ceil(cycle_time / max_internal_step)
internal_dt = cycle_time / substeps
```

但要注意：如果 `cycle_time=5s` 且内部仍固定执行 10 个 0.5s 子步，数值结果更接近参考模型，却不会获得 10 倍性能提升。因此较大周期是否允许减少子步，必须通过周期等效性测试决定。

### 5.8 物理边界和故障处理

每周期检查：

* 所有状态为有限数值；
* `M>0`；
* `0≤xE≤1`、`0≤yE≤1`；
* 温度在合理物理范围；
* 压力为正；
* 回流罐和塔釜体积不为负；
* 阀位在 0～100%；
* 每周期质量和能量残差有限。

只允许对 `1e-12` 量级浮点越界进行数值截断。明显越界必须抛出异常，使现有引擎连续失败保护进入 SAFE STATE。禁止持续 `clip()` 掩盖模型发散。

### 6.2 阀门动态

```text

```

| 阀门    | 特性        | 满行程时间 |
| ----- | --------- | ----: |

### 6.3 测量动态

```text

```

* 一阶测量滞后；
* 可配置采样间隔；

## 8. 控制结构

### 8.1 自由度分配

同一个阀门不能被两个控制器同时直接控制。第一版采用：

| 操作变量  | 控制目标         |
| ----- | ------------ |
| 进料阀   | 进料流量/装置负荷    |
| 冷却水阀  | 塔顶压力         |
| 塔顶采出阀 | 回流罐液位        |
| 塔底采出阀 | 塔釜液位         |
| 回流阀   | 塔顶质量或上部灵敏板温度 |
| 蒸汽阀   | 塔底质量或下部灵敏板温度 |

产量控制不能再直接占用塔顶采出阀。上层产量控制应调整进料流量设定值，并让库存与质量控制器保持产品合格。

### 8.2 基础控制 DSL

`乙醇水连续精馏_基础控制.yaml` 使用：

* 进料流量 PID；
* 塔顶压力 PID；
* 回流罐液位 PID；
* 塔釜液位 PID；
* 上部灵敏板温度主 PID + 回流流量副 PID；
* 下部灵敏板温度主 PID + 蒸汽流量副 PID。

副回路使用 `MODE=6`，主回路输出连接副回路 `CSV`。

### 8.3 质量控制 DSL

`乙醇水连续精馏_质量控制.yaml` 使用：

* 基础流量、压力和液位回路保持不变；
* 塔顶浓度主 PID 替代上部温度主 PID；
* 塔底浓度主 PID 替代下部温度主 PID；
* 浓度主 PID 的 MV 分别作为回流流量、蒸汽流量副回路的 CSV；
* AI 可以写入塔顶浓度、塔底浓度和进料流量回路的 SV。

温度主控和浓度主控不能同时处于 AUTO。采用两个独立 DSL，避免隐藏的控制权冲突。

### 8.4 正反作用

按当前 PID 语义：

```text
SWPN=1：error = SV - PV
SWPN=0：error = PV - SV
```

建议初始方向：

| 回路      | 高 PV 时期望动作 | SWPN |
| ------- | ---------- | ---: |
| 进料流量    | 减小进料阀      |    1 |
| 塔顶压力    | 增大冷却水阀     |    0 |
| 回流罐液位   | 增大塔顶采出阀    |    0 |
| 塔釜液位    | 增大塔底采出阀    |    0 |
| 塔顶乙醇浓度  | 浓度低时增加回流   |    1 |
| 塔底乙醇浓度  | 浓度高时增加蒸汽   |    0 |
| 回流流量副回路 | 流量高时减小阀位   |    1 |
| 蒸汽流量副回路 | 流量高时减小阀位   |    1 |

方向必须通过单步扰动测试验证，不能只靠表格。

## 9. DSL 拓扑约束

精馏塔模型读取多个 PID 的 MV，PID 又读取精馏塔 PV，会形成依赖环。

只在复合精馏塔模型上设置：

```yaml
execute_first: true
```

不要给所有 PID 同时设置 `execute_first`。

期望执行语义：

1. `column_1` 使用上一周期 PID 输出推进到本周期状态；
2. 基础主回路读取本周期工艺测量值；
3. 质量/温度主回路计算新的流量设定值；
4. 流量副回路读取主回路 MV 作为 CSV；
5. 下一周期 `column_1` 使用新的阀门命令。

控制输出存在一个基础周期的离散延迟，这是确定且可接受的。拓扑排序结果必须通过测试固定，不能依赖 Python `set` 的遍历顺序。

如果现有排序器在多个无依赖 PID 之间顺序不确定，Agent 应让同层节点按 DSL 原始顺序稳定排序。该修复只针对确定性，不改变依赖语义。

## 10. 初始化

### 10.1 第一版默认稳态初始化

默认启动方式：

```yaml
params:
  initialization_mode: STEADY
```

Agent 必须生成一组与参考工况一致的稳态状态，包括：

* 12 块塔板液相持液；
* 乙醇浓度剖面；
* 温度剖面；
* 压力剖面；
* 回流罐和塔釜存量；
* 正常气液流量；
* 六个执行机构正常开度。

稳态状态可以作为经过验证的数据文件提交，例如：

```text
review3/components/programs/data/ethanol_water_reference_state.json
```

但必须包含版本号和适用参数哈希。用户修改关键设备参数后，旧稳态不得静默复用。

### 10.2 稳态生成方式

允许使用以下流程生成参考状态：

1. 根据总物料、组分衡算得到 `D/B` 初值；
2. 根据回流比得到回流量初值；
3. 用稳态逐板方程得到浓度和温度初始剖面；
4. 用动态模型进行伪时间松弛；
5. 当所有状态导数、质量和能量残差达到限值后保存；
6. 重新加载状态运行 1 小时，确认无明显漂移。

禁止通过每周期强行拉回目标值制造“稳态”。

### 10.3 冷态开车

`initialization_mode=COLD` 第一版可以明确报 `NotImplementedError`，但不能伪装成已支持。后续再单独设计全回流、升温和合格切采出过程。

## 11. 时间、采样和周期认证

### 11.1 时间语义

```yaml
clock:
  mode: REALTIME
  cycle_time: 0.5
  sample_interval: 5.0
```

含义：

* `cycle_time`：过程模型和基础 PID 的执行周期；
* `sample_interval`：历史记录、CSV/Excel 和 AI 输入的采样间隔；
* 采样不改变任何位号名或后缀；
* AI 默认每 5 秒推理一次，其输出在中间周期零阶保持。

### 11.2 修复非整数倍采样问题

当前 `Clock` 使用：

```python
int(sample_interval / cycle_time)
```

这会导致 `cycle_time=2s`、`sample_interval=5s` 被错误执行成每 4 秒采样一次。

第一版选择严格规则：

```text
sample_interval / cycle_time 必须接近整数，否则 ClockConfig 抛 ValueError。
```

因此 5 秒采样的候选执行周期为：

```text
0.5 s
1.0 s
2.5 s
5.0 s
```

不要在第一版加入插值采样。插值会生成并非真实控制时刻的状态，还会增加位号质量语义问题。

### 11.3 参考周期和认证周期

定义：

```text
0.5 s = 参考真值周期
1.0/2.5/5.0 s = 候选加速周期
```

使用完全相同的：

* 物理参数；
* 初始状态；
* PID 参数；
* 扰动时间（按物理秒）；
* 扰动幅度；
* 随机种子。

比较：

| 指标             |         建议限值 |
| -------------- | -----------: |
| 稳态流量/压力/液位相对偏差 |        ≤0.5% |
| 塔顶、塔底浓度绝对偏差    |       ≤0.005 |
| 温度绝对偏差         |       ≤0.5 ℃ |
| 主要响应时间偏差       |         ≤10% |
| 超调量偏差          |    ≤20% 相对变化 |
| 乙醇回收率绝对偏差      |    ≤0.5 个百分点 |
| 守恒残差           |   不得因大周期显著恶化 |
| 稳定性            | 不得新增持续振荡或极限环 |

只有通过认证的周期才能成为该 DSL 的默认周期。

Agent 不得预设 5 秒一定通过。建议按 `0.5 → 1.0 → 2.5 → 5.0` 顺序测试，选择满足精度要求的最大周期。

## 12. 离线生成性能

### 12.1 当前基准

当前环境实测约为：

| 模型            | 保存策略        |           吞吐量 |
| ------------- | ----------- | ------------: |
| 单 PID         | 每 10 周期保存一次 | 约 65,000 周期/s |
| 双 PID         | 每 10 周期保存一次 | 约 36,000 周期/s |
| 串级双 PID+阀门+水箱 | 每 10 周期保存一次 | 约 21,500 周期/s |

精馏塔加入逐板 VLE 和能量衡算后会显著变慢，必须单独基准。

### 12.2 引擎快速采样路径

当前 `_step_once()` 每周期无条件执行 `_build_complete_snapshot()`。即使最终只保留每 10 个周期中的一个，也已支付完整字典构建成本。

必须新增向后兼容的采样生成路径，建议接口：

```python
def iter_generator_sampled(
    self,
    n: int,
    selected_variables: list[str] | None = None,
):
    """执行 n 个基础周期，仅在 need_sample=True 时 yield 快照。"""
```

内部要求：

1. 每个基础周期照常执行节点和时钟；
2. 非采样周期不调用 `vars.snapshot()`；
3. 非采样周期不遍历所有 `stored_attributes`；
4. 采样周期如果给了 `selected_variables`，只构建选定位号；
5. 保持现有 `run_generator(n)` 行为不变，避免破坏测试和调用方；
6. CSV 大数据生成使用迭代器分批或逐行写入，不先形成完整列表。

可以将周期执行拆为：

```python
def _execute_cycle(self) -> CycleMetadata: ...
def _build_complete_snapshot(self) -> dict: ...
def _build_selected_snapshot(self, keys: list[str]) -> dict: ...
```

不要让性能接口绕过 `VariableStore` 更新、外部覆写、SAFE STATE 或时钟。

另外，当前 `AlgorithmNode.step()` 会在每个基础周期遍历全部 `stored_attributes` 并写入 `VariableStore`。精馏模型展开全部塔板后可能有 70 个以上属性，因此仅跳过 `_build_complete_snapshot()` 未必足够。

先通过性能分析确认占比；如果属性写入成为主要瓶颈，再新增向后兼容的两级属性声明，例如：

```python
runtime_attributes = [
    # 会被下游 PID/表达式每周期读取的压力、液位、流量、分析值
]

sampled_attributes = [
    # 塔板完整剖面、统计量和诊断量
]

stored_attributes = runtime_attributes + sampled_attributes
```

生成器非采样周期只把 `runtime_attributes` 写入 `VariableStore`；采样周期写入完整集合。现有模型没有声明这两个列表时，仍按原有 `stored_attributes` 行为执行。不得凭猜测提前改造所有模型。

### 12.3 精馏模型性能实现约束

* 塔板状态使用 NumPy 数组；
* 不为每块塔板创建 Python 对象；
* VLE 对 12 块塔板批量计算；
* 上一周期温度作为泡点初值；
* 不在循环内重复读取 YAML 或构造常数表；
* 不在每周期拼接字符串位号；
* 对外标量位号仅在采样/发布时更新字典；
* 日志不允许每周期输出；
* 大型历史使用环形缓冲区；
* 30 天 CSV 流式写出；
* Excel 只用于较小数据或分 Sheet，不能承载 0.5 秒的整月全量数据。

### 12.4 性能门槛

在确定的参考机器上，使用“模型+全部基础 PID+质量 PID”的完整 DSL：

```text
预热：1,000 周期
基准：100,000 周期
长测：1,000,000 周期
导出：sample_interval=5s，仅选定 AI/DCS 位号
```

目标：

```text
硬下限：≥5,000 基础周期/s
期望值：≥10,000 基础周期/s
100 万周期内存不得随周期数线性增长
```

按 0.5 秒参考周期，30 天为 5,184,000 周期：

```text
5,000 周期/s 约 17.3 分钟
10,000 周期/s 约 8.6 分钟
```

如低于硬下限，停止扩功能，先用 `cProfile` 定位 VLE、求根、快照或表达式瓶颈。

不同工况可以按“完整仿真任务”并行；同一条时间轨迹不能按周期并行。

## 13. AI 数据和控制权限

### 13.1 默认 AI 输入

至少包含：

```text
所有控制回路的 PV/SV/MV/MODE/AUTO/CAS
进料流量、组成、温度
塔顶/塔底压力
关键塔板温度
回流罐/塔釜液位
回流、蒸汽、塔顶和塔底流量
塔顶/塔底分析仪浓度
产品流量、回收率、单位蒸汽消耗
```

真实浓度 `*_true` 可以作为训练标签，但部署输入模板可选择排除。

### 13.2 默认 AI 写值

推荐允许 AI 写：

```text
feed_flow_pid.SV
top_quality_pid.SV
bottom_quality_pid.SV
相关 PID 的 MODE
```

只有在明确的直接控制实验中才允许 AI 写流量副回路 CSV 或阀门 MV。AI 直接写阀位前必须将相关 PID 切到合适的手动/远程模式，避免两个控制源争抢。

所有写值仍受现有 PID 的量程、操作限和模式逻辑约束。

### 13.3 生产和经济指标

定义乙醇回收率：

[
Recovery=\frac{D x_D}{F z_F}\times100%
]

定义合格产品流量：

[
Q_{qualified}=\begin{cases}
D,&x_D\ge x_{spec}\
0,&x_D<x_{spec}
\end{cases}
]

定义单位蒸汽消耗：

[
SSC=\frac{\dot m_{steam}}{Q_{qualified}}
]

当产品不合格或产品流量接近 0 时，`SSC` 不得产生 Inf/NaN，应输出预定义无效状态或有限上限，并提供质量标志。

## 14. 工况数据集

### 14.1 基准数据

至少生成一份 24 小时基准数据：

1. 前 2 小时正常稳态；
2. 进料流量增加 10%，保持 3 小时；
3. 恢复并稳定；
4. 进料乙醇质量分数从 0.25 降至 0.20；
5. 冷却水温度上升 5 ℃；
6. 蒸汽供应压力下降 10%；
7. AI 或上层控制修改产量目标；
8. 最后保留足够时间观察新的稳态。

扰动时间必须使用物理秒或时间戳定义，不能使用“第几周期”，否则周期认证时工况时刻会变化。

### 14.2 长周期数据

30 天数据应由多个可解释工况段组成，不要每隔固定时间机械重复完全相同的波形。

必须保存随机种子和场景配置，使数据可以复现。

### 14.3 暂不实现的故障

第一版先保留接口，不要求同时实现：

* 阀门卡涩和死区；
* 换热器结垢；
* 塔板效率下降；
* 液泛与漏液；
* 分析仪校准失效；
* 进料中第三组分杂质。

这些应在基础模型守恒和性能合格后逐个加入。

## 18. 最终决策摘要

第一版正式决策如下：

```text
对象：常压乙醇—水连续溶剂回收中试塔
结构：12块塔板 + 全凝器 + 回流罐 + 釜式再沸器
进料：100 kg/h、乙醇质量分数0.25、60℃
产品目标：塔顶0.85、塔底约0.015、乙醇回收率≥95%
热力学：Antoine + NRTL，禁止固定相对挥发度作为正式后端
动态：逐板物料衡算 + 能量衡算 + 气相存量压力动态
控制：现有PID外置，基础控制和质量控制分成两份DSL
AI位号：PID后缀保持PV/SV/MV/MODE等原名
采样：默认5秒，不修改位号
参考周期：0.5秒
候选加速周期：1、2.5、5秒，必须通过等效性认证
性能：完整模型硬下限5000周期/s，期望10000周期/s
长数据：支持3天至30天流式生成，不在内存保存全部基础周期快照
```

这份设计的优先级依次为：

```text
物理边界正确
→ 热力学可信
→ 质量与能量守恒
→ 控制结构正确
→ AI位号兼容
→ 长周期数值稳定
→ 离线生成性能
→ 扩展故障和反应机理
```

不得颠倒顺序。
