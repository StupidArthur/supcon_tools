# OPC UA Mock Server 设计文档

## 1. 架构概览

- **入口**：`main.py`，解析唯一命令行参数（组态路径），初始化日志后调用 `run_server(config_path)`。
- **服务端**：`server_main.run_server()` 使用 asyncua 创建 Server，按组态在 **`Objects` 下创建对象 `mocker`**，所有组态变量均作为 `mocker` 的子节点创建；并启动周期更新任务。
- **配置与类型**：`config_loader` 加载 YAML；`type_mapping` 维护组态类型名到 OPC UA VariantType 及默认值的映射。
- **变更逻辑**：`change_engines.next_value()` 根据类型与当前值计算下一周期值（bool 翻转、数值锯齿波、字符串 a~z、DateTime +1 秒）。
- **日志**：`log_util` 将日志写到执行程序所在目录，支持脚本与 PyInstaller 打包两种运行方式。

## 2. 模块职责

| 模块 | 职责 |
|------|------|
| main.py | 入口；唯一命令行参数；调用 log_util.setup_logging、server_main.run_server |
| server_main.py | 加载组态、创建 OPC UA Server；在 `Objects` 下创建 `mocker` 对象，其下按组态创建变量；启动 change 周期任务 |
| config_loader.py | 从 YAML 加载组态并做必填项与格式校验 |
| type_mapping.py | 类型名→VariantType/默认值；数值类型判断；coerce_to_type（写回/默认值转换） |
| change_engines.py | next_value(variant_type, current)→下一周期值 |
| log_util.py | get_executable_directory、setup_logging（控制台+文件，文件在执行目录） |

## 3. 数据流

- 组态 YAML → `load_config()` → 字典。
- 节点项 → `get_variant_type_and_default(type_name)` → 初始值 + VariantType。
- change=true 节点 → 每 cycle 调用 `next_value(vt, await n.get_value())` → `coerce_to_type` 后 `write_value`。
- 可写节点：客户端通过 OPC UA 写值，由 asyncua 持久化在节点上，读时直接返回。

## 4. 关键常量

- 变量父对象 BrowseName：`mocker`（`server_main.CONTAINER_OBJECT_NAME`），位于标准节点 `Objects` 之下。
- 锯齿波：0~99，步长 1。
- 字符串循环：单字符 a~z。
- 命名空间索引：与组态中的 `namespace_index` 一致（不调用 register_namespace，直接使用配置值）。
- 日志文件名前缀：`ua_mocker`，按日期的文件名格式在 log_util 中定义。

## 5. 依赖

- asyncua：OPC UA 服务端。
- PyYAML：组态解析。
