# OPC UA Mock Server 需求文档

## 1. 项目概述

- **名称**：OPC UA Server & Data Mock Server（ua_mocker）
- **目标**：提供基于组态文件的 OPC UA 模拟服务器，用于测试与联调；无界面，命令行启动，支持多平台。

## 2. 功能需求

### 2.1 运行方式

- 命令行启动，**唯一一个命令行参数**：组态文件路径。
- 不需要图形界面。
- 支持 Windows、CentOS、Ubuntu；Python 3.13，可用 PyInstaller 打包。

### 2.2 组态文件

- **格式**：YAML。
- **必填项**：
  - `server`：监听地址（如 `"0.0.0.0"`）。
  - `port`：端口（如 `18950`）。
  - `cycle`：数据变化周期（毫秒，如 `1000`）。
  - `namespace_index`：OPC UA 命名空间索引（如 `1`）。
  - `nodes`：节点列表。
- **节点项**：
  - `name`：名称前缀，与 `count` 拼接得到 BrowseName/DisplayName/NodeId 字符串部分。
  - `type`：OPC UA 数据类型（Boolean, SByte, Byte, Int16, UInt16, Int32, UInt32, Int64, UInt64, Float, Double, String, DateTime）。
  - `count`：该 name 下节点个数（生成 name_1 … name_count）。
  - `default`：初始值，**仅当 change 为 false 时有效**；change 为 true 时不需要。
  - `change`：是否按周期自动变化。
  - `writable`：是否允许客户端写；写成功后持久化，下次读返回写入值。

### 2.3 节点标识

- NodeId 形式：`ns=<namespace_index>;s=<name><i>`（如 `ns=1;s=node_float_1`）。
- BrowseName、DisplayName 与配置中 name + 下标的字符串一致。

### 2.4 数据变化规则（change == true）

- **周期**：统一使用组态中的 `cycle`（毫秒）。
- **Boolean**：每个周期在 true/false 之间翻转，与 cycle 对齐。
- **数值类型**：0~99 锯齿波，步长 1，周期为 cycle。
- **String**：单字符 a~z 循环，周期为 cycle。
- **DateTime**：每周期 +1 秒。

### 2.5 可写与持久化

- 当节点 `writable=true` 时，客户端写成功后，该值持久化在服务端；下次读该节点应返回本次写入的值。

### 2.6 安全与日志

- **用户权限**：暂不实现（匿名访问）。
- **日志**：输出到**执行程序所在目录**下的日志文件（可按天命名）。

## 3. 非功能需求

- 组态缺失或格式错误时，应给出明确错误信息并退出。
- 长时间运行稳定，周期更新不阻塞主循环。
