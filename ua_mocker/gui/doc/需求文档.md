# UA Types Mock GUI — 需求文档

> 流程阶段①产出。状态：**待用户最终确认**（2026-07-22）。

## 1. 目标

开发一个**桌面 GUI 版的 OPC UA Mock 服务器工具**，内置一份 ua_types 风格的固定组态，
打开即用：外部 OPC UA 客户端（UaExpert 等）可直连获取全类型模拟数据，用于联调与测试。

## 2. 功能需求

### 2.1 内置固定组态（ua_types 风格）

- 覆盖全部 **13 种 OPC UA 类型**：Boolean、SByte、Byte、Int16、UInt16、Int32、UInt32、
  Int64、UInt64、Float、Double、String、DateTime
- 每类型 **2 个节点**：
  - 1 个**自动变化只读**节点（change=true, writable=false）
  - 1 个**可写固定值**节点（change=false, writable=true，带默认值）
- 合计 **26 个节点**，命名沿用 ua2_types 约定：`ua2_<type>_r_1` / `ua2_<type>_w_1`
- 命名空间索引 ns=2（与 ua2_types.yaml 一致）
- 节点构成**固定不可增删**

### 2.2 自动变化规则（沿用现有 ua_mocker 引擎）

| 类型 | 规则 |
|------|------|
| Boolean | 每周期翻转 |
| 数值类型（SByte~UInt64、Float、Double） | 0~99 锯齿波，步长 1 |
| String | 单字符 a~z 循环 |
| DateTime | 每周期 +1 秒（起点 2025-01-01T00:00:00） |

### 2.3 界面功能

| 功能 | 说明 |
|------|------|
| 启动 / 停止服务 | 按钮操作，带状态徽标（运行中/已停止） |
| 参数调整 | 端口（默认 18955）、更新周期 cycle（默认 1000ms），**停止状态下可改** |
| 节点列表展示 | 26 个节点的 NodeId、类型、模式（自变化/可写）、默认值 |
| endpoint 展示 | 显示 `opc.tcp://<host>:<port>/...`，可一键复制给客户端使用 |
| 反馈 | 启动失败（如端口占用）以 Toast 提示；空状态友好提示 |

### 2.4 可写节点

- 可写节点接受外部客户端写入（AccessLevel 含 CurrentWrite）
- 写入值持久在内存中（无落盘需求）

## 3. 非功能需求

- GUI 本体单 exe，**体积 ≤ 15MB**（dev-skill GUI 红线）
- 分发为双 exe（GUI + 服务程序，见设计文档路线 B）；**终端用户无需安装 Python 环境**
  （服务程序为 PyInstaller 独立 exe，现有打包配置 main.spec 可直接复用）
- 关闭窗口即释放端口（优雅停止）
- 界面遵循 Notion 式浅色简约设计规范

## 4. 边界（明确不做）

- ❌ 组态编辑（增删节点、改类型构成、改命名）
- ❌ 内置 OPC UA 客户端能力（实时监控值、写值回读）——用外部客户端验证
- ❌ 改造现有 Python 版 ua_mocker（零侵入，仅复用组态约定与变化规则）
- ❌ 报警/事件、历史数据、安全加密（Anonymous + 无加密，与现有 mock 定位一致）
- ❌ 多组态切换、配置导入导出

## 5. 验收标准

1. `wails build` 产物单 exe ≤ 15MB，双击可启动
2. 启动后外部客户端能连上 endpoint，Browse 到 26 个节点
3. 自变化节点按 cycle 周期更新值（锯齿波/翻转/字符循环/+1s 规则正确）
4. 可写节点可被客户端成功写入，读回为新值
5. 停止服务后端口释放，可再次启动
6. 端口被占用时启动失败有明确 Toast 提示
