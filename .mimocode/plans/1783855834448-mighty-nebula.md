## Part 3: baseline alive-wait 超时调长(K 复跑的前置修复)

### 根因(实测确认,非"网络问题")
第三/四轮 baseline BLOCKED 的真实错误是 `datasource 'ua_shared_ua2_types_ds' did not become alive after enable`。两个 mock 每轮都正常起(started:true,ready 2.03s)。**真实原因**:第 2 轮创建的共享 DS(id=80)在 mock 停后变 alive=False;第 3/4 轮 mock 重启后,provisioning 重新 enable 该 DS 并等 alive,但 **alive 等待超时只有 60s**,而"停过又起的 DS 恢复 alive 需要 1~2 分钟"(用户确认)-> 60s 不够 -> BLOCKED。

超时链:`TimeoutsConfig.ds_connect_sec` 默认 60s(config.py:42);provisioning `_ensure_one` 用 `getattr(ctx.config.timeouts, "ds_connect_sec", 60)`(provisioning/ua2_baseline.py:89,103);编排进程 `_provision_shared_baseline` 建 `pcfg = RunConfig()` 但**没设 `pcfg.timeouts.ds_connect_sec`**(run_automation_ua2.py:215-223)-> 用默认 60s。

### 修改(业务级专用 alive-wait,放进 provisioning 层,与通用配置解耦)
**设计原则**(用户确认):共享 DS 停过又起后恢复 alive 是 baseline provisioning 的**业务级**场景,不该复用通用 `ds_connect_sec`。应在 provisioning 层有专用轮询:每 1s 调 API 查 alive,直到 `alive=true`,上限 2 分钟。

- 文件: `ua_test_harness/provisioning/ua2_baseline.py`
- 加模块常量(业务级,非通用 config):
  - `BASELINE_ALIVE_WAIT_SEC = 120`(2 分钟上限,用户确认)
  - `BASELINE_ALIVE_POLL_SEC = 1.0`(轮询间隔,落在用户要求的 1~3s)
- `_wait_ds_alive`:轮询间隔改用 `BASELINE_ALIVE_POLL_SEC`(替硬编码 `time.sleep(1.0)`)。
- `_ensure_one` 两处 alive 等待(新建 DS 后、已有 DS re-enable 后,provisioning/ua2_baseline.py:89,103):`timeout=getattr(ctx.config.timeouts, "ds_connect_sec", 60)` -> `timeout=BASELINE_ALIVE_WAIT_SEC`。**不再读 `ds_connect_sec`**。
- **不改** `run_automation_ua2.py` 的 `_provision_shared_baseline`(无需再覆盖 pcfg.timeouts)。
- **不改** `TimeoutsConfig.ds_connect_sec` 默认值(UA-1/scenario_runtime 仍用 60,不受影响)。
- **不改** case run-config `dsConnectSec:60`(`require_shared_datasource` 只检查 alive 不等待)。
- **不改** provisioning 逻辑结构(不加 disable/enable toggle--用户确认 DS 会自行恢复,只是慢;仅把超时改成业务级专用)。
- 测试: `test_ua2_baseline.py` 已 mock `_wait_ds_alive`,不触发真实超时,无需改;可补一个断言 `_ensure_one` 用 `BASELINE_ALIVE_WAIT_SEC` 而非 `ds_connect_sec`(验证解耦)。

### 验收
- compileall + `pytest ua_test_harness\unit_tests\test_ua2_baseline.py ua_test_harness\unit_tests\test_ua2_first_batch.py -q` 通过。
- grep 确认 `provisioning/ua2_baseline.py` 不再读 `ds_connect_sec`。

### 关键文件
