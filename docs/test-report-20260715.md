## 三、分类详解

### A. TPT 实时值不反映源端变化(~40 条,产品 bug)

**业务影响:** 用户在 TPT 平台查看某位号的实时值,看到的值与 OPC UA 数据源里的实际值不一致。这意味着 TPT 的采集链路可能存在延迟、缓存或数据丢失问题。

**问题列表:**

#### A1. 实时值不变(16 条)
- **现象:** OPC UA 模拟器的节点值在持续变化(如 Double 从 56.0 变到 58.0),但 TPT 读到的实时值始终不变。
- **影响用例:** UA-1-2-01~08(8条,rt_changed timeout), UA-1-3-01~08(8条,[changing] not true)
- **验证方式:** 已用 asyncua 直接读 mock 节点确认值在变,排除 mock 问题。

#### A2. 源端值 ≠ TPT 实时值(17 条)
- **现象:** 同一时刻用 OPC UA 客户端直接读到的值,与 TPT 接口返回的实时值不一致。
- **影响用例:** UA-2-1-001/002/026/027/028/029/032/033/034/037/038/102(12条), UA-3-1-002/004/005/006/010(5条), UA-3-2-005(1条) — 断言名: opcua_matches_rt2

#### A3. 写入后回读不匹配(4 条)
- **现象:** 向位号写入一个值后,TPT 读到的实时值与写入值不一致。
- **影响用例:** UA-2-1-066/071/072/074 — 断言名: rt_matches_write

#### A4. 按位号 ID 查实时值不返回(2 条)
- **现象:** 按位号名称能查到实时值,但按位号 ID 查不返回数据。
- **影响用例:** UA-3-2-002/013 — 断言名: by_id_hit

#### A5. 质量码字段无效(1 条)
- **现象:** TPT 返回的实时值中 quality 字段值不符合预期。
- **影响用例:** UA-3-2-012 — 断言名: has_quality

---

### B. TPT 实时值采集超时(~20 条,产品 bug)

**业务影响:** 用户在 TPT 平台查看某些位号的实时值时,长时间无法获取数据,最终超时。这意味着 TPT 对某些位号的采集配置可能有问题,或者采集链路存在阻塞。

**问题列表:**

#### B1. getRTValue 超时(19 条)
- **现象:** 位号已注册且数据源 alive,但 TPT 在 60 秒内无法返回实时值。
- **影响用例:** UA-1-4-01~06(6条), UA-1-6-05(1条), UA-2-1-076/077/079/080/091/092(6条), UA-2-3-008/021/028(3条), UA-2-4-016(1条), UA-3-2-010/011/016/017/018/020/021(7条)

#### B2. 查询接口超时(1 条)
- **现象:** TPT 位号查询接口(queryWithQuality)超时无响应。
- **影响用例:** UA-2-1-005

---

### C. 测试代码 bug(~33 条,测试团队修复)

**业务影响:** 这些是测试用例代码本身的问题,不影响产品功能判断,但会导致测试无法正确执行。

**问题列表:**

#### C1. 数据源命名前缀错误(7 条)
- **现象:** 测试创建数据源时使用了不允许的名称前缀。
- **影响用例:** UA-1-5-02~09
- **修复方向:** DS 名称需要加 `ua_auto_` 前缀。

#### C2. 未捕获的 API 异常(12 条)
- **现象:** UA-2-5 整章(015~026)全部抛出未捕获异常,可能是导入/分组功能测试代码有 bug。
- **影响用例:** UA-2-5-015~026(12条)
- **修复方向:** 需逐条检查 traceback,大概率是 API 参数错误或接口变更未同步。

#### C3. 测试功能未实现(2 条)
- **现象:** UA-3-1-007/009 报 "precise_frequency: unsupported",测试框架未实现该功能。
- **影响用例:** UA-3-1-007/009

#### C4. 其他测试代码异常(9 条)
- **现象:** 各种未捕获异常,包括 tag group 不存在、参数错误、接口返回格式不符等。
- **影响用例:**
  - UA-1-6-04/10/13(3条): DS 禁用后调 API 未 catch 异常
  - UA-2-2-002(1条): tag group 不存在
  - UA-2-2-042(1条): 异常
  - UA-2-3-009(1条): "limit hierarchy not supported" 参数错误
  - UA-2-3-014(1条): 异常
  - UA-3-2-003/007(2条): 异常

#### C5. 断言数字错误(1 条)
- **现象:** UA-2-2-055 断言 `no_dup_bases expected=520 actual=26`,520 是旧的预期值,实际 mock 只有 26 个节点。
- **影响用例:** UA-2-2-055
- **修复方向:** 更新断言预期值为 26。

### D. 产品行为不符预期(~12 条,产品 bug)

**业务影响:** TPT 平台的某些行为与文档描述不一致,可能导致用户操作结果与预期不符。

**问题列表:**

#### D1. 非法日期未被拒绝(1 条)
- **现象:** 测试向 TPT 传入非法日期格式,TPT 未报错而是接受了。
- **影响用例:** UA-2-1-073 — 断言名: bad_date_rejected

#### D2. 写只读位号未失败(1 条)
- **现象:** 测试向只读位号写入值,TPT 未拒绝,写入"成功"。
- **影响用例:** UA-2-1-085 — 断言名: write_failed_or_ineffective

#### D3. 默认采集频率不符(1 条)
- **现象:** 新建位号的默认采集频率应为 10 秒,实际为 1 秒。
- **影响用例:** UA-2-1-086 — 断言名: default_frequency expected=10 actual=1

#### D4. tagTime 格式不可解析(1 条)
- **现象:** TPT 返回的实时值中 tagTime 字段格式无法被程序解析。
- **影响用例:** UA-2-2-039 — 断言名: tagTime_parseable

#### D5. 导出列数不符(4 条)
- **现象:** TPT 位号导出功能返回的列数为 24,文档预期为 21 列。
- **影响用例:** UA-2-3-001/002/003/006 — 断言名: column_count/header_cols expected=21 actual=24

#### D6. 软删除后仍可写入(1 条)
- **现象:** 位号被软删除(移入回收站)后,仍然可以对其进行写值操作,应被阻止。
- **影响用例:** UA-2-4-009 — 断言名: write_blocked_after_soft_delete

#### D7. 历史数据为空(1 条)
- **现象:** 位号应有历史数据,但查询返回空。
- **影响用例:** UA-2-1-104 — 断言名: history_has_points

#### D8. 静态节点值稳定性断言失败(1 条)
- **现象:** UA-2-2-040 用静态节点测值稳定性,第一次读到 None,第二次读到 77.7。
- **影响用例:** UA-2-2-040 — 断言名: value_stable expected=None actual=77.7
- **说明:** 已从 change=true 节点改为 change=false 静态节点,但仍有首次读取为 None 的问题。

---

### E. 前置条件不满足 / BLOCKED(39 条)

**业务影响:** 这些用例因前置条件不满足而无法执行,不是产品功能问题。

**问题列表:**

#### E1. UA-2-1-004~022 大面积 BLOCKED(18 条)
- **现象:** 这 18 条用例全部 BLOCKED,每条仅耗时 ~500ms,可能是数据源状态或 baseline 前置检查失败。
- **可能原因:** 共享数据源的 tags 残留导致 baseline 校验失败,或 DS alive 状态异常。
- **建议:** 检查 baseline provisioning 逻辑,清理残留 tags 后重跑。

#### E2. 历史数据验证 BLOCKED(6 条)
- **现象:** UA-3-4-001~006 历史数据导入后查询不到足够数据点。
- **可能原因:** TPT 处理导入数据需要时间,或时区不匹配。
- **说明:** 已修复导入后加 3 秒延迟 + 时区改为本地时间,可能仍需更长等待。

#### E3. 性能/探索类 BLOCKED(9 条)
- **现象:** UA-3-5-007~010(4条), UA-3-6-010~014(5条) 因前置条件不满足 BLOCKED。
- **可能原因:** 性能测试需要大量位号或特殊环境配置。

#### E4. 其他 BLOCKED(6 条)
- **现象:** UA-2-2-053(GUI 延迟), UA-2-4-011/012, UA-3-1-019/020, UA-3-3-019
- **说明:** UA-2-2-053 是 GUI 功能延迟实现,其余需逐条检查。
