# UA-3-1 位号采集

验证 OPC UA 源端数据能否按照位号配置进入 DataHub，并在数据源异常后正确恢复。

仅设计现有 API、Mock 控制和 asyncua 可执行的用例。需要历史数据的用例按[历史数据制造指南](../history-data-fixtures.md)执行。

## 约定

- 实时值：`getRTValue(isFromDB=false)`；辅助状态：`queryWithQuality`。
- 源端事实源：asyncua 直读 Mock 节点。
- 历史闭环：`getHistoryValueFromDB` 或 `getHistoryValue`。
- frequency 的固定容差尚未确认，首轮频率用例为探索并记录基线。
- 临时位号使用 `ua31_{runId}_` 前缀。

| 编号 | 三级点 | 类型 | 步骤 | 断言 | 验证 |
|---|---|---|---|---|---|
| UA-3-1-001 | 初始采集_新增后自动开始 | 回归 | 新增读取位号并等待采集 | 无需额外触发；RT 出现有效值且与源端一致 | RT + asyncua |
| UA-3-1-002 | 变化采集_源值连续变化 | 回归 | asyncua 按唯一序列修改源值 | DataHub 依次观察到新值，无长期停滞 | RT + asyncua |
| UA-3-1-003 | 静态采集_值不变化 | 回归 | 源值保持固定并连续查询 | 值不变时质量仍有效，不误判断线 | RT + quality |
| UA-3-1-004 | 数据类型_13种采集 | 回归 | 注册并采集 13 种类型 | 类型、值和表示正确，不串类型 | RT + asyncua |
| UA-3-1-005 | 大整数_Int64与UInt64 | 回归 | 源端设置超过 JS 安全整数范围的值 | 可无损还原，不发生舍入 | 字符串/大整数比较 |
| UA-3-1-006 | DateTime_值与时区 | 回归 | 源端设置已知 UTC 时间 | DataHub 值与源端表示同一时刻 | UTC 转换比较 |
| UA-3-1-007 | 采集频率_1秒5秒10秒 | 探索 | 三个位号设置不同 frequency | 实际更新时间间隔与配置具有对应关系 | RT 时间序列 |
| UA-3-1-008 | 采集频率_同源独立 | 回归 | 同源多个位号设置不同频率 | 各位号按自身频率更新，互不覆盖 | RT 时间序列 |
| UA-3-1-009 | 采集频率_运行中修改 | 探索 | 采集中修改 frequency | 无需重建；记录新频率生效时间 | update + RT |
| UA-3-1-010 | 正常质量与时间 | 回归 | 在线采集多个周期 | quality 有效；tagTime/appTime 可解析且不倒退 | RT + quality |
| UA-3-1-011 | 节点不存在 | 回归 | 绑定不存在 NodeId | 不产生伪造有效值；其他位号正常 | RT + quality |
| UA-3-1-012 | 配置类型与源类型不一致 | 探索 | Double 节点配置为错误类型 | 记录拒绝、转换或质量规则；服务不异常 | API + RT |
| UA-3-1-013 | 数据源断线_停止有效采集 | 回归 | 正常采集后停止 Mock | alive=false；质量无效；不产生伪造的新鲜值 | ds 状态 + RT |
| UA-3-1-014 | 数据源恢复_自动续采 | 回归 | 重启 Mock 并修改源值 | 无需重建；质量和数据自动恢复 | ds 状态 + RT + asyncua |
| UA-3-1-015 | 数据源禁用_停止采集 | 回归 | changeState 禁用数据源 | 采集停止；质量无效；配置仍存在 | ds 状态 + RT |
| UA-3-1-016 | 数据源重新启用 | 回归 | 重新启用数据源 | 自动恢复采集；值与源端一致 | ds 状态 + RT |
| UA-3-1-017 | 单节点异常隔离 | 回归 | 使一个节点读取异常 | 同源其他正常节点继续采集 | RT 集合 |
| UA-3-1-018 | 多数据源隔离 | 回归 | A 断线，B 保持运行 | B 的值、质量和时间不受 A 影响 | 双源 RT |
| UA-3-1-019 | 自动采集_历史落地 | 回归 | 按[方式 A](../history-data-fixtures.md#3-方式-a自动采集造数)制造唯一值序列并查询历史 | 关键唯一值进入历史；身份、时间和质量可核对 | history + RT + asyncua |
| UA-3-1-020 | 删除恢复_采集与历史生命周期 | 探索 | 按[方式 A](../history-data-fixtures.md#3-方式-a自动采集造数)造数，软删、等待、恢复后继续造数 | 记录删除期间是否停采；恢复后实时和历史采集可继续 | recycle + RT + history |

## 执行与清理

1. 恢复 Mock 和数据源启用状态；
2. 删除临时位号；
3. 恢复源端节点原值；
4. 保存 frequency、断线和恢复时间线；
5. 历史造数失败标记为 `setup_failed`，不得判定采集功能失败。
