# UA-2-2 位号查询

验证通过现有 API 能否准确查询已注册位号、配置与运行状态，并浏览可注册的底层节点。

本版本仅设计可由 API、RT 查询和 asyncua 执行并验证的用例。前端页面、按钮、输入框、跨页选择及客户端状态行为不进入当前回归范围。

---

## 设计约定

- 用例按用户查询任务组织，但每条正式用例必须有现有 API 执行入口。
- 配置字段以 `tag-info/page` 为主要事实源。
- 实时值、质量和时间以 `queryWithQuality`、`getRTValue` 为事实源，asyncua 作为源端对照。
- 可注册节点集合使用 `getNotUsedBaseTagInfoContinue` 获取，并以同一 `dsId + tagBaseName` 的已注册集合做差。
- `GUI-DEFERRED` 仅保留原编号占位，不设计步骤，等待前端功能范围明确。
- 临时数据使用 `ua22_{runId}_` 前缀并在结束后清理。

### 基础数据

- 数据源 A、B 均运行；A 至少 12 个已注册位号，B 至少 3 个；
- 两个数据源存在一个相同 `tagBaseName`；
- A 至少 20 个未注册节点；
- 至少两个普通分组和一个空分组；
- 至少一个变化节点、静态节点、软删除位号和收藏位号。
