## 快速开始

### 1. 安装依赖

```bash
cd web_frontend
npm install
```

### 2. 启动后端服务器

```bash
# 在项目根目录
python web_backend/start_server.py
```

### 3. 启动前端开发服务器

```bash
# 在 web_frontend 目录
npm run dev
```

### 4. 访问应用

打开浏览器访问：http://localhost:3000

## API 接口说明

### 1. 模拟预览接口

**POST** `/api/simulate/preview`

请求体：
```json
{
  "dsl_content": "program:\n  - name: sin1\n    type: SINE_WAVE\n    ...",
  "cycle_time": 0.5,
  "preview_steps": 2000,
  "start_time": 0.0,
  "time_format": "%Y-%m-%d %H:%M:%S",
  "total_steps": 10000
}
```

响应：
```json
{
  "status": "ok",
  "data": [...],  // 快照数据列表
  "variable_names": ["sin1.out", "valve1.current_opening", ...],
  "generation_time": 1.234,
  "data_points": 2000,
  "estimated_export_time": 6.17,
  "total_steps": 10000
}
```

### 2. 获取模板列表

**GET** `/api/templates/list`

响应：
```json
{
  "templates": ["ai_loop_tuning", "pid_loop_tuning", "prediction"]
}
```

### 3. 获取默认配置

**GET** `/api/config/default`

响应：
```json
{
  "status": "ok",
  "content": "program:\n  - name: sin1\n    ..."
}
```

### 4. 获取 README

**GET** `/api/readme`

响应：
```json
{
  "status": "ok",
  "content": "# Data Factory Next\n\n..."
}
```

## 使用说明

### 数据模拟页面使用流程

1. **配置参数**：
   - 填写周期（执行周期和采样周期）
   - 填写总周期数（用于预估导出时间）
   - 选择输出模板
   - 编辑或使用默认 DSL 配置
   - 设置模拟绘图周期数（建议 2000）

2. **点击"模拟绘图"按钮**：
   - 前端发送请求到后端
   - 后端创建 engine 实例并运行指定周期数
   - 返回模拟数据和统计信息

3. **查看结果**：
   - 右侧显示曲线图
   - 显示数据点数、生成时间、预估导出时间

## 注意事项

1. **开发环境**：
   - 前端开发服务器运行在 3000 端口
   - 后端 API 服务器运行在 8000 端口
   - Vite 配置了代理，自动转发 `/api/*` 请求到后端

2. **生产环境**：
   - 需要配置环境变量 `VITE_API_BASE_URL` 指定后端地址
   - 需要构建前端项目：`npm run build`
   - 需要配置 Web 服务器（如 Nginx）提供静态文件服务

3. **CORS 配置**：
   - 开发环境允许所有来源（`allow_origins=["*"]`）
   - 生产环境应限制允许的来源

4. **性能考虑**：
   - 模拟预览周期数建议不超过 10000
   - 大数据量导出可能需要较长时间
   - 图表渲染大量数据点可能影响性能
