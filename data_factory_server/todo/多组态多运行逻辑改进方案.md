# 多组态多运行逻辑改进方案

## 一、问题分析

### 1.1 当前架构的局限性

当前系统架构中，`UnifiedEngine` 主要支持：
- **运行模式**：基于表达式/程序图的实时计算
- **组态来源**：YAML/DSL 配置文件
- **单一运行逻辑**：所有 Engine 实例都使用相同的表达式计算逻辑

当需要支持以下场景时，现有架构存在局限性：
- **Excel 回放**：从 Excel/CSV 文件按时间线回放历史数据
- **混合运行**：部分变量由表达式计算，部分变量从文件回放
- **多组态隔离**：不同 Engine 实例需要运行完全不同的组态配置
- **运行逻辑扩展**：未来可能需要支持其他运行逻辑（如物理模型、控制算法等）

### 1.2 核心挑战

1. **运行逻辑耦合**：`UnifiedEngine` 的实现与表达式计算逻辑紧密耦合
2. **组态管理分散**：组态配置分散在各个 Engine 实例中，缺乏统一管理
3. **接口不统一**：不同运行逻辑的输出格式可能不一致，导致下游服务（StorageService、OPCUA）难以统一处理
4. **扩展性不足**：新增运行逻辑需要修改核心 Engine 代码，违反开闭原则

---

## 二、设计目标

### 2.1 核心目标

1. **统一运行接口**：所有运行逻辑都实现统一的接口，输出统一格式的快照
2. **多运行逻辑支持**：支持表达式计算、Excel 回放、混合运行等多种运行逻辑
3. **数据总线架构**：采用"分布式引擎+统一数据总线"架构，所有引擎数据汇聚到单一 Redis Hash，实现下游服务无感透明
4. **多组态管理**：通过 ConfigServer 统一管理多个组态配置，支持配置分离、全局位号唯一性检查和发布活动
5. **向后兼容**：现有代码和配置可以平滑迁移，不破坏现有功能
6. **多进程支持**：利用多进程运行不同引擎，避免 GIL 限制，充分利用多核 CPU

### 2.2 设计原则

- **单一职责原则**：每个 Runtime 只负责一种运行逻辑
- **开闭原则**：对扩展开放，对修改关闭
- **依赖倒置原则**：高层模块（ServiceManager、StorageService）依赖抽象接口，不依赖具体实现
- **接口隔离原则**：提供最小化的统一接口，避免接口臃肿

---

## 三、架构设计

### 3.1 整体架构：数据总线模式 (Data Bus Architecture)

系统解构为 **生产者 (Producer)**、**总线 (Bus)** 和 **消费者 (Consumer)** 三层：

```
┌─────────────────────────────────────────────────────────────┐
│                    ConfigServer (组态服务器)                 │
│  - 基础设施层配置（Engine 实例、Storage 周期、OPCUA 周期）   │
│  - 逻辑定义层配置（DSL 文件，业务算法、实例和变量定义）      │
│  - 全局位号唯一性检查                                        │
│  - 发布活动（统筹 Engine 启停，推送映射表与采集策略）        │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
┌─────────────────────────────────────────────────────────────┐
│                    ServiceManager (编排器)                   │
│  - 执行组态服务器指令                                        │
│  - 管理各计算进程的生命周期                                  │
│  - 根据 Engine 配置创建对应的 Runtime 实例                  │
└─────────────────────────────────────────────────────────────┘
                            │
                            ▼
        ┌───────────────────┼───────────────────┐
        │                   │                   │
        ▼                   ▼                   ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐
│ Expression   │   │  Playback    │   │  (未来扩展)  │
│ Runtime      │   │  Runtime     │   │              │
│ (Simulation) │   │  (Excel回放) │   │              │
└──────────────┘   └──────────────┘   └──────────────┘
        │                   │                   │
        └───────────────────┼───────────────────┘
                            │
                            ▼
              ┌─────────────────────────┐
              │   BaseRuntime (接口)     │
              │   - step()               │
              │   - get_metadata()       │
              └─────────────────────────┘
                            │
                            ▼
              ┌─────────────────────────┐
              │   RealtimePublisher     │
              │   - push_to_bus()       │
              └─────────────────────────┘
                            │
                            ▼
              ┌─────────────────────────┐
              │   数据总线 (Redis)        │
              │   Hash: data_factory:v2: │
              │         current          │
              │   Registry: 全局映射表    │
              │   Stream: (可选) 流式数据 │
              └─────────────────────────┘
                            │
                            ▼

        │                   │                   │
        ▼                   ▼                   ▼
┌──────────────┐   ┌──────────────┐   ┌──────────────┐

└──────────────┘   └──────────────┘   └──────────────┘
```

**核心设计理念：**
- **数据总线即协议**：所有引擎数据汇聚到单一 Redis Hash `data_factory:v2:current`，Field 为位号全名，Value 包含 `value`, `timestamp`, `engine_id`
- **全局映射表 (Global Registry)**：记录 `tag_name -> engine_id` 及其元数据，确保位号跨引擎唯一
- **空间换灵活性**：利用多进程运行不同引擎，避免 GIL 限制，未来可用不同语言开发组件

### 3.2 核心组件设计

#### 3.2.1 BaseRuntime（运行接口抽象）

**职责**：定义所有运行逻辑的统一接口

**接口定义**（伪代码，采用简洁设计）：
```python
from abc import ABC, abstractmethod
from typing import Dict, Any, Optional

class BaseRuntime(ABC):
    """运行逻辑基类（统一接口）"""
    
    def __init__(self, engine_id: str):
        self.engine_id = engine_id
    
    @abstractmethod
    def step(self, tick_info: Dict[str, Any]) -> Dict[str, Any]:
        """
        执行一个周期的逻辑
        
        Args:
            tick_info: 时钟信息，包含 cycle_count, sim_time, need_sample 等
        
        Returns:
            snapshot: 统一格式的快照字典
                {
                    "engine_id": str,           # Engine 实例 ID
                    "cycle_count": int,          # 周期计数
                    "sim_time": float,           # 模拟时间
                    "need_sample": bool,         # 是否需要采样
                    "time_str": str,             # 时间字符串
                    "params": Dict[str, Any],    # 参数字典 {tag_name: value}
                    "exec_ratio": float,         # 执行时间占比（可选）
                }
        """
        pass
    
    @abstractmethod
    def get_metadata(self) -> Dict[str, Any]:
        """
        返回该引擎负责的位号列表及类型
        
        Returns:
            {
                "engine_id": str,
                "tags": List[str],              # 位号全名列表 (namespace.instance.tag)
                "tag_types": Dict[str, str],   # 位号类型映射 {tag_name: type}
                "source_logic": str,           # 来源逻辑 (sim/playback/hybrid)
            }
        
        用于：
        - ConfigServer 全局位号唯一性检查
        - StorageService 预创建索引
        - OPCUA Server 预创建节点
        - 全局映射表 (Global Registry) 注册
        """
        pass
    
    @abstractmethod
    def stop(self) -> None:
        """停止运行"""
        pass
    
    # 可选：保持向后兼容的扩展接口
    def run_realtime(self) -> Iterable[Dict[str, Any]]:
        """实时运行模式（可选，用于向后兼容）"""
        # 默认实现：循环调用 step()
        pass
    
    def run_generator(self, steps: int) -> Iterable[Dict[str, Any]]:
        """生成器模式（可选，用于向后兼容）"""
        # 默认实现：循环调用 step()
        pass
```

#### 3.2.2 ExpressionRuntime（表达式运行逻辑）

**职责**：实现基于表达式/程序图的运行逻辑（当前 `UnifiedEngine` 的封装）

**实现要点**：
- 将现有的 `UnifiedEngine` 封装为 `ExpressionRuntime`
- 保持现有功能不变，只是接口适配
- 输出统一格式的快照

**伪代码**：
```python
class ExpressionRuntime(BaseRuntime):
    """表达式运行逻辑（基于 UnifiedEngine）"""
    
    def __init__(
        self,
        engine_id: str,
        program_config: ProgramConfig,
        clock_config: ClockConfig,
        realtime_config: Optional[RealtimeConfig] = None,
    ):
        self.engine_id = engine_id
        self._engine = UnifiedEngine(
            config=EngineConfig(
                clock=clock_config,
                expressions=program_config.expressions,
            ),
            instances=program_config.instances,
        )
        self._publisher = RealtimePublisher(realtime_config) if realtime_config else None
        self._running = False
    
    def run_realtime(self) -> Iterable[Dict[str, Any]]:
        """实时运行"""
        self._running = True
        for snapshot in self._engine.run_realtime():
            # 统一快照格式
            unified_snapshot = {
                "engine_id": self.engine_id,
                "cycle_count": snapshot.get("cycle_count", 0),
                "sim_time": snapshot.get("sim_time", 0.0),
                "need_sample": snapshot.get("need_sample", False),
                "time_str": snapshot.get("time_str", ""),
                "params": {
                    k: v for k, v in snapshot.items()
                    if k not in ["cycle_count", "need_sample", "time_str", "sim_time", "exec_ratio"]
                },
                "exec_ratio": snapshot.get("exec_ratio", 0.0),
            }
            
            # 推送快照
            if self._publisher:
                self._publisher.push_snapshot(unified_snapshot)
            
            yield unified_snapshot
    
    def run_generator(self, steps: int) -> Iterable[Dict[str, Any]]:
        """生成器模式"""
        results = self._engine.run_generator(steps)
        for snapshot in results:
            unified_snapshot = {
                "engine_id": self.engine_id,
                "cycle_count": snapshot.get("cycle_count", 0),
                "sim_time": snapshot.get("sim_time", 0.0),
                "need_sample": snapshot.get("need_sample", False),
                "time_str": snapshot.get("time_str", ""),
                "params": snapshot.get("params", {}),
                "exec_ratio": snapshot.get("exec_ratio", 0.0),
            }
            yield unified_snapshot
    
    def get_param_names(self) -> list[str]:
        """获取参数名称列表"""
        # 从 program_config 中提取所有参数名称
        param_names = []
        for item in self._engine._program_items:
            if hasattr(item, 'output'):
                param_names.append(item.output)
        return param_names
```

#### 3.2.3 ReplayExcelRuntime（Excel 回放运行逻辑）

**职责**：从 Excel/CSV 文件按时间线回放数据

**实现要点**：
- 读取 Excel/CSV 文件，解析时间列和数值列
- 按时间线推进，生成快照
- 支持加速回放（时间缩放）

**伪代码**：
```python
import pandas as pd
from datetime import datetime, timedelta

class ReplayExcelRuntime(BaseRuntime):
    """Excel 回放运行逻辑"""
    
    def __init__(
        self,
        engine_id: str,
        excel_path: str,
        time_column: str,
        value_columns: Dict[str, str],  # {param_name: excel_column_name}
        clock_config: ClockConfig,
        realtime_config: Optional[RealtimeConfig] = None,
        replay_speed: float = 1.0,  # 回放速度倍数（1.0=正常速度）
    ):
        self.engine_id = engine_id
        self.clock_config = clock_config
        self.replay_speed = replay_speed
        self._publisher = RealtimePublisher(realtime_config) if realtime_config else None
        
        # 读取 Excel 数据
        df = pd.read_excel(excel_path)
        if time_column not in df.columns:
            raise ValueError(f"时间列不存在: {time_column}")
        
        # 转换时间列为 datetime
        df[time_column] = pd.to_datetime(df[time_column])
        df = df.sort_values(by=time_column)
        
        self._data = df
        self._time_column = time_column
        self._value_columns = value_columns
        self._current_index = 0
        self._running = False
        self._current_snapshot: Optional[Dict[str, Any]] = None
    
    def run_realtime(self) -> Iterable[Dict[str, Any]]:
        """实时回放模式"""
        self._running = True
        self._current_index = 0
        
        if len(self._data) == 0:
            return
        
        start_time = datetime.now()
        first_row_time = self._data.iloc[0][self._time_column]
        
        while self._running and self._current_index < len(self._data):
            row = self._data.iloc[self._current_index]
            row_time = row[self._time_column]
            
            # 计算应该等待的时间（考虑回放速度）
            if self._current_index > 0:
                prev_row_time = self._data.iloc[self._current_index - 1][self._time_column]
                time_diff = (row_time - prev_row_time).total_seconds() / self.replay_speed
                if time_diff > 0:
                    time.sleep(time_diff)
            
            # 构建快照
            params = {}
            for param_name, excel_col in self._value_columns.items():
                if excel_col in row:
                    params[param_name] = row[excel_col]
            
            snapshot = {
                "engine_id": self.engine_id,
                "cycle_count": self._current_index,
                "sim_time": (row_time - first_row_time).total_seconds(),
                "need_sample": True,  # Excel 回放通常每个点都需要采样
                "time_str": row_time.isoformat(),
                "params": params,
                "exec_ratio": 0.0,
            }
            
            self._current_snapshot = snapshot
            
            # 推送快照
            if self._publisher:
                self._publisher.push_snapshot(snapshot)
            
            yield snapshot
            self._current_index += 1
    
    def run_generator(self, steps: int) -> Iterable[Dict[str, Any]]:
        """生成器模式（快速回放，不等待）"""
        self._running = True
        end_index = min(self._current_index + steps, len(self._data))
        
        if len(self._data) == 0:
            return
        
        first_row_time = self._data.iloc[0][self._time_column]
        
        for idx in range(self._current_index, end_index):
            row = self._data.iloc[idx]
            row_time = row[self._time_column]
            
            params = {}
            for param_name, excel_col in self._value_columns.items():
                if excel_col in row:
                    params[param_name] = row[excel_col]
            
            snapshot = {
                "engine_id": self.engine_id,
                "cycle_count": idx,
                "sim_time": (row_time - first_row_time).total_seconds(),
                "need_sample": True,
                "time_str": row_time.isoformat(),
                "params": params,
                "exec_ratio": 0.0,
            }
            
            self._current_snapshot = snapshot
            yield snapshot
        
        self._current_index = end_index
    
    def get_param_names(self) -> list[str]:
        """获取参数名称列表"""
        return list(self._value_columns.keys())
```

#### 3.2.4 HybridRuntime（混合运行逻辑）

**职责**：部分变量由表达式计算，部分变量从文件回放

**实现要点**：
- 内部包含一个 `ExpressionRuntime` 和一个 `ReplayExcelRuntime`
- 合并两者的快照，表达式计算的参数优先

**伪代码**：
```python
class HybridRuntime(BaseRuntime):
    """混合运行逻辑（表达式 + 回放）"""
    
    def __init__(
        self,
        engine_id: str,
        expression_runtime: ExpressionRuntime,
        replay_runtime: ReplayExcelRuntime,
        merge_strategy: str = "expression_first",  # "expression_first" | "replay_first" | "merge"
    ):
        self.engine_id = engine_id
        self._expr_runtime = expression_runtime
        self._replay_runtime = replay_runtime
        self._merge_strategy = merge_strategy
    
    def run_realtime(self) -> Iterable[Dict[str, Any]]:
        """实时运行（混合）"""
        expr_iter = self._expr_runtime.run_realtime()
        replay_iter = self._replay_runtime.run_realtime()
        
        for expr_snapshot, replay_snapshot in zip(expr_iter, replay_iter):
            # 合并快照
            merged_snapshot = self._merge_snapshots(expr_snapshot, replay_snapshot)
            yield merged_snapshot
    
    def _merge_snapshots(
        self,
        expr_snapshot: Dict[str, Any],
        replay_snapshot: Dict[str, Any],
    ) -> Dict[str, Any]:
        """合并两个快照"""
        if self._merge_strategy == "expression_first":
            merged = replay_snapshot.copy()
            merged["params"].update(expr_snapshot["params"])
        elif self._merge_strategy == "replay_first":
            merged = expr_snapshot.copy()
            merged["params"].update(replay_snapshot["params"])
        else:  # merge
            merged = expr_snapshot.copy()
            merged["params"].update(replay_snapshot["params"])
        
        # 使用表达式运行的时间信息
        merged["cycle_count"] = expr_snapshot["cycle_count"]
        merged["sim_time"] = expr_snapshot["sim_time"]
        merged["time_str"] = expr_snapshot["time_str"]
        
        return merged
```

### 3.3 ConfigServer（组态服务器）- 核心大脑

**职责**：统一管理多个组态配置和 Engine 实例配置，作为系统的核心大脑

**功能**：
1. **配置分离**：
   - **基础设施层**：定义 Engine 实例（ID、类型）、Storage 周期（采样间隔）、OPCUA 周期（更新频率）
   - **逻辑定义层**：原有的 DSL 文件，仅关注业务算法、实例和变量定义
2. **全局位号唯一性检查**：在"组态发布"前，Load 所有逻辑层定义，进行全局位号冲突检测。若存在重复位号，拒绝发布
3. **发布活动 (Publishing)**：
   - 统筹所有 Engine 的启停
   - 向下游服务推送全量映射表与采集策略
   - 触发 OPCUA Server 节点树重建
4. **全局映射表管理**：维护运行时副本，记录 `tag_name -> engine_id` 及其元数据

**配置模型**（伪代码）：
```python
from dataclasses import dataclass
from typing import Dict, List, Optional, Set
from enum import Enum

class EngineKind(str, Enum):
    """Engine 运行类型"""
    EXPRESSION = "EXPRESSION"        # 表达式引擎（Simulation）
    REPLAY_EXCEL = "REPLAY_EXCEL"   # Excel 回放（Playback）
    # 未来可扩展：PHYSICAL_MODEL, CONTROL_ALGORITHM 等

class EngineMode(str, Enum):
    """Engine 运行模式"""
    REALTIME = "REALTIME"            # 实时运行
    GENERATOR = "GENERATOR"          # 生成器模式
    REPLAY = "REPLAY"                # 回放模式

@dataclass
class InfrastructureConfig:
    """基础设施层配置"""
    engines: List[Dict[str, Any]]  # Engine 实例配置列表
    storage: Dict[str, Any]        # Storage 配置（采样间隔等）
    opcua: Dict[str, Any]          # OPCUA 配置（更新频率等）

@dataclass
class EngineInstanceConfig:
    """Engine 实例配置（基础设施层）"""
    engine_id: str
    kind: EngineKind
    mode: EngineMode
    namespaces: List[str]  # 使用的组态命名空间（仅对 EXPRESSION 有效）
    clock_config: ClockConfig
    
    # Expression 相关配置
    program_config_path: Optional[str] = None  # YAML/DSL 配置文件路径
    
    # Replay 相关配置
    replay_source: Optional[Dict[str, Any]] = None  # 回放源配置
    # {
    #     "type": "excel",
    #     "path": "path/to/file.xlsx",
    #     "time_column": "t",
    #     "value_columns": {
    #         "ns101.sin1.out": "sin_out",
    #         "ns101.tank1.level": "tank_level",
    #     },
    #     "replay_speed": 1.0,
    # }
    
    # 实时推送配置
    realtime_config: Optional[RealtimeConfig] = None

@dataclass
class ConfigServerConfig:
    """ConfigServer 配置"""
    infrastructure_config_path: str  # 基础设施层配置文件路径
    logic_config_base_path: str     # 逻辑定义层配置文件基础路径
    enable_dynamic_reload: bool = True  # 是否启用动态重载

class ConfigServer:
    """组态服务器（核心大脑）"""
    
    def __init__(self, config: ConfigServerConfig):
        self.config = config
        self._engine_configs: Dict[str, EngineInstanceConfig] = {}
        self._namespace_configs: Dict[str, ProgramConfig] = {}  # 命名空间 -> 组态配置
        self._global_registry: Dict[str, Dict[str, Any]] = {}  # 全局映射表 {tag_name: {engine_id, type, ...}}
    
    def load_infrastructure_config(self, config_path: str) -> None:
        """加载基础设施层配置"""
        # 解析 YAML 配置文件
        with open(config_path, 'r', encoding='utf-8') as f:
            infra_config = yaml.safe_load(f)
        
        # 加载 Engine 实例配置
        for engine_config_dict in infra_config.get('engines', []):
            engine_config = EngineInstanceConfig(**engine_config_dict)
            self._engine_configs[engine_config.engine_id] = engine_config
            
            # 加载对应的逻辑定义层配置
            if engine_config.kind == EngineKind.EXPRESSION:
                for namespace in engine_config.namespaces:
                    logic_path = f"{self.config.logic_config_base_path}/{namespace}.yaml"
                    self.load_namespace_config(namespace, logic_path)
    
    def load_namespace_config(self, namespace: str, config_path: str) -> None:
        """加载命名空间组态配置（逻辑定义层）"""
        parser = DSLParser()
        config = parser.parse_file(config_path)
        self._namespace_configs[namespace] = config
    
    def validate_global_tag_uniqueness(self) -> bool:
        """
        检查全局位号唯一性
        
        Returns:
            True 如果所有位号唯一，否则抛出 ValueError
        
        Raises:
            ValueError: 如果存在位号冲突
        """
        all_tags: Set[str] = set()
        conflicts: List[str] = []
        
        # 收集所有引擎的位号
        for engine_id, engine_config in self._engine_configs.items():
            if engine_config.kind == EngineKind.EXPRESSION:
                # 从命名空间配置中提取位号
                for namespace in engine_config.namespaces:
                    ns_config = self._namespace_configs.get(namespace)
                    if ns_config:
                        tags = self._extract_tags_from_config(ns_config, namespace)
                        for tag in tags:
                            if tag in all_tags:
                                conflicts.append(tag)
                            all_tags.add(tag)
            elif engine_config.kind == EngineKind.REPLAY_EXCEL:
                # 从回放配置中提取位号
                if engine_config.replay_source:
                    tags = list(engine_config.replay_source.get("value_columns", {}).keys())
                    for tag in tags:
                        if tag in all_tags:
                            conflicts.append(tag)
                        all_tags.add(tag)
        
        if conflicts:
            raise ValueError(f"全局位号冲突检测失败，存在重复位号: {conflicts}")
        
        return True
    
    def _extract_tags_from_config(self, config: ProgramConfig, namespace: str) -> List[str]:
        """从配置中提取位号列表"""
        tags = []
        for instance_name, instance in config.instances.items():
            # 假设实例有 output 属性或类似机制
            if hasattr(instance, 'output'):
                tags.append(f"{namespace}.{instance_name}.{instance.output}")
        return tags
    
    def publish_configuration(self) -> Dict[str, Any]:
        """
        发布组态配置（发布活动）
        
        执行流程：
        1. 验证全局位号唯一性
        2. 构建全局映射表
        3. 通知下游服务（StorageService、OPCUA Server）
        
        Returns:
            发布结果，包含全局映射表等信息
        """
        # 1. 验证唯一性
        self.validate_global_tag_uniqueness()
        
        # 2. 构建全局映射表
        self._build_global_registry()
        
        # 3. 通知下游服务
        self._notify_downstream_services()
        
        return {
            "status": "published",
            "engine_count": len(self._engine_configs),
            "tag_count": len(self._global_registry),
            "registry": self._global_registry,
        }
    
    def _build_global_registry(self) -> None:
        """构建全局映射表"""
        self._global_registry = {}
        
        for engine_id, engine_config in self._engine_configs.items():
            if engine_config.kind == EngineKind.EXPRESSION:
                for namespace in engine_config.namespaces:
                    ns_config = self._namespace_configs.get(namespace)
                    if ns_config:
                        tags = self._extract_tags_from_config(ns_config, namespace)
                        for tag in tags:
                            self._global_registry[tag] = {
                                "engine_id": engine_id,
                                "namespace": namespace,
                                "type": "simulation",
                                "source_logic": "sim",
                            }
            elif engine_config.kind == EngineKind.REPLAY_EXCEL:
                if engine_config.replay_source:
                    tags = list(engine_config.replay_source.get("value_columns", {}).keys())
                    for tag in tags:
                        self._global_registry[tag] = {
                            "engine_id": engine_id,
                            "type": "playback",
                            "source_logic": "playback",
                        }
    
    def _notify_downstream_services(self) -> None:
        """通知下游服务组态变更"""
        # 通过消息总线或 Redis 发布组态更新事件
        # 触发 StorageService 更新采集策略
        # 触发 OPCUA Server 重建节点树
        pass
    
    def create_runtime(self, engine_id: str) -> BaseRuntime:
        """根据 Engine 配置创建对应的 Runtime 实例"""
        engine_config = self.get_engine_config(engine_id)
        if engine_config is None:
            raise ValueError(f"Engine 配置不存在: {engine_id}")
        
        if engine_config.kind == EngineKind.EXPRESSION:
            program_config = self._merge_namespace_configs(engine_config.namespaces)
            return ExpressionRuntime(
                engine_id=engine_id,
                program_config=program_config,
                clock_config=engine_config.clock_config,
                realtime_config=engine_config.realtime_config,
            )
        elif engine_config.kind == EngineKind.REPLAY_EXCEL:
            return PlaybackRuntime(
                engine_id=engine_id,
                excel_path=engine_config.replay_source["path"],
                time_column=engine_config.replay_source["time_column"],
                value_columns=engine_config.replay_source["value_columns"],
                clock_config=engine_config.clock_config,
                realtime_config=engine_config.realtime_config,
                replay_speed=engine_config.replay_source.get("replay_speed", 1.0),
            )
        else:
            raise ValueError(f"不支持的 Engine 类型: {engine_config.kind}")
    
    def _merge_namespace_configs(self, namespaces: List[str]) -> ProgramConfig:
        """合并多个命名空间的组态配置"""
        merged_instances = {}
        merged_expressions = []
        
        for namespace in namespaces:
            ns_config = self.get_namespace_config(namespace)
            if ns_config:
                for instance_name, instance in ns_config.instances.items():
                    merged_instances[f"{namespace}.{instance_name}"] = instance
                for expr in ns_config.expressions:
                    merged_expressions.append(self._prefix_expression(expr, namespace))
        
        return ProgramConfig(
            instances=merged_instances,
            expressions=merged_expressions,
        )
    
    def get_engine_config(self, engine_id: str) -> Optional[EngineInstanceConfig]:
        """获取 Engine 实例配置"""
        return self._engine_configs.get(engine_id)
    
    def get_namespace_config(self, namespace: str) -> Optional[ProgramConfig]:
        """获取命名空间组态配置"""
        return self._namespace_configs.get(namespace)
    
    def get_global_registry(self) -> Dict[str, Dict[str, Any]]:
        """获取全局映射表"""
        return self._global_registry.copy()
```

### 3.4 RealtimePublisher 改造（数据总线模式）

**改造要点**：
1. **多路复用大 Hash**：所有引擎数据写入统一的 Redis Hash `data_factory:v2:current`
2. **数据结构**：Field 为位号全名 (`namespace.instance.tag`)，Value 为包含 `value`, `timestamp`, `engine_id` 的 JSON 字符串
3. **注册机制**：启动时自动将自己负责的位号清单写入 `data_factory:registry`

**伪代码**：
```python
class RealtimePublisher:
    """实时数据发布器（数据总线模式）"""
    
    REDIS_HASH_KEY = "data_factory:v2:current"  # 统一的数据总线 Hash
    REGISTRY_KEY = "data_factory:registry"      # 全局映射表
    
    def __init__(self, config: RealtimeConfig, engine_id: str):
        self.config = config
        self.engine_id = engine_id
        self._redis_client = redis.Redis(...)
        self._metadata: Optional[Dict[str, Any]] = None
    
    def register_metadata(self, metadata: Dict[str, Any]) -> None:
        """注册引擎元数据到全局映射表"""
        self._metadata = metadata
        # 将位号清单写入全局映射表
        registry_data = {
            self.engine_id: {
                "tags": metadata.get("tags", []),
                "tag_types": metadata.get("tag_types", {}),
                "source_logic": metadata.get("source_logic", "unknown"),
            }
        }
        self._redis_client.hset(
            self.REGISTRY_KEY,
            self.engine_id,
            json.dumps(registry_data[self.engine_id])
        )
    
    def push_to_bus(self, snapshot: Dict[str, Any]) -> None:
        """
        推送快照到数据总线（统一 Hash）
        
        Args:
            snapshot: 快照字典，包含 params 字段 {tag_name: value}
        """
        timestamp = time.time()
        engine_id = snapshot.get("engine_id", self.engine_id)
        
        # 将每个位号写入统一 Hash
        for tag_name, value in snapshot.get("params", {}).items():
            # Value 为 JSON 字符串，包含 value, timestamp, engine_id
            hash_value = json.dumps({
                "value": value,
                "timestamp": timestamp,
                "engine_id": engine_id,
                "cycle_count": snapshot.get("cycle_count", 0),
                "sim_time": snapshot.get("sim_time", 0.0),
            })
            
            # Field 为位号全名，写入统一 Hash
            self._redis_client.hset(
                self.REDIS_HASH_KEY,
                tag_name,
                hash_value
            )
        
        # 发布 Pub/Sub 通知（可选）
        self._redis_client.publish(
            self.config.pubsub_channel,
            json.dumps({
                "event": "snapshot_updated",
                "engine_id": engine_id,
                "timestamp": timestamp,
            })
        )
```

### 3.5 StorageService 改造（数据总线消费）

**改造要点**：
1. **监听模式**：从统一 Hash `data_factory:v2:current` 中读取数据
2. **方案 A（SCAN 模式）**：使用 `HGETALL` 或 `HSCAN` 命令读取整个 Hash
3. **方案 B（Streams 模式，推荐）**：使用 Redis Streams，所有引擎作为生产者写入同一个 Stream，Storage 作为消费组

**伪代码**：
```python
class StorageService:
    """存储服务（数据总线消费）"""
    
    REDIS_HASH_KEY = "data_factory:v2:current"  # 统一的数据总线 Hash
    REDIS_STREAM_KEY = "data_factory:v2:stream"  # Redis Streams（可选）
    
    def __init__(self, config: StorageServiceConfig):
        self.config = config
        self._redis_client = redis.Redis(...)
        self._use_streams = config.use_redis_streams  # 是否使用 Streams 模式
    
    def _read_from_bus_hash(self) -> Dict[str, Any]:
        """从统一 Hash 读取数据（方案 A）"""
        # 使用 HGETALL 读取整个 Hash
        all_data = self._redis_client.hgetall(self.REDIS_HASH_KEY)
        
        # 解析每个位号的数据
        snapshot = {}
        for tag_name_bytes, value_json_bytes in all_data.items():
            tag_name = tag_name_bytes.decode('utf-8')
            value_data = json.loads(value_json_bytes.decode('utf-8'))
            snapshot[tag_name] = value_data
        
        return snapshot
    
    def _read_from_bus_stream(self) -> List[Dict[str, Any]]:
        """从 Redis Stream 读取数据（方案 B，推荐）"""
        # 使用 XREADGROUP 从消费组读取
        messages = self._redis_client.xreadgroup(
            groupname="storage_service",
            consumername="storage_worker_1",
            streams={self.REDIS_STREAM_KEY: ">"},
            count=100,
            block=1000,
        )
        
        snapshots = []
        for stream, msgs in messages:
            for msg_id, fields in msgs:
                # 解析消息
                snapshot = json.loads(fields[b'data'].decode('utf-8'))
                snapshots.append(snapshot)
                # 确认消息已处理
                self._redis_client.xack(self.REDIS_STREAM_KEY, "storage_service", msg_id)
        
        return snapshots
    
    def _run_loop(self) -> None:
        """主运行循环"""
        while self._running:
            try:
                if self._use_streams:
                    snapshots = self._read_from_bus_stream()
                    for snapshot in snapshots:
                        self._store_snapshot(snapshot)
                else:
                    snapshot = self._read_from_bus_hash()
                    if snapshot:
                        self._store_snapshot(snapshot)
                
                time.sleep(self.config.sample_interval)
            except Exception as e:
                logger.error(f"Error in storage loop: {e}", exc_info=True)
                time.sleep(1)
```

### 3.6 ServiceManager 改造（演进为编排器）

**改造要点**：
1. 引入 `ConfigServer`，统一管理组态和 Engine 配置
2. 根据 ConfigServer 的发布活动执行 Engine 启停
3. 支持多进程运行不同引擎（避免 GIL 限制）

**伪代码**：
```python
import multiprocessing

class ServiceManager:
    """服务管理器（演进为编排器）"""
    
    def __init__(self, config: ServiceManagerConfig):
        self.config = config
        
        # 初始化 ConfigServer
        self.config_server = ConfigServer(
            ConfigServerConfig(
                infrastructure_config_path=config.infrastructure_config_path,
                logic_config_base_path=config.logic_config_base_path,
                enable_dynamic_reload=config.enable_dynamic_reload,
            )
        )
        
        # Engine 进程管理（多进程模式）
        self._engine_processes: Dict[str, multiprocessing.Process] = {}
        # 或使用多线程模式（初期）
        self._engine_threads: Dict[str, threading.Thread] = {}
        self._use_multiprocess = config.use_multiprocess  # 是否使用多进程
    
    def start_all_engines(self) -> None:
        """启动所有引擎（根据 ConfigServer 发布活动）"""
        # 1. 加载基础设施配置
        self.config_server.load_infrastructure_config(
            self.config_server.config.infrastructure_config_path
        )
        
        # 2. 发布组态配置（会进行唯一性检查）
        publish_result = self.config_server.publish_configuration()
        logger.info(f"组态配置已发布: {publish_result}")
        
        # 3. 启动所有引擎
        for engine_id in self.config_server._engine_configs.keys():
            self.start_engine(engine_id)
    
    def start_engine(self, engine_id: str) -> None:
        """启动 Engine 实例"""
        engine_config = self.config_server.get_engine_config(engine_id)
        if engine_config is None:
            raise ValueError(f"Engine 配置不存在: {engine_id}")
        
        if self._use_multiprocess:
            # 多进程模式（推荐，避免 GIL 限制）
            process = multiprocessing.Process(
                target=self._run_engine_process,
                args=(engine_id,),
                daemon=True,
            )
            process.start()
            self._engine_processes[engine_id] = process
        else:
            # 多线程模式（初期，向后兼容）
            runtime = self.config_server.create_runtime(engine_id)
            thread = threading.Thread(
                target=self._run_engine_thread,
                args=(runtime,),
                daemon=True,
            )
            thread.start()
            self._engine_threads[engine_id] = thread
    
    def _run_engine_process(self, engine_id: str) -> None:
        """在独立进程中运行引擎"""
        # 在子进程中重新初始化 Runtime（避免共享状态）
        runtime = self.config_server.create_runtime(engine_id)
        self._run_engine_loop(runtime)
    
    def _run_engine_thread(self, runtime: BaseRuntime) -> None:
        """在独立线程中运行引擎"""
        self._run_engine_loop(runtime)
    
    def _run_engine_loop(self, runtime: BaseRuntime) -> None:
        """引擎运行循环"""
        clock = Clock(runtime.config.clock_config)
        
        while True:
            tick_info = clock.step()
            snapshot = runtime.step(tick_info)
            
            # 推送到数据总线
            if runtime._publisher:
                runtime._publisher.push_to_bus(snapshot)
            
            # 根据 mode 决定是否 sleep
            if runtime.config.mode == EngineMode.REALTIME:
                time.sleep(runtime.config.clock_config.cycle_time)
```

## 七、技术细节

### 7.1 快照格式统一

所有 Runtime 输出的快照必须遵循统一格式：

```python
{
    "engine_id": str,              # Engine 实例 ID（必需）
    "cycle_count": int,            # 周期计数（必需）
    "sim_time": float,             # 模拟时间（秒）（必需）
    "need_sample": bool,           # 是否需要采样（必需）
    "time_str": str,               # 时间字符串（ISO 格式）（必需）
    "params": Dict[str, Any],      # 参数字典（必需）
    "exec_ratio": float,           # 执行时间占比（可选）
}
```

### 7.2 参数命名规范

- **Expression Runtime**：参数名格式为 `{namespace}.{instance_name}.{param_name}`
- **Replay Runtime**：参数名由配置中的 `value_columns` 映射决定
- **Hybrid Runtime**：参数名合并两个 Runtime 的参数

### 7.3 时间同步

- **Expression Runtime**：使用 `Clock` 管理模拟时间
- **Replay Runtime**：使用 Excel 文件中的时间列
- **Hybrid Runtime**：以 Expression Runtime 的时间为准

### 7.4 数据总线架构

- **统一 Redis Hash**：`data_factory:v2:current`
  - Field：位号全名 (`namespace.instance.tag`)
  - Value：JSON 字符串 `{"value": ..., "timestamp": ..., "engine_id": ...}`
- **全局映射表**：`data_factory:registry`
  - 记录 `engine_id -> {tags, tag_types, source_logic}` 映射
- **Redis Streams（可选）**：`data_factory:v2:stream`
  - 所有引擎作为生产者写入 Stream
  - StorageService 作为消费组从 Stream 读取
- **StorageService**：从统一 Hash 或 Stream 读取，按 `engine_id` 和 `tag_name` 存储
- **OPCUA Server**：基于全局映射表动态构建节点树，节点路径为 `Objects/{tag_name}`

## 十一、总结与建议

本改进方案通过引入 **数据总线架构** 和 **ConfigServer** 统一管理，实现了：

1. **数据总线模式**：所有引擎数据汇聚到单一 Redis Hash，实现下游服务无感透明
2. **多运行逻辑支持**：表达式计算、Excel 回放等，通过启动多个独立引擎在总线上汇聚实现混合运行
3. **多组态管理**：配置分离（基础设施层 + 逻辑定义层），全局位号唯一性检查，发布活动机制
4. **向后兼容**：现有功能不受影响，可以渐进式迁移
5. **可扩展性**：新运行逻辑只需实现 `BaseRuntime` 接口，无需修改核心代码
6. **多进程支持**：利用多进程运行不同引擎，避免 GIL 限制，充分利用多核 CPU

**核心建议：空间换灵活性**

1. **不要合并进程**：利用多进程运行不同的 Engine，可以避免 GIL 锁限制，充分利用多核 CPU 处理大规模模拟
2. **数据总线即协议**：只要定义好了 Redis 中的快照格式，未来甚至可以用 Node.js 或 Go 开发组件并直接并入系统，无需修改现有 Python 核心
3. **分阶段实施**：先做数据总线架构和 ConfigServer（确保不坏），再做回放系统的并入

通过分阶段实施，可以逐步完善功能，降低实施风险。
