# 消息中间件示例代码

本目录包含消息中间件的各种使用场景示例，每个场景都可以独立运行。

## 场景列表

### 场景1：命令-响应模式（同步调用）

演示如何使用消息中间件进行同步的服务调用。

**文件：**
- `场景1_命令响应模式/service.py` - 服务端（计算服务）
- `场景1_命令响应模式/client.py` - 客户端
- `场景1_命令响应模式/完整示例.py` - 完整示例（包含服务端和客户端）

**运行方式：**
```bash
# 方式1：分别运行
python 场景1_命令响应模式/service.py
python 场景1_命令响应模式/client.py

# 方式2：运行完整示例
python 场景1_命令响应模式/完整示例.py
```

### 场景2：异步命令模式

演示如何使用异步命令模式，发送命令后不等待响应，可以继续执行其他操作。

**文件：**
- `场景2_异步命令模式/service.py` - 服务端（数据处理服务）
- `场景2_异步命令模式/client.py` - 客户端
- `场景2_异步命令模式/完整示例.py` - 完整示例

**运行方式：**
```bash
python 场景2_异步命令模式/完整示例.py
```

### 场景3：发布-订阅模式（事件驱动）

演示如何使用发布-订阅模式实现事件驱动的消息传递。

**文件：**
- `场景3_发布订阅模式/publisher.py` - 发布者
- `场景3_发布订阅模式/subscriber1.py` - 订阅者1（配置重载）
- `场景3_发布订阅模式/subscriber2.py` - 订阅者2（缓存清理）
- `场景3_发布订阅模式/完整示例.py` - 完整示例

**运行方式：**
```bash
# 方式1：分别运行
python 场景3_发布订阅模式/subscriber1.py
python 场景3_发布订阅模式/subscriber2.py
python 场景3_发布订阅模式/publisher.py

# 方式2：运行完整示例
python 场景3_发布订阅模式/完整示例.py
```

## 前置要求

1. **Redis 服务**：确保 Redis 服务正在运行
   ```bash
   # 默认配置：localhost:6379
   ```

2. **Python 环境**：确保已安装所需依赖
   ```bash
   pip install redis
   ```

## 注意事项

1. **Redis 数据库**：所有示例默认使用 `db=0`，如需修改请编辑各示例文件中的 `BusConfig`
2. **Key 前缀**：每个场景使用不同的 key 前缀，避免冲突
3. **独立运行**：每个场景的示例代码都可以独立运行，互不干扰

## 示例说明

每个场景目录下都有：
- `README.md` - 场景说明和运行方式
- 独立的服务端/客户端文件 - 可以分别运行
- `完整示例.py` - 在一个进程中运行所有组件，方便测试
