## 快速开始

### 1. 前置要求

- Redis 服务正在运行（默认：localhost:6379）
- Python 3.7+
- 已安装 redis 库：`pip install redis`

### 2. 运行示例

#### 方式1：运行完整示例（推荐）

每个场景都提供了 `完整示例.py`，可以在一个进程中运行所有组件：

```bash
# 场景1：命令-响应模式
python message_bus/example/场景1_命令响应模式/完整示例.py

# 场景2：异步命令模式
python message_bus/example/场景2_异步命令模式/完整示例.py

# 场景3：发布-订阅模式
python message_bus/example/场景3_发布订阅模式/完整示例.py

# 场景4：服务注册与发现
python message_bus/example/场景4_服务注册与发现/完整示例.py

# 场景5：综合场景
python message_bus/example/场景5_综合场景/完整示例.py
```

#### 方式2：分别运行服务端和客户端

对于需要交互的场景，可以分别运行服务端和客户端：

**场景1：命令-响应模式**
```bash
# 终端1：启动服务端
python message_bus/example/场景1_命令响应模式/service.py

# 终端2：启动客户端
python message_bus/example/场景1_命令响应模式/client.py
```

**场景3：发布-订阅模式**
```bash
# 终端1：启动订阅者1
python message_bus/example/场景3_发布订阅模式/subscriber1.py

# 终端2：启动订阅者2
python message_bus/example/场景3_发布订阅模式/subscriber2.py

# 终端3：启动发布者
python message_bus/example/场景3_发布订阅模式/publisher.py
```

## 场景说明

### 场景1：命令-响应模式（同步调用）

**功能**：演示同步服务调用，客户端发送命令后等待响应。

**关键代码**：
```python
# 客户端
result = client.call("calculator_service", "calculate", {"a": 10, "b": 20})

# 服务端
def handle_calculate(payload):
    return {"result": payload["a"] + payload["b"]}
```

**适用场景**：
- 需要立即获取结果的调用
- 简单的请求-响应模式
- 同步操作

### 场景2：异步命令模式

**功能**：演示异步命令，发送命令后不等待响应，可以继续执行其他操作。

**关键代码**：
```python
# 客户端
message_id = client.call_async("data_processor_service", "process_data", {"data": "task1"})
# ... 执行其他操作 ...
result = client.get_response(message_id, timeout=30)
```

**适用场景**：
- 耗时操作
- 需要并行处理多个任务
- 不立即需要结果的调用

### 场景3：发布-订阅模式（事件驱动）

**功能**：演示事件驱动的消息传递，多个订阅者可以同时接收同一个事件。

**关键代码**：
```python
# 发布者
client.publish("config_updated", {"config_path": "/path/to/config.yaml"})

# 订阅者
def on_config_updated(message):
    print(f"收到事件: {message.payload}")

bus.subscribe_events(["config_updated"], on_config_updated)
```

**适用场景**：
- 事件通知
- 一对多消息传递
- 解耦的服务间通信

### 场景4：服务注册与发现

**功能**：演示服务注册与发现，服务启动时自动注册，其他服务可以发现并调用。

**关键代码**：
```python
# 服务端：注册服务
registry.register("data_service", metadata={"version": "1.0.0"})

# 客户端：发现服务
service_info = registry.discover("data_service")
is_healthy = registry.check_health("data_service")
```

**适用场景**：
- 微服务架构
- 动态服务发现
- 服务健康监控

### 场景5：综合场景

**功能**：演示多个服务之间的交互，包括配置服务、数据服务、Web服务。

**特点**：
- 服务注册与发现
- 命令-响应模式
- 发布-订阅模式
- 多个服务协同工作

**适用场景**：
- 完整的微服务架构
- 复杂的服务交互
- 实际业务场景

## 配置说明

### Redis 配置

所有示例默认使用以下配置：
- **主机**：localhost
- **端口**：6379
- **数据库**：0
- **Key 前缀**：每个场景使用不同的前缀（如 `example_scenario1`）

如需修改，请编辑各示例文件中的 `BusConfig`：

```python
config = BusConfig(
    redis_host="localhost",      # 修改主机
    redis_port=6379,             # 修改端口
    redis_db=0,                  # 修改数据库
    key_prefix="example_scenario1"  # 修改前缀
)
```

### 服务名称

每个场景使用不同的服务名称，避免冲突：
- 场景1：`calculator_service`
- 场景2：`data_processor_service`
- 场景3：无服务（事件驱动）
- 场景4：`data_service`
- 场景5：`config_service`, `data_service`

## 常见问题

### 1. 连接 Redis 失败

**错误**：`ConnectionError: Error connecting to Redis`

**解决方案**：
- 确保 Redis 服务正在运行
- 检查 Redis 主机和端口配置
- 确认防火墙设置

### 2. 服务未找到

**错误**：`未找到服务: xxx_service`

**解决方案**：
- 确保服务端已启动并注册
- 检查服务名称是否匹配
- 等待服务注册完成（通常需要 1-2 秒）

### 3. 超时错误

**错误**：`TimeoutError: Command timeout`

**解决方案**：
- 增加超时时间：`client.call(..., timeout=60)`
- 检查服务端是否正常运行
- 检查 Redis 连接是否正常

### 4. 事件未收到

**问题**：订阅者未收到发布的事件

**解决方案**：
- 确保订阅者在发布者之前启动
- 检查事件类型名称是否匹配
- 确认使用相同的 key_prefix

## 扩展开发

### 添加新的处理器

```python
def handle_new_action(payload):
    # 处理逻辑
    return {"result": "ok"}

server.register_handler("new_action", handle_new_action)
```

### 订阅多个事件

```python
bus.subscribe_events(
    ["event1", "event2", "event3"],
    on_event
)
```

### 使用连接池

```python
config = BusConfig(
    use_connection_pool=True,
    connection_pool_size=10
)
```

## 最佳实践

1. **服务命名**：使用有意义的服务名称，避免冲突
2. **错误处理**：始终处理异常，避免程序崩溃
3. **超时设置**：根据业务需求设置合适的超时时间
4. **资源清理**：使用 `try-finally` 确保资源正确释放
5. **日志记录**：添加适当的日志，便于调试和监控
