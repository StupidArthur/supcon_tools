# 测试运行指南

## 快速开始

### 1. 环境准备

#### 检查 Redis 服务器

```bash
# Windows (PowerShell)
redis-cli ping
# 应该返回: PONG

# Linux/Mac
redis-cli ping
# 应该返回: PONG
```

如果 Redis 未运行，请先启动：

```bash
# Windows
# 启动 Redis 服务（如果已安装为服务）

# Linux
sudo systemctl start redis

# Mac
brew services start redis
```

#### 安装 Python 依赖

```bash
# 确保已安装 pytest 和 redis
pip install pytest redis

# 或者从项目根目录安装所有依赖
pip install -r requirements.txt
```

### 2. 运行测试

#### 方式 1：使用 pytest（推荐）

```bash
# 在项目根目录运行
pytest message_bus/tests/ -v

# 或者进入 tests 目录
cd message_bus/tests
pytest -v
```

#### 方式 2：使用 Python 模块

```bash
# 在项目根目录运行
python -m pytest message_bus/tests/ -v
```

#### 方式 3：使用测试脚本

```bash
# 在项目根目录运行
python message_bus/tests/run_tests.py
```

### 3. 运行特定测试

#### 运行单个测试文件

```bash
# 运行消息格式测试
pytest message_bus/tests/test_message.py -v

# 运行核心功能测试
pytest message_bus/tests/test_bus.py -v

# 运行性能测试（会输出性能指标）
pytest message_bus/tests/test_performance.py -v -s

# 运行并发测试
pytest message_bus/tests/test_concurrency.py -v -s
```

#### 运行特定测试类

```bash
pytest message_bus/tests/test_bus.py::TestMessageBus -v
```

#### 运行特定测试方法

```bash
pytest message_bus/tests/test_bus.py::TestMessageBus::test_send_and_receive_command -v
```

### 4. 测试输出选项

#### 详细输出（推荐）

```bash
pytest message_bus/tests/ -v
```

#### 显示 print 输出（性能测试需要）

```bash
pytest message_bus/tests/test_performance.py -v -s
```

#### 显示所有输出（包括日志）

```bash
pytest message_bus/tests/ -v -s --log-cli-level=DEBUG
```

#### 只显示失败的测试

```bash
pytest message_bus/tests/ -v --tb=short
```

## 常见问题

### 问题 1：Redis 连接失败

```
redis.exceptions.ConnectionError: Error connecting to Redis
```

**解决方案**：

1. 检查 Redis 是否运行：
   ```bash
   redis-cli ping
   ```

2. 检查 Redis 配置：
   ```bash
   # 查看 Redis 配置
   redis-cli CONFIG GET bind
   redis-cli CONFIG GET port
   ```

3. 如果 Redis 不在 localhost:6379，设置环境变量：
   ```bash
   # Windows PowerShell
   $env:TEST_REDIS_HOST="your_redis_host"
   $env:TEST_REDIS_PORT="6379"
   
   # Linux/Mac
   export TEST_REDIS_HOST="your_redis_host"
   export TEST_REDIS_PORT="6379"
   ```

### 问题 2：找不到模块

```
ModuleNotFoundError: No module named 'message_bus'
```

**解决方案**：

1. 确保在项目根目录运行：
   ```bash
   # 应该在项目根目录（包含 message_bus 目录的地方）
   cd f:\gitee\data_factory_next
   pytest message_bus/tests/ -v
   ```

2. 或者添加项目根目录到 Python 路径：
   ```bash
   # Windows PowerShell
   $env:PYTHONPATH="f:\gitee\data_factory_next"
   pytest message_bus/tests/ -v
   
   # Linux/Mac
   export PYTHONPATH="/path/to/data_factory_next"
   pytest message_bus/tests/ -v
   ```

### 问题 3：测试数据库冲突

如果其他模块也在使用 db=15，可以修改测试数据库：

```bash
# Windows PowerShell
$env:TEST_REDIS_DB="14"
pytest message_bus/tests/ -v

# Linux/Mac
export TEST_REDIS_DB=14
pytest message_bus/tests/ -v
```

### 问题 4：测试超时

```
TimeoutError: Command timeout
```

**解决方案**：

1. 检查 Redis 性能
2. 增加测试超时时间（修改测试代码中的 timeout 参数）
3. 检查系统负载

## 持续集成

### GitHub Actions 示例

```yaml
name: Test Message Bus

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    services:
      redis:
        image: redis:7
        ports:
          - 6379:6379
    
    steps:
      - uses: actions/checkout@v3
      
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.11'
      
      - name: Install dependencies
        run: |
          pip install pytest redis
          pip install -r requirements.txt
      
      - name: Run tests
        run: |
          pytest message_bus/tests/ -v
```
