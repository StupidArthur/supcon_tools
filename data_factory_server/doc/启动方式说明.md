# 启动方式说明

## 三种启动模式

项目提供三种启动模式，满足不同的使用场景：

### 1. WebService 联动启动（数据生成和实时模式）

**适用场景**：需要实时数据生成、Web API 交互、OPCUA Server 等功能

**启动方式**：
```bash
# 启动后端服务
python web_backend/start_server.py

# 或使用函数参数方式
python -c "from web_backend.start_server import start_server; start_server(host='0.0.0.0', port=8000)"
```

**功能**：
- 实时数据生成引擎（`services/realtime_runner.py`）
- Web API 接口（配置管理、参数修改、数据查询等）
- 数据管理功能（Redis 推送、历史存储等）
- OPCUA Server（通过 WebService 联动启动）

**特点**：
- 一站式服务，所有功能集成
- 支持动态配置修改
- 适合生产环境使用

### 2. 本地运行数据导出

**适用场景**：一次性数据导出，不需要实时交互

**使用方式**：
```python
from services.export_runner import run_export

# 执行导出
result = run_export(
    config_path="config/display_demo.yaml",
    steps=1000,
    template_name="moban_1",
    output_path="output.csv",
)
# 与 YAML 模板解耦的多格式导出（可选）：
# result = run_export(
#     dsl_content=open("config/display_demo.yaml", encoding="utf-8").read(),
#     steps=1000,
#     template_name="moban_1",
#     output_path="out.xlsx",
#     export_format={
#         "header_rows": 2,
#         "title_names": "timeStamp,时间戳",
#         "time_format": "%Y/%m/%d %H:%M:%S",
#         "file_format": "xlsx",
#         "sheet_name": "控制器",
#     },
# )
```

**功能**：
- 解析 DSL 配置
- 执行指定步数的数据生成
- 未传 `export_format` 时：按 `template_name` 加载 YAML 模板并导出 **CSV**
- 传入 `export_format` 时：按请求体导出 **csv / xlsx / xls**（需 `openpyxl`、`xlwt`）

**特点**：
- 轻量级，无需启动服务
- 适合批量数据导出
- 适合脚本化使用

### 3. 本地启动存储服务

**适用场景**：需要独立运行存储服务，从 Redis 读取数据并存储到 DuckDB

**启动方式**：
```bash
# 独立启动存储服务
python -m datacenter.run_storage_service

# 或使用函数参数方式
python -c "from datacenter.run_storage_service import run_storage_service; run_storage_service()"
```

**功能**：
- 从 Redis 读取实时数据
- 订阅组态信息事件（消息总线）
- 存储数据到 DuckDB
- 提供历史数据查询接口

**特点**：
- 独立进程运行
- 与 Engine 完全解耦
- 适合分布式部署

以下启动脚本和目录已删除，相关功能已整合到 WebService 中：

## 推荐使用方式

### 开发环境
- 使用 WebService 联动启动（`python web_backend/start_server.py`）
- 所有功能集成，便于开发和调试

### 生产环境
- 使用 WebService 联动启动（数据生成和实时模式）
- 独立启动存储服务（如果需要分布式部署）

### 批量导出
- 使用 `services/export_runner.py` 进行一次性数据导出
