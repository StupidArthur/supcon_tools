# Data Factory Next 用户手册 (V2.0)

## 1. 快速启动

本系统采用前后端分离架构，支持一键启动整个多引擎生态系统。

### 1.1 环境要求
- Python 3.8+ (推荐 3.10)
- Node.js & npm (用于前端界面)
- Redis Server (必须，用于数据总线)

### 1.2 一键启动
在项目根目录下运行以下脚本（Windows）：
```batch
start_system.bat
```
该脚本会自动：
1. 启动后端 FastAPI 服务（含所有引擎实例）。
2. 启动前端 Vite 界面。
3. 打开默认浏览器跳转至管理后台。

## 2. 系统配置 (engines_manifest.yaml)

所有的引擎实例与服务参数均在根目录下的 `engines_manifest.yaml` 中定义。

### 2.1 仿真引擎示例
```yaml
instances:
  - id: sim_boiler        # 引擎唯一 ID
    type: simulation     # 类型：仿真
    source: boiler.yaml  # DSL 逻辑文件路径
```

### 2.2 回放引擎示例
```yaml
instances:
  - id: history_player
    type: playback
    source: data/history_logs.xlsx
    sheet_name: SensorData
    time_col: Timestamp
    cycle_time: 1.0      # 播放步长（秒/行）
```

### 2.3 存储与协议配置
```yaml
storage:
  sample_interval: 5.0   # 全局采样间隔（秒）

opcua:
  publish_interval: 1.0  # OPC UA 更新频率
```

## 3. 操作指南

### 3.1 基础设施管理 (Infrastructure)
访问前端 `基础设施` 页面，你可以查看当前所有定义的引擎实例。点击“热重载 (Hot Reload)”按钮，系统将根据最新的配置文件重新编排所有引擎，无需重启进程。

### 3.2 服务状态与拓扑 (Service Status)
该页面提供实时诊断信息：
- **架构拓扑图**：动态展示各引擎（Producer）如何将数据推送到总线，以及存储服务（Storage）和 OPC UA 服务的消费情况。
- **引擎指标**：查看每个引擎的循环计数、仿真时间和 CPU 执行占比（Exec Ratio）。

### 3.3 数据查看 (Real-time View)
在实时运行页面，可以通过左上角的“引擎切换”下拉框，快速在不同的仿真实例/回放实例之间切换，查看其生成的最新快照。
- 实时快照统一来自 Redis V2 Hash `data_factory:v2:current`；系统不再依赖 V1 键 `data_factory:current`。

### 3.4 历史查询 (History)
系统会自动将所有引擎产生的位号存入 DuckDB。在历史页面中，你可以根据 `engine_id` 过滤并导出特定引擎的历史数据。

## 4. 开发者进阶

### 4.1 编写 DSL
请参考 `doc/dsl_rules.md` 了解如何编写计算逻辑。在 V2 版本中，你无需关心命名空间，`ConfigServer` 会自动将位号映射为 `engine_id.tag_name` 格式。

### 4.2 性能调优
如果你定义的位号数量超过 10 万个，建议：
- 增加 `storage.sample_interval` 以减轻磁盘吞吐压力。
- 检查 `Service Status` 中的 Exec Ratio，若超过 1.0，说明算法过于复杂，需要拆分为多个引擎并行运行。

## 5. 数据模拟与位号配置

### 5.1 默认曲线、绘图缩放与导出列
- 点击「模拟绘图」后，图表列**仅**由 DSL 中每条 `program` 的**非空** **`display_args`** 决定；未写 `display_args` 或 `display_args: []` 的项不参与默认曲线。可在 YAML 中使用 `"attr[ref]"` 形式：仅**曲线**按 `y_plot = y_raw*(100/ref)` 缩放，**导出 CSV 仍为原始值**。
- 图表卡片右上角可选 **单 Y 轴** 或 **自动双 Y 轴**（按缩放后的量级分组），便于多曲线同图。

### 5.2 数据生成（导出）
- 点击「数据生成」将打开**对话框**：左侧为 YAML **模板名列表**；系统会在打开弹窗和切换模板时，从后端实时加载该模板默认值并填充右侧。
- 右侧可编辑：**TITLE 行数**（1/2）、**TITLE 名**（1 行时为时间列表头整串；2 行时为英文逗号分隔两段，仅描述时间列第 1、2 行）、**时间戳格式**（`strftime`）、**文件格式**（`csv` / `xlsx` / `xls`）、**Sheet 名**（Excel，默认「控制器」）。
- 导出列仍默认与 DSL 非空 **`display_args`** 一致（请求体不传 `selected_variables` 时由后端 `get_display_variables()` 筛选）；未配置任何展示列时文件可能仅含时间等表头列。
- **依赖**：Excel 导出需安装 `openpyxl`（xlsx）、`xlwt`（xls），见 `doc/requirements.txt`。
- **接口**：`GET /export/format-defaults/{template_name}` 返回对话框预设；`POST /export/run` 可带 `export_format` 对象。`csv` 响应含文本字段 `file_content`；`xlsx`/`xls` 含 `file_content_base64` 与 `mime_type`、`filename`，供浏览器下载。

### 5.3 导出模板维护
- 模板文件目录：`components/export_templates/templates/`。
- **独立合成数据文件（可选）**：在工程根目录设置 `PYTHONPATH` 包含项目根后，可在代码中调用 `components.export_templates.generate_synthetic_export_file(tags, template, starttime, endtime, interval=5)`，按模板生成仅含锯齿波测试数据的 CSV/Excel，默认写入 `components/export_templates/generated_data/`。用于不跑仿真时的格式联调。
- 每个模板一个 YAML 文件，结构如下：
```yaml
name: example_template
defaults:
  header_rows: 1
  title_names: "timeStamp"
  time_format: "%Y/%m/%d %H:%M:%S"
  file_format: "csv"
  sheet_name: "控制器"
```
- 新增默认模板时，只需新增一个同结构 YAML 文件；刷新页面后即可在模板列表中看到。

### 5.4 PID 运行模式（MODE）
- `PID` 在 `init_args` 中可增加 **`MODE`**，默认 **`1`**。
- **`MODE == 1`**（按数值比较，支持 `1.0`）：每周期执行完整 PID 运算并更新 `MV`。
- **`MODE != 1`**：本周期**不执行** PID 运算（不更新 `MV`，也不累积积分、不推进微分状态）；`PV`/`SV` 仍可按 `execute(PV=..., SV=...)` 更新。
- 亦可在表达式中传入 `pid1.execute(..., MODE=某值)` 按周期覆盖（详见文档页「程序 / PID」参数表）。

### 5.5 水箱与阀门 DSL 要点
- 阀门：`valve_1.execute(target_opening=pid_1.MV, inlet_flow=source_flow)`，其中 `source_flow` 常为 `VARIABLE` 定义的常量或上游流量。
- 水箱：`tank_1.execute(inlet_flow=valve_1.outlet_flow)`；多级串联时将上游的 `outlet_flow` 接入下游的 `inlet_flow`。

### 5.6 TAG 信号程序（可写位号）
- 新增 `TAG` 程序，提供统一位号对：`in_value`（可写）与 `out`（输出）。
- 默认执行方式：`tag.execute()`，每周期透传 `out = in_value`。
- 可在表达式中连接输入：`tag.execute(in_value=other_tag.out)`。
- 写值请通过前端/接口对参数名 `in_value` 下发。
