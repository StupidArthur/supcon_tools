# 架构优化说明

## 优化概述

根据实际功能需求，移除了 `RealtimeDataService` 中间层，采用更简洁直接的架构设计。

## 优化后的架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Engine                                │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  RealtimePublisher                                   │   │
│  │  ├─> 直接写入 Redis Hash (data_factory:v2:current) │   │
│  │  └─> 消息总线发布组态信息事件 (config_update)       │   │
│  └──────────────────────────────────────────────────────┘   │
│  ┌──────────────────────────────────────────────────────┐   │
│  │  MessageServer (后台线程)                            │   │
│  │  └─> 接收 OPCUA 写值命令 (opcua_write_value)       │   │
│  └──────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
         │                    │                    │
         │                    │                    │
         ▼                    ▼                    ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  Redis K-V      │  │  消息总线       │  │  消息总线       │
│  (数据存储)     │  │  (组态信息)     │  │  (写值命令)     │
└─────────────────┘  └─────────────────┘  └─────────────────┘
         │                    │                    │
         │                    │                    │
         ▼                    ▼                    ▼
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│ StorageService  │  │ StorageService  │  │  OPCUA Server   │
│ 从 Redis 读取  │  │ 订阅组态信息    │  │ 订阅组态信息   │
│ 数据            │  │ 事件            │  │ 事件（可选）   │
└─────────────────┘  └─────────────────┘  └─────────────────┘
                                 │
                                 │ 写值命令
                                 ▼
                          ┌─────────────────┐
                          │  Engine         │
                          │  (处理写值)     │
                          └─────────────────┘
```

## 数据流向

### 1. 实时数据流

```
Engine → RealtimePublisher.push_snapshot()
  ├─> Redis Hash (data_factory:v2:current)  [直接写入]
  └─> Redis Pub/Sub (data_factory)          [通知更新]

StorageService / OPCUA Server
  └─> 从 Redis 读取 (data_factory:v2:current)
```

### 2. 组态信息流

```
Engine → RealtimePublisher.push_config()
  └─> 消息总线发布事件 (config_update)

StorageService
  └─> 订阅消息总线事件 (config_update)

OPCUA Server
  └─> 订阅消息总线事件 (config_update)  [可选]
```

### 3. OPCUA 写值流

```
OPCUA Client → OPCUA Server._on_write_value()
  └─> 消息总线发送命令 (engine, opcua_write_value)

Engine.MessageServer
  └─> _handle_opcua_write_value()
      └─> queue_param_update() / queue_variable_update()
```

## 主要改动

### 1. RealtimePublisher (`controller/realtime_publisher.py`)

**改动：**
- ✅ 统一写入 Redis Hash（`data_factory:v2:current`）
- ✅ `push_config()` 改为通过消息总线发布 `config_update` 事件
- ✅ 同时保留写入 Redis（向后兼容）

**关键代码：**
```python
def push_config(...):
    config_data = {...}
    
    # 方式1：通过消息总线发布事件（推荐）
    if self._client:
        self._client.publish(self.CONFIG_EVENT, config_data)
    
    # 方式2：写入 Redis（向后兼容）
    self._redis_client.set(self.CONFIG_KEY, json.dumps(config_data))
```

### 2. Engine (`core/engine.py`)

**改动：**
- ✅ 添加消息总线支持（用于接收 OPCUA 写值命令）
- ✅ 注册 `opcua_write_value` 命令处理器
- ✅ 在后台线程运行 `MessageServer`（不阻塞主线程）

**关键代码：**
```python
def enable_realtime_data(self, config, enable_message_bus=True):
    self._realtime_publisher = RealtimePublisher(config)
    self._push_config_to_redis()
    
    if enable_message_bus:
        self._init_message_bus(config)

def _handle_opcua_write_value(self, message):
    # 解析参数名，调用 queue_param_update 或 queue_variable_update
    ...
```

### 3. OPCUA Server (`datacenter/opcua_server.py`)

**改动：**
- ✅ 添加消息总线支持（用于发送写值命令）
- ✅ 实现 `_on_write_value()` 回调函数
- ✅ 节点创建时设置为可写（如果启用写值功能）

**关键代码：**
```python
async def _on_write_value(self, node, value, data_type):
    param_name = str(node.nodeid.Identifier)
    python_value = value.Value.Value
    
    # 通过消息总线发送命令给 Engine
    result = await loop.run_in_executor(
        None,
        lambda: self._client.call("engine", "opcua_write_value", 
                                 {"param_name": param_name, "value": python_value})
    )
```

### 4. StorageService (`datacenter/storage_service.py`)

**改动：**
- ✅ 添加消息总线支持（用于订阅组态信息事件）
- ✅ 订阅 `config_update` 事件（替代定期从 Redis 读取）
- ✅ 保留从 Redis 读取配置的方式（向后兼容）

**关键代码：**
```python
def _subscribe_config_events(self):
    def on_config_update(message):
        config_data = message.payload
        self._apply_config(config_data)
    
    self._bus.subscribe_events([CONFIG_EVENT], on_config_update)
```

## 配置说明

### Engine 配置

```python
from controller.engine import UnifiedEngine, EngineConfig
from controller.realtime_publisher import RealtimeConfig

engine = UnifiedEngine(engine_config)

# 启用实时数据发布（默认启用消息总线）
realtime_config = RealtimeConfig(
    redis_host="localhost",
    redis_port=6379,
    redis_db=0,
    bus_config=None  # 使用默认配置
)

engine.enable_realtime_data(realtime_config, enable_message_bus=True)
```

### StorageService 配置

```python
from datacenter import StorageService, StorageServiceConfig

config = StorageServiceConfig(
    redis_host="localhost",
    redis_port=6379,
    redis_db=0,
    db_path="storage_service.duckdb",
    use_message_bus=True  # 使用消息总线订阅组态信息

service = StorageService(config)
service.start()
```

OPCUA Server 通过 WebService 联动启动，无需独立配置。

## 优势对比

### 优化前（有 RealtimeDataService）

| 维度 | 说明 |
|------|------|
| 架构复杂度 | 高（多一层中间服务） |
| 数据流向 | Engine → 消息总线 → RealtimeDataService → Redis K-V |
| 性能 | 中等（多一次消息传递） |
| 维护成本 | 高（需要维护中间服务） |

### 优化后（无 RealtimeDataService）

| 维度 | 说明 |
|------|------|
| 架构复杂度 | 低（直接写入 Redis） |
| 数据流向 | Engine → Redis K-V（直接） |
| 性能 | 高（减少中间环节） |
| 维护成本 | 低（减少一个服务） |

## 向后兼容性

- ✅ **组态信息**：同时写入 Redis 和发布消息总线事件，支持旧版本 StorageService
- ✅ **数据读取**：StorageService 和 OPCUA Server 继续从 Redis 读取数据
- ✅ **配置选项**：StorageService 支持 `use_message_bus=False` 回退到 Redis 方式

## 注意事项

1. **消息总线依赖**：如果消息总线不可用，会自动回退到 Redis 方式
2. **OPCUA 写值**：需要 Engine 启用消息总线（`enable_message_bus=True`）
3. **组态信息**：建议使用消息总线方式（实时性更好），Redis 方式作为备份
