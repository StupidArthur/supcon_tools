# TPT DataHub 系统 UA 客户端测试用例

被测系统 = UA 客户端 + 实时数据管理 + 历史数据管理。

测试手段：
- mock server 可控注入值、改类型组态、启停模拟断连
- TPT API：getRTValue（实时）、query_history_value（历史）、writeTagValues（回写）
- asyncua 直连 mock，绕过 TPT 对照源端真实值
- ds-info / tag-info API 查询数据源和位号状态

用例编号规则：UA-{域}-{场景}-{序号}

三级详情按二级拆到 `test_cases/` 子目录下，每个二级一个文件。历史相关测试数据按 [`history-data-fixtures.md`](history-data-fixtures.md) 制造。

---

## 一级结构

| 编号 | 一级点 | 理由 |
|------|--------|------|
| UA-1 | 数据源管理 | 连接层：建立、启停、断线恢复、多源并存、删除和数据源测试 |
| UA-2 | 位号管理 | 配置层：新增、查询、导入导出、删除恢复和节点树管理 |
| UA-3 | 位号数据采集与读写 | 数据层：自动采集、两种实时读、实时写、历史查询、响应时间和性能 |

---

## 二级结构

### UA-1 数据源管理

| 编号 | 二级点 | 描述 |
|------|--------|------|
| UA-1-1 | 连接建立 | 从无到有建立数据源：URL、不可达地址、鉴权、质量码配置和重复地址拒绝 |
| UA-1-2 | 启停控制 | 数据源禁用、启用、重复启停、状态监控和循环稳定性 |
| UA-1-3 | 断线与自动重连 | 断线检测、自动恢复、延迟统计及断连期间操作 |
| UA-1-4 | 多数据源并存 | 多源采集、隔离、相同底层位号和系统位号名唯一性 |
| UA-1-5 | 数据源删除 | 删除矩阵、关联位号、删除后重建和隔离性 |
| UA-1-6 | 数据源测试功能 | ds-info/test 的枚举、源端实时读、库实时读、历史读和写值 |

### UA-2 位号管理

| 编号 | 二级点 | 描述 |
|------|--------|------|
| UA-2-1 | 位号新增 | 一次/二次/虚位号、数据源状态、名称规则、13 种类型、单位、只读、频率、限值、实时推送和新增后可用性 |
| UA-2-2 | 位号查询 | 基础列表、带实时值和质量查询、未导入位号浏览、过滤和分页 |
| UA-2-3 | 位号导入导出 | Excel 导出、Excel 导入、冲突策略和从数据源批量添加 |
| UA-2-4 | 位号删除 | 软删除、回收站恢复、物理删除、不可恢复、删除影响和删除后重建 |
| UA-2-5 | 位号节点管理 | 分组树查询、创建、编辑、移动、删除、位号移动、收藏和取消收藏 |

### UA-3 位号数据采集与读写

| 编号 | 二级点 | 描述 | 状态 |
|------|--------|------|------|
| UA-3-1 | 位号采集 | 源端值自动采集、类型、频率、质量、断线恢复、隔离和历史落地 | ✅ |
| UA-3-2 | 实时读取 | 实时库与数据库两种方式；按名称、ID、分组读取；异常和一致性 | ✅ |
| UA-3-3 | 实时写入 | 单个与批量写入、13 种类型、失败隔离、写后实时和历史闭环 | ✅ |
| UA-3-4 | 历史查询 | 两种历史接口、时间窗、分页、排序、采样、偏移、来源和完整性 | ✅ |
| UA-3-5 | 系统响应时间 | 常规负载下实时读、实时写和历史查询的延迟分布 | ✅ |
| UA-3-6 | 性能测试 | 实时读、实时写、历史查询的并发、吞吐、容量、混合负载和长稳 | ✅ |
