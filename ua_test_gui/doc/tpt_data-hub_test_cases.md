# TPT DataHub 系统 UA 客户端测试用例

被测系统 = UA 客户端 + 实时数据管理 + 历史数据管理。
本文件覆盖 **UA 客户端** 层（实时/历史 API 在此阶段作为验证工具使用）。

测试手段：
- mock server 可控注入值、改类型组态、启停模拟断连
- TPT API：getRTValue（实时）、query_history_value（历史）、writeTagValues（回写）
- asyncua 直连 mock，绕过 TPT 对照源端真实值
- ds-info / tag-info API 查询数据源和位号状态

用例编号规则：UA-{域}-{场景}-{序号}

三级详情按二级拆到 `test_cases/` 子目录下，每个二级一个文件。

---

## 一级结构

划分逻辑：功能测试按 UA 客户端能力域递进（连上 -> 管位号 -> 读对 -> 写对），非功能测试拆出延迟和吞吐两个维度。

### 功能测试

| 编号 | 一级点 | 理由 |
|------|--------|------|
| UA-1 | 数据源管理 | 连接层：能不能连上、断线能不能恢复、多源能不能并存 |
| UA-2 | 位号管理 | 配置层：连上后管哪些节点——注册、枚举、删除、配频率 |
| UA-3 | 数据采集 | 读路径：类型对不对、值准不准、周期稳不稳 |
| UA-4 | 数据回写 | 写路径：类型对不对、源端更新快不快、只读保护 |

### 非功能测试

| 编号 | 一级点 | 理由 |
|------|--------|------|
| UA-5 | 系统响应性 | 延迟类：值变化到 RT 反映多久、写到源端多久、API 响应时间 |
| UA-6 | 读写吞吐与容量 | 吞吐类：最大位号数、批量写上限、并发采集能力、压力下的表现 |

## 二级结构

### UA-1 数据源管理

| 编号 | 二级点 | 描述 |
|------|--------|------|
| UA-1-1 | 连接建立 | 从无到有建立数据源：URL 格式（无 path/有 path）、不可达地址、鉴权（有无凭据）、好值质量码配置、重复地址拒绝 |
| UA-1-2 | 启停控制 | 主动控制数据源生命周期：禁用运行中的数据源、重新启用、重复启停、状态监控、循环稳定性 |
| UA-1-3 | 断线与自动重连 | 被动故障恢复：服务端断开后的检测延迟、自动重连恢复延迟、5 次时延统计、断连期间写值/增删位号、短/长时间断连 |
| UA-1-4 | 多数据源并存 | 多数据源隔离性：同时采集、一源断连/启停不影响其他、跨数据源相同底层位号、系统位号名全局唯一 |
| UA-1-5 | 数据源删除 | 生命周期终结：禁用/启用 × 有/无位号 2×2 删除矩阵、位号进回收站后的删除、删除后同地址重建、对其他数据源的影响 |
| UA-1-6 | 数据源测试功能 | ds-info/test 接口本身的 5 种 testType：枚举位号、读实时值（源端/库）、读历史值、写值（不注册位号直接测） |

### UA-2 位号管理

| 编号 | 二级点 | 描述 |
|------|--------|------|
| UA-2-1 | 位号新增 | 注册一个或批量：按位号类型分类（一次/二次/虚）、数据源状态 4 种、底层位号 tagBaseName 规范与跨数据源、系统位号名 tagName 全局唯一、13 种数据类型的写入/读取/边界、单位、只读 2×2、采集频率、量程上下限超限处理、高/低限三档一致性、实时推送、位号描述、添加成功后可用性 |
| UA-2-2 | 位号查询 | 三种查询入口：tag-info/page（基础列表）、tag-group/queryWithQuality（带实时值+质量码+dsName）、tag-info/getNotUsedBaseTagInfoContinue（从数据源浏览未导入位号+游标分页）；按 dsId/tagName/groupId/tagBaseName 过滤；分页 |
| UA-2-3 | 位号导入导出 | Excel 导出（21 列结构，含实时值+限值）+ Excel 导入（conflictStrategy 0=跳过/1=覆盖）；从数据源批量添加位号（选未导入位号→批量提交） |
| UA-2-4 | 位号删除 | 软删（进回收站 groupId="1"）+ 物理删；删后重建同名位号；删除对采集的影响 |
| UA-2-5 | 位号节点管理 | 分组树 CRUD（创建/编辑/移动/删除，isForce 控制是否连带删位号）；收藏位号（groupId="2"）和取消收藏 |
