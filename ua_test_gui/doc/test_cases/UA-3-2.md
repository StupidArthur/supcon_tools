# UA-3-2 实时读取

验证 `getRTValue` 的实时库和数据库两种读取方式。仅设计现有 API、RT 和 asyncua 可执行的用例。

## 约定

- 实时库：`isFromDB=false`；数据库：`isFromDB=true`。
- 支持按 `tagNames`、`tagInfoIds`、`groupId` 查询。
- 动态值按允许时间窗口比较，不要求两种方式同毫秒一致。
- 临时位号使用 `ua32_{runId}_` 前缀。

| 编号 | 三级点 | 类型 | 步骤 | 断言 | 验证 |
|---|---|---|---|---|---|
| UA-3-2-001 | 实时库_按名称读取 | 回归 | 传 tagNames | 返回目标位号，值与源端一致 | RT + asyncua |
| UA-3-2-002 | 实时库_按ID读取 | 回归 | 传 tagInfoIds | ID 与返回位号正确对应 | RT + 配置 |
| UA-3-2-003 | 实时库_按分组读取 | 回归 | 传 groupId | 只返回该组位号 | RT + 分组集合 |
| UA-3-2-004 | 实时库_批量跨数据源 | 回归 | 查询 A、B 多个位号 | dsId、tagName、值不串源 | RT + asyncua |
| UA-3-2-005 | 实时库_13种类型与字段 | 回归 | 批量查询 13 种类型 | value、dataType、quality、时间字段正确 | RT + asyncua |
| UA-3-2-006 | 实时库_不存在名称 | 回归 | 查询不存在 tagName | 明确失败或空结果，无 5xx | 响应 |
| UA-3-2-007 | 实时库_不存在ID | 回归 | 查询不存在 ID | 不返回错误位号 | 响应 |
| UA-3-2-008 | 实时库_有效无效混合 | 回归 | 同批查询有效和无效目标 | 有效项保持正确，失败项可定位 | 响应 + RT |
| UA-3-2-009 | 查询条件_全部为空 | 回归 | 不传名称、ID、分组 | 明确拒绝，不误触发全量查询 | 响应 |
| UA-3-2-010 | 查询条件_重复目标 | 探索 | 重复传入名称或 ID | 记录去重/重复规则，不错配 | 响应集合 |
| UA-3-2-011 | 查询条件_多选择器并用 | 探索 | 同时传名称、ID、分组 | 记录并集、交集或优先级规则 | 响应集合 |
| UA-3-2-012 | 数据库_按名称读取 | 回归 | isFromDB=true 查询名称 | 返回数据库实时值，身份正确 | DB RT |
| UA-3-2-013 | 数据库_按ID和分组读取 | 回归 | 分别用 ID、groupId | 查询范围与选择器一致 | DB RT + 集合 |
| UA-3-2-014 | 两种方式_最终值一致 | 回归 | 源端停止变化后查询两种模式 | 允许落地窗口后值最终一致 | 两种 RT + asyncua |
| UA-3-2-015 | 两种方式_可见延迟 | 探索 | 源端写唯一值并轮询 | 记录两种模式首次可见时间差 | 时间线 |
| UA-3-2-016 | 两种方式_断线差异 | 探索 | 断线前后查询两种模式 | 记录缓存值、质量和时间差异 | ds 状态 + RT |
| UA-3-2-017 | 指定时间_queryTime | 探索 | 传有效 queryTime | 支持时返回对应结果，身份正确 | RT |
| UA-3-2-018 | 指定时间_option组合 | 探索 | 不同 option 与 queryTime | 记录采样规则；服务不异常 | 响应 |
| UA-3-2-019 | 指定时间_部分源不支持 | 回归 | 同批查询支持和不支持的数据源 | 失败项可定位，其他项正常 | 混合响应 |
| UA-3-2-020 | 删除恢复后的读取 | 回归 | 软删后查询，再恢复查询 | 删除后不正常返回；恢复后可读 | recycle + RT |
| UA-3-2-021 | 连续读取稳定性 | 回归 | 静态数据下执行 20 次 | 无随机失败，集合和值稳定 | 连续结果 |

## 执行与清理

1. 恢复分组关系和数据源状态；
2. 删除临时位号；
3. 保存实时库与数据库可见延迟；
4. 探索项保留完整请求、响应和时间线。
