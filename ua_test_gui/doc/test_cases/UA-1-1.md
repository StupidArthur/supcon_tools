# UA-1-1 连接建立

从无到有建立数据源：URL 格式（无 path/有 path）、不可达地址、鉴权（有无凭据）、好值质量码配置、重复地址拒绝。

---

## 可达案例

| 编号 | 三级点 | 前置条件 | 测试步骤 | 预期结果 | 验证手段 |
|------|--------|---------|---------|---------|---------|
| UA-1-1-01 | 正常连接(URL 无 path) | mock 已启动，端口可达 | 1. add_ds_info(url=opc.tcp://ip:port) 2. 启用 3. 注册位号 4. 等待采集 | alive=true；getRTValue 返回值 = mock 节点当前值 | list_ds_info + getRTValue |
| UA-1-1-02 | 正常连接(URL 有 path) | mock 已启动，endpoint 带 path | 同上，url=opc.tcp://ip:port/path | alive=true；getRTValue 返回值 = mock 节点当前值 | list_ds_info + getRTValue |
| UA-1-1-03 | 两种 URL 格式区别 | 同一 mock server | 1. 用 opc.tcp://ip:port 注册数据源 A 2. 用 opc.tcp://ip:port/path 注册数据源 B 3. 都启用 4. 各注册位号 | 两条数据源都 alive=true；验证是否视为不同数据源、位号是否独立采集 | list_ds_info 对比 + getRTValue |

## 好值质量码配置案例

| 编号 | 三级点 | 前置条件 | 测试步骤 | 预期结果 | 验证手段 |
|------|--------|---------|---------|---------|---------|
| UA-1-1-09 | 不配好值质量码 | mock 已启动 | 1. add_ds_info 不设好值质量码 2. 启用 3. 注册位号 4. 采集 | 验证 RT 的 quality 字段默认值是什么；值能否正常采集 | getRTValue 查 quality |
