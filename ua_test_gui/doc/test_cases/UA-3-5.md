# UA-3-5 系统响应时间

验证正常规模、低并发情况下实时读、实时写和历史查询的单次响应时间。首轮用于建立基线，产品给出 SLA 后再转为固定阈值回归。

需要历史数据的用例按[历史数据制造指南](../history-data-fixtures.md)执行。

## 测量规则

- 单执行线程；预热 5 次；正式执行至少 30 次。
- 使用单调时钟测量完整 HTTP 请求。
- 记录 min、mean、P50、P95、P99、max、错误率。
- 错误响应不能计作低延迟成功；每次必须验证返回数据正确。

## 结果要求

1. 首轮输出基线，不直接按毫秒判定通过。
2. 数据正确率必须为 100%。
3. 无随机 5xx、请求挂死或返回集合错乱。
4. 历史数据准备未完成时标记 `setup_failed`，不计入响应时间样本。
