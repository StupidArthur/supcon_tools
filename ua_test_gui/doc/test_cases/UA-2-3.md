# UA-2-3 位号导入导出

验证通过现有 API 导出已有位号、使用 Excel 批量创建或更新位号，并从数据源批量注册底层节点。

仅设计可由导入导出 API、查询 API、RT 和 asyncua 执行并验证的用例。前端文件选择器、按钮、跨页选择和交互提示暂不设计。

## 约定

- 导入成功必须回查数量、配置和可用性，不能只检查成功码。
- 导出按输入 ID 集合与文件内容核对，不依赖页面选择状态。
- 数据源批量注册前，用同一 `dsId + tagBaseName` 的已注册集合做差。
- 临时数据使用 `ua23_{runId}_`；覆盖原记录时保存并恢复快照。
- 历史值文件导入不属于本文件；超大容量和性能压力归 UA-6。

导出文件预期 21 列：Tag Name、Base Tag Name、Tag Type、Datasource Name、Unit、Data Type、Expression、Tag Value、Frequency、High Limit、HH Limit、HHH Limit、Low Limit、LL Limit、LLL Limit、Description、Group Name、Real-time Push、Readonly、Lo EU、Hi EU。

## GUI-DEFERRED

等待前端版本明确后再设计：文件选择器校验、上传/导出按钮状态、列表跨页选择、进度展示、用户确认和交互提示。

## 执行与清理

1. 导入前保存目标名称、ID 和配置快照；
2. 导出文件记录校验值，并读取原始单元格；
3. 导入后按名称和 ID 集合核对；
4. 新建记录删除，覆盖记录恢复；
5. 探索用例保存文件、请求响应和前后快照；
6. 清理失败标记为 `cleanup_failed`。
