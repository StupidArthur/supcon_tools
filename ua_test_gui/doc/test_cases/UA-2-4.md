# UA-2-4 位号删除

验证位号软删除、回收站恢复、物理删除、删除影响及删除后重建。仅设计现有 API、RT 查询和 asyncua 可执行的用例。

## 约定

- 软删除：`tag-info/batchDeleteLogic`；物理删除：`tag-info/batchDelete`。
- 回收站查询：`tag-group/get(groupId="1")`。
- 恢复：`tag-group/batchDelRelation`，请求体使用 `groupId="1"`；接口可能返回 `false`，以最终状态判定。
- 临时数据使用 `ua24_{runId}_`；写入前保存并恢复 UA 源值。
- GUI 确认框、按钮、页面提示暂不设计。

## 执行与清理

1. 删除前保存 ID、配置、分组、RT 和必要历史快照。
2. 恢复和物理删除均以最终状态判定，不只看返回值。
3. 物理删除后确认源端 UA 节点仍存在。
4. 重建用例结束后再次软删并物理删除。
5. 探索用例保存请求、响应和所有查询证据；清理失败标记 `cleanup_failed`。
