# UA-2-5 位号节点管理

验证位号节点树的查询、创建、编辑、移动、删除，以及位号移动和收藏关系。仅设计现有 API 可执行的用例；回收站恢复归 UA-2-4。

## 约定

- 分组树：`tag-group/groupTree`。
- 创建、更新、删除节点：`tag-group/add`、`tag-group/update`、`tag-group/batchDelete`。
- 位号移动：`tag-info/batchUpdate(groupId=...)`。
- 收藏与取消：`tag-group/batchAddRelation`、`tag-group/batchDelRelation(groupId="2")`。
- 更新接口返回内容可能陈旧，必须重新查询树确认。
- 临时节点使用 `ua25_{runId}_` 前缀；GUI 拖拽、展开状态和交互提示暂不设计。

## 执行与清理

1. 每条用例前保存完整树和相关位号配置快照。
2. 节点更新和关系删除必须通过重新查询确认，不能只信任返回体。
3. `isForce` 用例使用专用临时位号，避免影响共享环境。
4. 删除顺序从叶节点到父节点；收藏关系和位号分组在结束后恢复。
5. 探索用例保存请求、响应、树前后快照和位号最终状态；清理失败标记 `cleanup_failed`。
