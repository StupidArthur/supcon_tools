# UA-2-2 位号查询

验证用户能否从位号管理功能中准确找到目标位号、查看其配置与运行状态，并在数据变化后获得最新查询结果。

本文件按**用户查询任务和业务场景**组织，不按 API 拆分。API 仅用于构造数据、执行等价操作或独立核验结果。

---

## 设计约定

### 功能边界

本二级点覆盖：

1. 查看当前可管理的位号列表；
2. 按系统位号名、底层位号名、数据源和分组查找位号；
3. 使用多个条件组合缩小结果范围；
4. 查看位号配置、实时值、质量码和更新时间；
5. 浏览数据源中可注册的底层节点；
6. 在大量结果中分页、排序和刷新；
7. 新增、编辑、移动、收藏、删除和数据源状态变化后，查询结果正确更新。

不在本文件验证查询接口自身的内部实现细节。分页结构、字段层级和错误码仅作为功能断言的技术证据。

### 执行原则

- GUI 已提供对应操作时，优先通过 GUI 执行用户步骤；API 用于核验结果集合和字段。
