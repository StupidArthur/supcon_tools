# UA-2-2 位号查询

验证三种查询入口在基础查询、字段返回、过滤、分页、状态变化和跨入口一致性方面是否正确：

- `tag-info/page`：配置列表，不含实时值；
- `tag-group/queryWithQuality`：运行态列表，含实时值、质量码、时间戳和数据源名；
- `tag-info/getNotUsedBaseTagInfoContinue`：从数据源浏览底层节点，支持游标分页。

本文件沿用 `UA-2-1.md` 的编号、`area_title`、回归/探索/BLOCKED 和清理约定。

---

## 用例约定

### 查询事实源

1. 配置字段以 `tag-info/page` 为主要事实源；
2. 实时值、质量码、采集时间和 `dsName` 以 `queryWithQuality` 为主要事实源；
3. UA 底层节点是否真实存在，以 `getNotUsedBaseTagInfoContinue` 或 asyncua browse 为辅助事实源；
4. 三个入口返回同一位号时，使用 `id`、`tagName`、`tagBaseName`、`dsId` 做关联，不仅依赖列表顺序；
5. 查询类用例不得修改原始业务数据；需要创建测试位号时，使用唯一前缀 `ua22_{runId}_` 并在结束后清理。

### 基础测试数据

至少准备：

- 数据源 A、B：均已创建并运行；
- A 中 12 个已注册位号，覆盖两个分组、多个数据类型、不同 `tagName` 与 `tagBaseName`；
- B 中 3 个已注册位号，其中至少一个底层节点名与 A 相同；
- A 中至少 20 个未注册底层节点；
- 至少一个持续变化节点、一个可写静态节点；
- 至少一个软删除位号和一个收藏位号，用于可见性测试。

## 执行与证据要求

1. 每个查询请求保存请求参数、HTTP 状态、业务 code、响应耗时和原始响应摘要；
2. 分页用例保存每页的 `page/current/size/total/pages` 或 `continueID`；
3. 集合一致性比较统一使用稳定键，优先 `id`，底层节点使用 `dsId + tagBaseName`；
4. 动态 RT 比较必须设置刷新容差，不直接比较两个非同时刻请求的原始值；
5. 探索用例输出实际匹配规则、默认行为和错误信息，确认后再转为回归用例；
6. 查询过程创建的位号、分组、收藏关系和数据源状态必须恢复；清理失败单独记为 `cleanup_failed`。
