# 用户管理工具 用户手册

> 把 TPT 后台的用户管理 API（登录 / 列用户 / 单建 / 重置密码 / 批量 xlsx 导入）包成一个桌面 GUI，给运营 / 测试同学用。
>
> **当前版本**：v0.1
>
> designed by yzc（@yuzechao）

---

## 1. 这是什么

**用户管理工具**（Wails v2 桌面应用，Go + React 18）是对 TPT 后台「用户管理域」REST API 的 GUI 封装，覆盖五个常用动作：

- **登录**：URL / 账号 / 密码 / 租户 ID 一次填好，登录态在内存里复用
- **用户列表**：分页拉取 + 关键词搜索 + 一键刷新
- **新建用户**：表单创建单个后台用户
- **重置密码**：选中行 → 输入新密码 → 一键 reset
- **批量创建**：下载 xlsx 模板 → 填表 → 选文件 → 预览摘要 → 并发创建 + 实时进度

工具栏右下角有一行水印 `v0.1  designed by @yuzechao`，标明版本与作者。

## 2. 使用入门

### 2.1 最终用户怎么启动

工具以**单文件 exe** 发布，路径：

```
USER_MANAGER/build/bin/user-manager.exe
```

双击即可启动。启动后窗口标题 `用户管理工具`，默认尺寸 1180×780（最小 920×620），中文界面，无控制台窗口。

> Windows 10 / 11 64 位运行；不需要安装 Go / Node / Wails 运行时。
>
> Linux / macOS：v0.1 暂未出对应打包版，需要自行 `wails build` 跨平台编译（见 2.3）。

### 2.2 首次登录

启动后默认弹出**登录 TPT 后台**对话框，四个字段已预填：

| 字段 | 预填值 | 说明 |
|---|---|---|
| 服务器 URL | `https://supcontpt.supcon.com` | TPT 后台地址，HTTPS / HTTP 都行 |
| 账号 | `admin` | 默认占位，登录前可改 |
| 密码 | （留空） | 必填，每次启动要重新输入 |
| 租户 ID | `A54Z32M2` | HTTPS 多租户必填；HTTP 单租户环境留空 |

> 字段标签里的提示原文：「租户 ID（HTTPS 多租户必填，HTTP 留空）」。
>
> 登录成功后 URL / 租户 ID 会写入 `$HOME/.user-manager/config.json`，下次启动自动回填（账号、密码每次都空着）。

### 2.3 开发者怎么运行源码（可选）

仓库根目录进 `USER_MANAGER/`：

```bash
# 装好 Wails CLI 后：
wails dev          # 热重载开发模式（前端改了即时刷新）
wails build        # 产出 build/bin/user-manager.exe
```

`wails dev` 给改前端 / 改 Go 的开发者用；最终用户拿到的是 `wails build` 出来的 exe。

## 3. 功能介绍

### 3.1 总述 — 主界面长什么样

工具窗口分三块：

```
+---------------------------------------------------------------+
|  [搜索框]  [刷新]      [+ 新建用户]  [批量创建]  [登出]   v0.1 |
|                                                               |
|  [ID] [用户名] [昵称] [邮箱] [手机] [类型] [状态] [最后登录] [操作]
|  ...                                                          |
|  ...                                                          |
|                                                               |
+---------------------------------------------------------------+
```

- **顶栏（工具栏）**：搜索框 + 4 个按钮 + 右下水印
- **中间区**：用户列表表格，未登录时显示空
- **右下角**：Toast 提示区（操作成功 / 失败会飘字 3.5 秒后消失）

### 3.2 支持的操作系统

| 系统 | 状态 |
|---|---|
| **Windows** 10 / 11（64 位） | ✅ 已打包 + 实测 |
| Linux | ⚠️ 需自行 `wails build` 跨平台编译，未提供预编译 |
| macOS | ⚠️ 同上 |

打包产物是单文件，目标机不需要装 Go / Node / Wails 运行时。

### 3.3 工具栏按钮一览

| 按钮 | 文案 | 何时可用 | 行为 |
|---|---|---|---|
| 搜索框 | `搜索 username / nickName / phone / email` | 登录后 | 模糊匹配 4 个字段，回车触发刷新 |
| 刷新 | `刷新` / `刷新中…` | 始终 | 重新拉一次全量用户列表 |
| 新建用户 | `+ 新建用户` | 登录后 | 弹出新建用户对话框 |
| 批量创建 | `批量创建` | 登录后 | 弹出批量创建对话框 |
| 登出 | `登出` | 登录后 | 清内存登录态，保留 URL / 租户 ID，回到登录对话框 |

## 4. 登录

### 4.1 字段说明

| 字段 | 是否必填 | 默认 | 校验 |
|---|---|---|---|
| 服务器 URL | ✓ | `https://supcontpt.supcon.com` | 非空；按 http/https URL 走 |
| 账号 | ✓ | `admin` | 非空 |
| 密码 | ✓ | 空 | 非空；不持久化 |
| 租户 ID | （按部署） | `A54Z32M2` | HTTPS 多租户必填，HTTP 留空 |

按钮 `登录` 在三必填字段都非空时可点；点击后变 `登录中…` 防双击。

### 4.2 登录后会发生什么

1. 后端调 `POST /tpt-admin/system-manager/umsAdmin/login`，body 包成 `{"data": {...}}`（平台强约束）
2. 返回的 token 存进 App 内存的 `Client`，后续所有请求自动注入 `Bearer` header
3. URL / 租户 ID 写入 `$HOME/.user-manager/config.json`（权限 0600，不存密码）
4. 关闭登录对话框，自动调一次 `getAllUsers("")` 拉列表
5. Toast 飘 `登录成功` 或 `登录失败: [code] msg`

### 4.3 错误提示

登录失败时 Toast 显示形如 `登录失败: [A0400] 用户请求参数错误`，code + msg 原样透传平台返回值。

## 5. 用户列表

### 5.1 表格列

| 列 | 来源字段 | 说明 |
|---|---|---|
| ID | `id` | 后台主键 |
| 用户名 | `username` | 登录账号 |
| 昵称 | `nickName` | 显示名 |
| 邮箱 | `email` | 可空，表格留空 |
| 手机 | `phone` | 可空，表格留空 |
| 类型 | `type` | `type=0` 显示「管理员」，其它显示「普通」 |
| 状态 | `status` | `status=0` 显示绿色「启用」 tag，否则红色「禁用」 tag |
| 最后登录 | `loginTime` | 平台返回时间字符串；空显示 `-` |
| 操作 | — | 每行一个 `重置密码` secondary 按钮 |

### 5.2 搜索规则

- 搜索框 placeholder：`搜索 username / nickName / phone / email`
- 按 `Enter` 触发搜索；`刷新` 按钮等价于带当前关键词的全量刷新
- 后端对 `nickName / username / phone / email` 这 4 个字段做 `LIKE '%keyword%'` 模糊匹配（任一字段命中即返回）
- 关键词留空 = 不过滤，返回当前租户全部用户

### 5.3 翻页

v0.1 **没有显式分页 UI**。后端自动翻页拉完整个租户的所有用户（`pageSize=200`，循环到 `len(records) < pageSize` 为止），单租户上千用户也能一次看完。

## 6. 新建用户

### 6.1 表单字段

点击工具栏 `+ 新建用户` 弹窗，字段：

| 字段标签 | 必填 | 前端校验 | 后端校验 |
|---|---|---|---|
| 用户名 `*` | ✓ | 非空 | 平台判定是否重复 |
| 初始密码 `*` | ✓ | 非空 | 平台密码策略（复杂度由平台侧决定） |
| 昵称 `*` | ✓ | 非空 | — |
| 邮箱 | | 客户端不做格式校验，靠服务端 | 非空时含 `@`（来自 xlsx 解析侧约束，单条表单不强制） |
| 手机 | | 客户端不做长度校验，靠服务端 | 非空时长度 ≥ 6（同上） |

带 `*` 是必填；只有三必填都填了，按钮 `创建` 才可点。

### 6.2 默认参数（不可选）

对话框底部有个折叠块 `默认参数（不可选 · v1 限制）`，展开后看到 7 行硬编码值：

| 字段 | 写死的值 | 备注 |
|---|---|---|
| 状态 | `启用 (status=0)` | 创建后立即启用 |
| 类型 | `普通用户 (type=2)` | admin (type=0) 暂不支持创建 |
| 组织 | `默认组织 (orgId=1)` | 仅支持默认组织 |
| 角色 | `默认角色 (roleId=5)` | 枚举待确认；切换角色需补 `i_list_roles` |
| 性别 | `男 (gender="1")` | 平台枚举未核实 |
| 头像 | `无 (icon="")` | — |
| code | `= username` | 自动用 username 填充 |

> 这些字段当前为硬编码默认值。如需修改请联系开发者更新功能（需补枚举端点 + 表单控件 + 模板列）。

### 6.3 操作流程

1. 工具栏点 `+ 新建用户`
2. 填 5 个字段（至少前三必填）
3. 点 `创建`
4. 成功：Toast `创建成功: <username>`，对话框关闭，列表自动刷新
5. 失败：Toast `创建失败: [code] msg`，对话框保持打开，可改完再提交

## 7. 重置密码

### 7.1 操作流程

1. 在用户列表某行点 `重置密码`（secondary 按钮）
2. 弹出 `重置密码` 对话框，标题下显示 `用户：<username> (<nickName>)`
3. 在 `新密码` 输入框输入新密码（input 自动获焦）
4. 点 `重置`（primary）
5. 成功：Toast `密码已重置（旧密码仍可能有效 — 平台行为）`，对话框关闭

### 7.2 平台怪行为（必读）

对话框底部一行红字提示：

> 平台行为：reset 后旧密码仍然有效

后端调 `POST /xpt-system/api/system-manager/umsAdmin/resetPwd`，body 形如 `{"data": {"id": ..., "newPwd": ..., "confirmPwd": ...}}`。平台返回成功 ≠ 旧密码失效——reset 后**新旧密码都能登**（属于平台行为，工具无法规避）。

### 7.3 字段校验

前端只校验「新密码非空」；复杂度（如大小写、长度、特殊字符）由平台决定，工具不做硬约束。

## 8. 批量创建

工具栏点 `批量创建`（primary 按钮），弹出 480px 宽的对话框，包含三步：**选文件 → 看摘要 → 启动**。

### 8.1 第一步：选 xlsx

对话框顶部一行三个控件：

| 控件 | 文案 | 行为 |
|---|---|---|
| 按钮 | `选择 xlsx 文件` / `解析中…` | 弹出系统文件选择框；选完后自动解析 |
| 按钮 | `下载模板` | 弹出系统保存框，把内置 xlsx 模板（含 1 行示例）写到用户选的位置 |
| 文件名 | 选中后显示路径最后一段 | 只读，hover 看完整路径 |

### 8.2 xlsx 模板格式

模板由工具**现场生成**（不走仓库打包二进制），结构如下：

| 列 | 字段 | 必填 | 示例（模板自带第 2 行） |
|---|---|---|---|
| A | username | ✓ | `zhangsan` |
| B | password | ✓ | `Init@2026` |
| C | nickName | ✓ | `张三` |
| D | email | | `zhangsan@example.com` |
| E | phone | | `13800138000` |

列宽预设：`username/password/nickName = 16`，`email = 24`，`phone = 14`。

> 第 1 行 = 表头，**不要删**；第 2 行是示例，可以直接覆盖或在它后面追加。
>
> 文件名默认建议 `batch_users_template.xlsx`（保存框的 `DefaultFilename`）。

### 8.3 解析与摘要视图

选完 xlsx 后，对话框显示一段摘要（背景色块）：

```
共 100 条   ✓ 95 条可创建   ✗ 5 条有问题
```

- **共 N 条**：xlsx 数据行总数（不含表头）
- **✓ X 条可创建**：通过所有校验的行
- **✗ Y 条有问题**：校验未通过的行；点击 `查看 N 条行级错误` 折叠展开，最多可见 5 条起，可滚动查看

行级校验规则（来自 `internal/excel/parser.go:validate`）：

| 规则 | 触发文案 |
|---|---|
| `username` 为空 | `username 不能为空` |
| `password` 为空 | `password 不能为空` |
| `nickName` 为空 | `nickName 不能为空` |
| `email` 非空但不包含 `@` | `email 格式不对` |
| `phone` 非空且长度 < 6 | `phone 长度过短` |

文件级错误（缺列等）会在摘要块里直接红字列出，例如 `missing required column "username" in header`。

### 8.4 并发度

摘要下面一行：

```
并发数：[  3  ] （1-20）
```

- 默认 `3`，可手输 1–20 的整数（输入框自动 clamp）
- **建议** 默认 3；并发超过 5 会触发平台限流风险（实测），高并发不是更快而是被节流
- 选 1 = 严格串行；选 20 = 满速（慎用）

### 8.5 启动与进度

`开始批量创建（N 条）` 按钮：

- N = 摘要里的「可创建」数
- 当有效行数 = 0 时按钮禁用
- 点击后关闭摘要对话框，弹出**批量创建进度**对话框（最小宽度 720px）

进度对话框内容：

| 元素 | 说明 |
|---|---|
| 进度文字 | `进度：<done> / <total>（<pct>%）` |
| 标签 | 绿色 `<succeeded> 成功` / 红色 `<failed> 失败` |
| 进度条 | 按百分比填充，0.2s 缓动 |
| 实时表 | 每完成一条追加一行：`行 / 用户名 / 昵称 / 状态 / 消息` |
| 取消按钮 | 红色 `danger`，未完成时可点；点完发请求取消，结果仍可见 |
| 完成后 | 按钮变 `关闭`（secondary）+ `刷新列表`（primary） |

进度靠 Wails 事件 `batch:progress`（每条完成一次）+ `batch:done`（全部完成）推送到前端。

### 8.6 完整操作流程示例

假设要给 50 个新员工开账号：

1. 工具栏点 `批量创建` → 弹出批量对话框
2. 点 `下载模板` → 保存为 `employees.xlsx`，表格已含表头 + 1 行示例
3. 在 Excel 里删掉示例行，按 `username / password / nickName / email / phone` 5 列填好 50 行
4. 回到工具，点 `选择 xlsx 文件` → 选 `employees.xlsx`
5. 看摘要：`共 50 条 ✓ 48 条可创建 ✗ 2 条有问题`
6. 展开折叠区，看到 `行 12: email 格式不对`、`行 37: phone 长度过短` —— 回去改 xlsx 重选，或直接继续（错的 2 条会被跳过）
7. 并发数保持默认 3
8. 点 `开始批量创建（48 条）` → 进度对话框弹出
9. 看进度条涨 / 表里逐条变绿 → 完成后 `关闭` 或 `刷新列表` 回到主界面

## 9. 已知限制（v0.1）

| 项 | 状态 |
|---|---|
| **不**支持编辑已有用户（UPDATE） | v1 没做；端点待用户给样本 |
| **不**支持删除用户（DELETE） | 同上 |
| **不**支持角色 / 组织管理 | `i_list_orgs` / `i_list_roles` 端点未接，硬编码默认值 |
| **不**支持 Shadcn UI | shadcn init 是交互式 CLI 重型，v0.1 跳过；自写 CSS Notion 风 |
| **不**支持用户自改密码 | 平台有独立 `/changePwd` 端点，v1 没接 |
| 批量对话框**不**做行内编辑 | 原设计 5 列 input × N 行表格，因「界面精简一点」用户反馈改成摘要视图 |
| 批量任务**不**支持崩溃恢复 | 进程崩溃后已完成的条数不落盘，v2 计划补 batch state file |
| 单元测试覆盖率报告 | v0.1 未配 |
| 平台密码 reset 后旧密码仍有效 | 平台行为，工具无法规避（重置成功 ≠ 旧密码失效） |

## 10. 配置文件

工具只在登录成功后持久化两项**非敏感**配置：

- 路径：`$HOME/.user-manager/config.json`（Windows 上 = `C:\Users\<你>\.user-manager\config.json`）
- 格式：JSON
- 权限：0600（仅 owner 可读写）
- 字段：`url` + `tenantId`，**不存密码**
- 文件不存在时按代码内默认值（见 4.1）启动

示例内容：

```json
{
  "url": "https://supcontpt.supcon.com",
  "tenantId": "A54Z32M2"
}
```

登出**不会**清这个文件，下次启动 URL / 租户 ID 还会自动回填。删除文件后重启即恢复默认占位值。

## 11. 常见问题

**Q：登录提示 `A0400 用户请求参数错误` 怎么办？**
A：检查 4 个字段是否填全（特别是租户 ID），URL 是否带 `https://`。如果平台返回该错误码，通常是租户 ID 不匹配。

**Q：批量对话框里「文件级错误：missing required column」怎么修？**
A：xlsx 表头**必须**包含 `username / password / nickName`（区分大小写），且这是第一行。重新保存 xlsx 时确认表头行没被合并 / 删除。

**Q：重置密码成功了，但用户反馈旧密码还能登？**
A：见 7.2，这是平台行为，工具只负责发请求，平台侧的密码历史策略不受工具控制。

**Q：批量创建跑到一半卡了 / 崩了，怎么办？**
A：v0.1 没有崩溃恢复（见第 9 节）。已完成的条目已在平台侧真实创建，重新跑会因 username 重复报错——可以选 `开始批量创建（X 条）` 后看进度表，把失败的 username 剔除后重跑，或直接用「刷新列表」看实际结果。

**Q：能改成管理员账号吗？**
A：v0.1 创建的类型写死为 `type=2`（普通用户，见 6.2），不能直接建管理员账号；如确有必要，需找平台管理员在后台手工改。

**Q：能多租户切换吗？**
A：v0.1 一次只连一个租户，切换租户 = 改登录对话框的「租户 ID」字段重新登录。
