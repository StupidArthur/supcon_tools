# ua_player 用户手册

> 把 CSV 数据变成 OPC UA Server，工业客户端（UAExpert / KEPServer / SCADA 等）按时间戳或固定间隔读到这些值。
>
> **当前版本**：v7
>
> designed by yzc

---

## 1. 这是什么

ua_player 是一款 OPC UA 数据播放器：把本地 CSV 文件加载到一个 OPC UA Server，按时间戳或固定间隔循环推送到工业客户端。

---

## 2. 使用入门

ua_player 是命令行工具，最简单的调用只需要一个 CSV 文件。

### 2.1 命令格式与参数

```
ua_player_v7.exe <data_file> [--port <port>] [--interval <seconds>]
```

> 上面是打包版（exe）的命令；用源码运行是 `python main.py ...`，参数完全一样。下面所有示例都按打包版写。

参数说明：

- `<data_file>`（必填）— CSV 文件路径。CSV 怎么写见第 4 节；最简可用的示例见 4.4。
- `--port`（选填，默认 `18950`）— OPC UA 监听端口。改了之后客户端那边也要跟着用新端口。
- `--interval`（选填）— 固定播放间隔（秒/行）。传了就**忽略 CSV 第一列的时间戳**。详细行为见第 4.6 节。

**一个具体的案例**：你手上有一个 `pid_demo.csv`，想用 ua_player 让它按 1 秒一行的速度播给 UAExpert 看，并换到 4840 端口：

```cmd
ua_player_v7.exe pid_demo.csv --port 4840 --interval 1.0
```

执行后 ua_player 在 4840 端口监听，UAExpert 用 `opc.tcp://127.0.0.1:4840/` 连上即可看到所有节点，每秒更新一次。

### 2.2 停止

按 `Ctrl + C` 中断。

### 2.3 客户端连上去

默认 `opc.tcp://127.0.0.1:18950/`，跨机时把 `127.0.0.1` 换成 ua_player 运行机的 IP，端口随 `--port` 改变。

## 3. 功能介绍



### 3.1 总述 — 播放数据的逻辑

ua_player 把 CSV 数据变成 OPC UA Server 节点，按节奏循环推送。流程：

1. 启动时一次性把 CSV 加载到内存
2. 在 OPC UA 命名空间 `ns=1` 下创建容器对象 `MYDATA`
3. 为 CSV 每一列创建变量节点（浮点 / 布尔），挂在 `MYDATA` 下
4. 进入播放循环：按时间戳差分或固定间隔，从第 1 行开始逐行写入所有节点
5. 播完最后一帧回到第 1 行继续循环
6. 客户端连上后可订阅或读取，写入则对带 `[w]` 后缀的节点生效

整个流程在 **asyncio 事件循环**里跑，单进程；OPC UA 通信在事件循环内，CSV 加载在启动时同步完成。

### 3.2 支持的操作系统

- **Windows** 10 / 11（64 位）
- **Linux**：CentOS、Ubuntu；龙蜥（Anolis）需要匹配 GLIBC 版本

打包产物是单文件，目标机不需要装 Python。

注意：

- 跨 Linux 发行版需要在 **glibc 同档或更旧** 的环境上打包，否则会报 `GLIBC_x.xx not found`
- `/tmp` 挂载为 noexec 的系统，打包时需调整 `runtime_tmpdir`（当前已默认设为 `/var/tmp/ua_player_pyi`）



### 3.3 支持的数据类型

- **浮点**（默认）：CSV 单元格的浮点数字符串
- **布尔**：CSV 列名带 `[bool]` 后缀；值为 `1` / `true` / `yes` / `是` 视为 true，其他视为 false
  - **建议统一用** `1` **/** `0`：虽然 `true` / `yes` / `是` 都能解析，但很多下游工具（Excel 导出、SCADA、组态软件）只认 `1` / `0`，统一写法兼容性最好

**不**支持：整数、字符串、日期时间、数组等其它类型。如果 CSV 里有这些列，它们不会出现为 OPC UA 节点。

### 3.4 客户端读写支持

所有节点客户端**都能读**。

写入只对带 `[w]` 后缀的列开放：


| CSV 表头          | 客户端能读 | 客户端能写 |
| --------------- | ----- | ----- |
| `name`          | ✅     | ❌     |
| `name[bool]`    | ✅     | ❌     |
| `name[w]`       | ✅     | ✅     |
| `name[bool][w]` | ✅     | ✅     |


**可写节点的具体行为**：

- **默认值 = CSV 第 1 行的值**：服务启动时按 CSV 第 1 行初始化，运行中不变化（除非被客户端写入）
- **不参与播放循环**：服务端在循环写值时**不会**改写可写节点
- **写入后保持**：客户端写入的值会一直保留，直到下次客户端写入或服务重启
- **时间戳为写入当时**：写入时的 SourceTimestamp / ServerTimestamp 都由服务端覆盖为当前 UTC 时间，用于规避某些客户端不带时间戳导致显示 1601 年



### 3.5 多实例的支持

可以同时跑多个 ua_player，给不同的 `--port`：

```cmd
ua_player_v7.exe dataset_a.csv --port 18950
ua_player_v7.exe dataset_b.csv --port 18951
ua_player_v7.exe dataset_c.csv --port 18952
```

各实例相互独立，监听不同端口，客户端可分别连。

注意：

- 多个实例的 CSV 不应包含同名节点，否则各实例的 OPC UA 地址空间都会出现同名 NodeId
- 资源消耗随实例数线性增加，注意机器负载



### 3.6 播放速度自定义

两种模式：

**时间戳模式**（默认）

按 CSV 第一列时间戳差分播放，节奏由 CSV 数据决定。CSV 里两行间隔 5 秒，就每 5 秒播一行。

**固定间隔模式**（`--interval`）

按指定秒数播放一行，**忽略** CSV 第一列的时间戳。适用于需要快放 / 慢放，或与其它系统对齐节奏的场景：

```cmd
ua_player_v7.exe data.csv --interval 0.5   :: 0.5 秒一行
ua_player_v7.exe data.csv --interval 5.0   :: 5 秒一行
```



### 3.7 命名空间定义

- **命名空间索引**：`ns=1`（固定，不可改）
- **容器对象**：`ns=1;s=MYDATA`
- **变量节点**：`ns=1;s=<节点名>`，节点名 = CSV 表头去掉 `[bool]` / `[w]` 后缀

浏览路径：

```
Objects
  └── MYDATA
       ├── tagname_1    ← CSV 表头 "tagname_1"
       ├── tagname_2    ← CSV 表头 "tagname_2[bool]"
       └── tagname_3    ← CSV 表头 "tagname_3[w]"
```

## 4. 准备输入：CSV 文件



### 4.1 格式

- **编码**：UTF-8
- **第一行是表头**：第一列**可空**，从第二列起为**节点名**
- **第二行起是数据**：第一列为**时间戳**，从第二列起为各节点的值



### 4.2 节点名后缀


| 表头写法                 | 节点名         | 类型  | 可写  |
| -------------------- | ----------- | --- | --- |
| `tagname_1`          | `tagname_1` | 浮点  | 否   |
| `tagname_2[bool]`    | `tagname_2` | 布尔  | 否   |
| `tagname_3[w]`       | `tagname_3` | 浮点  | 是   |
| `tagname_4[bool][w]` | `tagname_4` | 布尔  | 是   |


> NodeId / BrowseName 看到的永远是去后缀的"干净"名字。`tagname_2[bool]` 在 OPC UA 客户端里就叫 `tagname_2`。



### 4.3 时间戳格式（任选其一）

- `2025/01/01 00:00:00`
- `2025/01/01 00:00`
- `2025-01-01 00:00:00`
- `2025-01-01 00:00`

> 月日可单位数（如 `2025/1/3`），时分秒不补零会按 `strptime` 规则解析；通常建议补零，避免歧义。



### 4.4 一个完整可用的示例（直接复制保存为 `.csv`）

```csv
timeStamp,tagname_1,tagname_2[bool],tagname_3[w],tagname_4[bool][w]
2025-01-01 00:00:00,25.0,0,100.0,0
2025-01-01 00:00:02,25.5,0,100.0,0
2025-01-01 00:00:05,26.0,1,100.0,0
2025-01-01 00:00:09,26.5,1,100.0,0
```

行为：

- 第 1→2 行间隔 2 秒
- 第 2→3 行间隔 3 秒
- 第 3→4 行间隔 4 秒
- 播完最后一帧后**停留 4 秒**（与上一条间隔相同），再回到第 1 行循环



### 4.5 布尔值怎么写

以下写法都算 true：`1`、`true`、`yes`、`是`（大小写不敏感）。其他写法都算 false。

### 4.6 用固定间隔播放

如果启动时传了 `--interval 0.5`：

- CSV 第一列的时间戳**不再生效**，每 0.5 秒播一行
- 第一列**填什么都行**（甚至乱填），但**这一列要保留**，否则表头对不上
